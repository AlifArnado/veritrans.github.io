Installment

Installment is a type of transaction charge that allows a payment transaction to be conducted regularly in a certain period of time. Installment is only applicable for transactions using a credit card. The installment function specification is the same as the specification of transaction charge. Since the tutorial for implementing installment on VT-Web and VT-Direct are quite different, the tutorial will be explained separately.

  • VT-Web
  • VT-Direct


Similar to a VT-Web transaction charge, output from the installment function is payment URL. Customer has to be redirected to the VT-Web payment URL to complete the transaction proccess. Implementing the installment function on VT-Web is very simple, you only need to add the payment_options and installment function on the JSON request of the transaction charge. The following is the JSON request for the installment function on VT-Web: 

{
	// Required
	"payment_type": "vtweb",
	// Required
	"vtweb": {
		"credit_card_3d_secure": true,
		"enabled_payments": ["credit_card"],
		"payment_options": {
			"installment": {
				"required": true,
				"installment_terms": {
					"bni": [],
					"mandiri":[6, 12]
				}
			}
		}
	},
	// Required
	"transaction_details": {
		"order_id": "K17550",
		"gross_amount": 145000
	},
	// Optional
	"customer_details": {
		"first_name": "Andri",
		"last_name": "Litani",
		"email": "andri@litani.com",
		"phone": "081122334455",
	// Optional
	"billing_address": {
		"first_name": "Andri",
		"last_name": "Litani",
		"address": "Mangga 20",
		"city": "Jakarta",
		"postal_code": "16602",
		"phone": "081122334455",
		"country_code": "IDN"
		},
	// Optional
	"shipping_address": {
		"first_name": "Obet",
		"last_name": "Supriadi",
		"address": "Manggis 90",
		"city": "Jakarta",
		"postal_code": "16601",
		"phone": "08113366345",
		"country_code": "IDN"
		}
	},
	// Optional
	"item_details": [
		{
			"id": "a1",
			"price": 50000,
			"quantity": 2,
			"name": "Apel"
		},
		{
			"id": "a2",
			"price": 45000,
			"quantity": 1,
			"name": "Jeruk"
		}
	]
}


Take note that when the installment term of BNI bank is an empty array, as found in specification, all valid installment terms will be displayed on VT-Web.

The JSON response structure in installment function is the same as that of the VT-Web transaction charge. Below is the a successful JSON response: 

{
	"status_code": "201",
	"status_message": "OK, success do VTWeb transaction, please go to redirect_url",
	"redirect_url": "https://vtweb.sandbox.veritrans.co.id/v2/vtweb/550ad19d-e01b-4c8f-945a-67902c7b83da"
}


For VT-Direct you have to acquire a token ID. Token ID represents customer's credit card information that has been stored inside Veritrans server. To get the token, you should follow the tutorial on VT-Direct – get the Token. With some concerns, the parameter request token for installment is different than a normal transaction token. You should add installment and installment_term on the parameter.

After you obtain the token, installment transaction charge can be done by adding the installment_term attribute on credit_card: 

{
	// Required
	"payment_type": "credit_card",
	// Required
	"credit_card": {
		"bank": "bni",
		"token_id": "< your installment token ID >",
		"installment_term": 12
	},
	// Required
	"transaction_details": {
		"order_id": "K17550",
		"gross_amount": 145000
	},
	// Required
	"customer_details": {
		"first_name": "Andri",

		"email": "andri@litani.com",
		"phone": "081122334455",
