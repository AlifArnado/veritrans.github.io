Installment

Installment is a type of transaction charge that allows payment transaction to be conducted regularly in certain period of time. Installment is only applicable for transaction using credit card. Installment function specification is the same as specification of transaction charge. Since the tutorial of implementing installment on VT-web and VT-Direct are quite different, the tutorial will be explained separately.

  • VT-Web
  • VT-Direct

Similar to VT-Web transaction charge, output from installment function is payment URL. Customer have to be redirected to VT-Web payment URL to complete the transaction proccess. Implementing installment function on VT-Web is very simple, you only need to add payment_options and installment function on JSON request of transaction charge. The following is JSON request for installment function on VT-Web:

{
    // Required
    "payment_type": "vtweb",
    // Required
    "vtweb": {
        "credit_card_3d_secure": true,
        "enabled_payments": ["credit_card"],
          "payment_options": {
            "installment": {
              "required": true,
              "installment_terms": {
                "bni": [],
                "mandiri":[6, 12]
            }
          }
        }
    },
    // Required
    "transaction_details": {
		"order_id": "K17550",
		"gross_amount": 145000
	},
    // Optional
    "customer_details": {
		"first_name": "Andri",
		"last_name": "Litani",
		"email": "andri@litani.com",
		"phone": "081122334455",
	// Optional
	"billing_address": {
		"first_name": "Andri",
		"last_name": "Litani",
		"address": "Mangga 20",
		"city": "Jakarta",
		"postal_code": "16602",
		"phone": "081122334455",
		"country_code": "IDN"
		},
	// Optional
	"shipping_address": {
		"first_name": "Obet",
		"last_name": "Supriadi",
		"address": "Manggis 90",
		"city": "Jakarta",
		"postal_code": "16601",
		"phone": "08113366345",
		"country_code": "IDN"
		}
	},
    // Optional
    "item_details": [
		{
			"id": "a1",
			"price": 50000,
			"quantity": 2,

		},

		}
    ]
}

Take note that when the installment term of BNI bank is an empty array, as specified on specification, all valid installment term will be displayed on VT-Web.

JSON response structure on installment function is the same as VT-Web transaction charge. Below is the successful JSON response:

{
    "status_code": "201",
    "status_message": "OK, success do VTWeb transaction, please go to redirect_url",
    "redirect_url": "https://vtweb.sandbox.veritrans.co.id/v2/vtweb/550ad19d-e01b-4c8f-945a-67902c7b83da"
}

For VT-Direct you have to get token ID, Token ID represents customer's credit card information that has been stored inside Veritrans server. To get the token, you should follow the tutorial on VT-Direct – get the Token. It has to be concerned that parameter request token for installment is different with normal transaction token. You should add installment and installment_term on the parameter.

After you gain the token, installment transaction charge can be done by adding installment_term attribute on credit_card:

    },

		"gross_amount": 145000
