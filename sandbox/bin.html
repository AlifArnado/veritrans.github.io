BIN Promo

BIN (Bank Identification Number) is the first 6-digits of a credit card number. Credit Card issuer bank can be recognized from its BIN. BIN promo enables transaction charge to be done only by specified credit card, every BIN you used can be represented by first 1 to 8 digits of your credit card number.

Specification of BIN Promo is similar to transaction charge specification. Since the tutorial of implementing Bin Promo on VT-web and VT-Direct are quite different, the tutorial will be explained separately.

  • VT-Web
  • VT-Direct

Similar to VT-Web transaction charge, output from BIN Promo function is payment URL. Customer have to be redirected to VT-Web payment URL to complete the transaction proccess. Implementing BIN Promo on VT-Web is very simple, you only need to add credit_card_bins function on JSON request of transaction charge. The following is JSON request for BIN Promo function on VT-Web:

{
    // Required
    "payment_type" : "vtweb",
    // Required
    "vtweb" : {
        "credit_card_3d_secure" : true,
        "enabled_payments": ["credit_card"],
        "credit_card_bins": ["48111111", "3111", "5"]
    },
    // Required
    "transaction_details" : {
	  "order_id" : "M17550",
	  "gross_amount" : 145000
	},
    // Optional
    "customer_details" : {
	    "first_name" : "Andri",
	    "last_name" : "Litani",
	    "email" : "andri@litani.com",
	    "phone" : "081122334455",
	    // Optional
	    "billing_address" : {
			    "first_name" : "Andri",
			    "last_name" : "Litani",
			    "address" : "Mangga 20",
			    "city" : "Jakarta",
			    "postal_code" : "16602",
			    "phone" : "081122334455",
			    "country_code" : "IDN"
		    },
		// Optional
	    "shipping_address" : {
			    "first_name" : "Obet",
			    "last_name" : "Supriadi",
			    "address" : "Manggis 90",
			    "city" : "Jakarta",
			    "postal_code" : "16601",
			    "phone" : "08113366345",
			    "country_code" : "IDN"
		    }
	    },
    // Optional
    "item_details" : [
		{
		    "id" : "a1",
		    "price" : 50000,
		    "quantity" : 2,
		    "name" : "Apel"
		},
		{
		    "id" : "a2",
		    "price" : 45000,
		    "quantity" : 1,
		    "name" : "Jeruk"
		}

    ]
}

JSON response structure on BIN Promo function is the same as VT-Web transaction charge. Below is the successful JSON response:

{
    "status_code": "201",
    "status_message": "OK, success do VTWeb transaction, please go to redirect_url",
    "redirect_url": "https://vtweb.sandbox.veritrans.co.id/v2/vtweb/550ad19d-e01b-4c8f-945a-67902c7b83da"
}

For VT-Direct you have to get token ID, Token ID represents customer's credit card information that has been stored inside Veritrans server. To get the token, is similiar to normal transaction token.

After getting the token, BIN Promo transaction charge can be done by adding attribute bins on credit_card:

{
    // Required
    "payment_type" : "credit_card",
    // Required
    "credit_card" : {
        "bank" : "bni",
        "token_id": "< your token ID >",
        "bins" : ["48111111", "3111", "5"]
    },
    // Required
    "transaction_details" : {
	  "order_id" : "P17550",
	  "gross_amount" : 145000
	},
    // Required
    "customer_details" : {
	    "first_name" : "Andri",
	    "last_name" : "Litani", // Optional
	    "email" : "andri@litani.com",
	    "phone" : "081122334455",
        // Optional
	    "billing_address" : {
			    "first_name" : "Andri",
			    "last_name" : "Litani",
			    "address" : "Mangga 20",
			    "city" : "Jakarta",
			    "postal_code" : "16602",
			    "phone" : "081122334455",
			    "country_code" : "IDN"
		    },
        // Optional
	    "shipping_address" : {
			    "first_name" : "Obet",
			    "last_name" : "Supriadi",
			    "address" : "Manggis 90",
			    "city" : "Jakarta",
			    "postal_code" : "16601",
			    "phone" : "08113366345",
			    "country_code" : "IDN"
		    }
	    },
    // Optional
    "item_details" : [
		{
		    "id" : "a1",
		    "price" : 50000,
		    "quantity" : 2,
		    "name" : "Apel"
		},
		{
		    "id" : "a2",
		    "price" : 45000,
		    "quantity" : 1,
		    "name" : "Jeruk"
		}

    ]
}

On above example, the registered BINs are 48111111, 3111, dan 5. If credit card with 411111 is used, transaction charge will be denied.

Installment JSON Response structure is similar to transaction charge VT-Direct using credit card. The only difference is on the status_message content.

The example of success transaction JSON Response as follows:

{
    "transaction_id": "1a1a66f7-27a7-4844-ba1f-d86dcc16ab27",
    "order_id": "P17550",
    "gross_amount": "145000.00",
    "payment_type": "credit_card",
    "transaction_time": "2014-08-24 15:39:22",
    "transaction_status": "capture",
    "fraud_status": "accept",
    "masked_card": "481111-1114",
    "status_code": "200",
    "status_message": "OK, Credit Card 3D Secure transaction is successful",
    "approval_code": "1408869563148"
}

The example of denied transaction is as follows:

{
    "status_code": "400",
    "status_message": "Validation Error, [credit card number does not match with the list of credit_card.bins]",
    "validation_messages": [
        "credit card number does not match with the list of credit_card.bins"
    ]
}
