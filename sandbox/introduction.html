Introducing Veritrans API

This tutorial will help you understand the basic of Veritrans API and its implementation. If you find any difficulty, please don't hesitate to contact us at support@veritrans.co.id or visit our official support page.


Veritrans API is a RESTful Web Service developed by Veritrans Indonesia as a communication bridge between you and the payment provider. You can utilize Veritrans API to access various Veritrans services and products for your online payment.

Veritrans API uses One Message concept, whereby the message format is similar to many other types of payment. Veritrans API utilizes the JSON (JavaScript Object Notation) format to communicate between merchant and Veritrans. JSON is sent by HTTP(S) POST.

We have already provided several eCommerce plugins and PHP libraries. Make sure you check the available plugins and libraries before implementing Veritrans API to speed up the integration process.

Payment Type

Below are several payment methods supported by Veritrans API :

  • Credit Card
    • Normal
    • 3D Secure
  • Internet Banking
    • Mandiri ClickPay
    • CIMB Clicks
  • Mobile Banking (TO COME)
    • T-Cash

  • Virtual Accounts (Bank Transfer)
    • Permata Virtual Account


Veritrans API Request and Response

As stated above, communication with Veritrans API is sent by HTTP(S). The Veritrans API request and response contents use JSON format. This section explains in detail the format of HTTP(S) request and response used by Veritrans API.

HTTP(S) request to Veritrans API generally has 3 components :

  1. URL Endpoint
  2. Header Authorization
  3. Header Accept and Content-Type

API URL Endpoint

Every Veritrans API function has different URL endpoint. Every Veritrans API function has the same BASE_URL, eventhough the URL endpoint is different. The following is Base_URL for sandbox and the production environment:

  1. Sandbox Environment:
    https://api.sandbox.veritrans.co.id/v2
  2. Production Environment:
    https://api.veritrans.co.id/v2

Sandbox Environment is used during testing or integration in progress. Every activity and transaction in sandbox environment is dummy in nature. When you are ready to go live, you need to use URL Endpoint for production environment.

Based on the specification of transaction charge, you will find sandbox URL endpoint as followed:

URL endpoint : BASE_URL/charge


Hence URL endpoint for transaction charge in Sandbox is:

https://api.sandbox.veritrans.co.id/v2/charge


And URL endpoint for transaction charge in Production is:

https://api.veritrans.co.id/v2/charge


Header Authorization

To pass the authentication process, you need to place the server_key in Authorization header as standard specification of BASIC AUTH. Server Key for sandbox environment can be obtained in Settings - Access Keys inside MAP.

BASIC AUTH must use the following format:

Username:Password


Since there is no password for the server key, we will leave the password field blank. Therefore, if our server key is:

94960ece-9513-4265-9cf2-67a4da330213


BASIC AUTH Username and password will become:

94960ece-9513-4265-9cf2-67a4da330213:

(Note: make sure there is ":" character in place after server key)

Afterward you need to convert it into BASE 64 format, hence the result is:

OTQ5NjBlY2UtOTUxMy00MjY1LTljZjItNjdhNGRhMzMwMjEzOg==


The following is header example of BASIC AUTH:

Authorization: Basic OTQ5NjBlY2UtOTUxMy00MjY1LTljZjItNjdhNGRhMzMwMjEzOg==

Make sure the merchant put "Basic " before BASE 64 server key


Header Accept and Content-Type

In Veritrans API, you will see a function with input and output parameters in JSON format. If the function has JSON input parameter, you need to add following HTTP(S) header:

Content-type: application/json


If RESTful API function has JSON output parameter, you need to add HTTP(S) header as followed:

Accept: application/json


Below is the summary of required header for HTTP(S) request of Veritrans API:

Header Value
Accept application/json
Content-Type application/json
Authorization Basic base64(server_key:)
