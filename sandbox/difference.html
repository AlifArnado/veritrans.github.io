Sandbox Environtment

This tutorial will help you understand about our Sandbox Environment. If you find any trouble, please don't hesitate to contact us at support@veritrans.co.id or visit our official support page.

Sandbox environment is a special environment that is created to ease merchants when conducting testing with Veritrans’ system. On Sandbox mode, merchant can run trials with few possible payment scenarios without fear of being charged by the bank, as all transactions on Sandbox mode are of dummy data (unreal and for testing purpose only). After trial period on Sandbox mode, merchant can expect to be familiar with Veritrans’ system before moving on to the real mode (Production Environment).

Why is Sandbox Environment necessary?

In development/integration process with Veritrans, there will be times when mechant wishes to conduct testing to see whether the system is up and running properly. With all the different payment methods Veritrans offers, merchant will be able to do trial transaction with each when on Sandbox mode. True, all testing can only be done on Production Environment, but it will be risky since all transactions on Production mode will involve real charges by the bank. Besides, some payment methods on Production Environment might not be active yet due to unfinish administration process with the bank.

This is the function of Sandbox Environment. On Sandbox mode, merchant will be able to run trial transactions with all payment methods since automatization of payment method has been activated. You can also use it from shopper's eye without the risk of being charged.

Starting on Sandbox Environment

Sign-up

Merchant account registration on Sandbox mode can be accessed from https://my.sandbox.veritrans.co.id/register. You have to click the confirmation link that will be sent to your email address to continue with the login process.

Login

To login to Merchant Administration Portal (MAP) on Sandbox mode, please access it from https://my.sandbox.veritrans.co.id/login.
