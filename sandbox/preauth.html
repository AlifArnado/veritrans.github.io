Pre-authorization & Capture

Pre-authorization is used to check whether a credit card is able to perform the transaction with a specified value or not. New transaction actually occurs after the capture process. Only VT-Direct using credit cards is allowed to use this feature. Further explanation can be checked on VT-Direct - Pre-Authorization & Capture.

Pre-authorization

In VT-Direct, you should obtain a authorization token ID first. As to be concerned, the request token parameter for pre-authorization differs with normal transaction. you have to add the "type" attribute on request token parameter.

After pre-authorization token is obtained, add "type" attribute in "credit_card" part when charge is being done. pre-authorization function specification is similar with credit card transaction charge.

{
    // Required
    "payment_type" : "credit_card",
    // Required
    "credit_card" : {
        "bank" : "bni",
        "token_id": "< your token ID >",
        "type" : "authorize"
    },
    // Required
    "transaction_details" : {
	  "order_id" : "A27550",
	  "gross_amount" : 145000
	},
    // Required
    "customer_details" : {
	    "first_name" : "Andri",
	    "last_name" : "Litani", // Optional
	    "email" : "andri@litani.com",
	    "phone" : "081122334455",
        // Optional
	    "billing_address" : {
			    "first_name" : "Andri",
			    "last_name" : "Litani",
			    "address" : "Mangga 20",
			    "city" : "Jakarta",
			    "postal_code" : "16602",
			    "phone" : "081122334455",
			    "country_code" : "IDN"
		    },
        // Optional
	    "shipping_address" : {
			    "first_name" : "Obet",
			    "last_name" : "Supriadi",
			    "address" : "Manggis 90",
			    "city" : "Jakarta",
			    "postal_code" : "16601",
			    "phone" : "08113366345",
			    "country_code" : "IDN"
		    }
	    },
    // Optional
    "item_details" : [
		{
		    "id" : "a1",
		    "price" : 50000,
		    "quantity" : 2,
		    "name" : "Apel"
		},
		{
		    "id" : "a2",
		    "price" : 45000,
		    "quantity" : 1,
		    "name" : "Jeruk"
		}

    ]
}

Pre-authorization JSON response structure is similar with credit card transaction charge. When the process is success, "transaction_status" will have "authorize" value. As to be concerned, the transaction has not happened yet, the transaction occurs when the capture function performed. The following is example for pre-authorization JSON response :

{
    "status_code": "200",
    "status_message": "OK, Credit Card 3D Secure transaction is successful",
    "transaction_id": "1ac1a089d-a587-40f1-a936-a7770667d6dd",
    "order_id": "A27550",
    "payment_type": "credit_card",
    "transaction_time": "2014-08-25 10:20:54",
    "transaction_status": "authorize",
    "fraud_status": "accept",
    "masked_card": "481111-1114",
    "approval_code": "1408936854718",
    "gross_amount": "145000.00"
}

Capture

After a successful pre-authorization process, transaction charge can be done using capture function. You need to record pre-authorization "transaction_id" and use it as one of the attribute of JSON request capture.

Spesification

Description Transaction charge which have passed pre-authorization
Endpoint BASE_URL/capture
HTTP Method POST
HTTP Header Accept: application/json
Content-Type: application/json
Authentication: Basic Base64(server_key:)

JSON Request

The following is Description of JSON Request capture transaction:

JSON Attribute Data Type Required note
transaction_id String Yes Transaction ID which have escaped pre-authorization
gross_amount Long Ya The value of transactions which will be paid by the customer.
Note: Should not exceed the value of the current pre-authorization transaction

The following is example of JSON request capture :

{
    "transaction_id" : "1ac1a089d-a587-40f1-a936-a7770667d6dd",
    "gross_amount" : "55000.00"
}

As to be concerned the value of transactions is smaller than the current value of the pre-authorization transaction.

JSON Response

Description of JSON response is similar with normal transaction charge. The following is example of successful capture response :

{
    "status_code": "200",
    "status_message": "OK, Credit Card capture transaction is successful",
    "transaction_id": "1ac1a089d-a587-40f1-a936-a7770667d6dd",
    "order_id": "A27550",
    "payment_type": "credit_card",
    "transaction_time": "2014-08-25 10:20:54",
    "transaction_status": "capture",
    "fraud_status": "accept",
    "masked_card": "481111-1114",
    "approval_code": "1408937217061",
    "gross_amount": "55000.00"
}
