Getting Started

This tutorial will help you understand the basic of Veritrans API and its implementation.
If you need further assistance, please contact us at support@veritrans.co.id or visit our official support page.

Welcome to Veritrans API!

Veritrans API is a RESTful Web Service served as a communication bridge between merchant and our services.
Veritrans API uses One Message concept, in which message format is similar for all payment channels.

Info: Note:
Veritrans has various plugins and libraries available to speed up the integration process.



HTTP(S) Request

Veritrans API can be requested through HTTP(S) Request to Veritrans Base URL endpoint.
The HTTP(S) Header has to be used to allow proper authentication.

API Base URL

Development Environment https://api.sandbox.veritrans.co.id/v2
Production Environment https://api.veritrans.co.id/v2


API End Point

Endpoint HTTP Method Definition
BASE_URL/token POST Tokenize Credit Card information before being charged
BASE_URL/charge POST Perform a transaction with various available payment methods and features
BASE_URL/capture POST Charge a Pre-Authorized transaction for card payment
BASE_URL/order_id/approve POST Approve a transaction with certain order_id which gets challenge status from Fraud Detection System.
BASE_URL/order_id/cancel POST Cancel a transaction with certain order_id. Cancelation can only be done before settlement process.
BASE_URL/order_id/status GET Get information status of a transaction with certain order_id


Caution: NOTE:
Data sent in Development Environment will not trigger an actual purchase.
Veritrans service can be fully utilized in Production Environment.


HTTP(S) Header

Header Value Definition
Content-Type application/json The Content-Type field indicates that JSON type is acceptable to send to the recipient
Accept application/json The Accept field is used to specify that JSON type is acceptable for the response
Authorization Basic base64(‘server key’ , ‘:’) The Authorization field credentials can be found in Veritrans Merchant Admin Portal

Content-Type and Accept Header

In Veritrans API, the input and output parameters of the methods will be in JSON format.

  • To accept JSON input and output parameters, you need to add the following HTTP(S) header:

Content-type: application/json
Accept: application/json

Authorization Header

The authorization header utilizes Veritrans Server Key following HTTP(S) BASIC AUTH convention

  • For Development environment, obtain server key in Sandbox MAP Settings - Access Keys.
  • For Production environment, obtain server key in MAP Settings - Access Keys.


Place Server Key as Username, following the format of BASIC AUTH

`Username:Password`

Since there is no Password for the server key, leave the password field blank.

If our server key is:

`94960ece-9513-4265-9cf2-67a4da330213`

Following BASIC AUTH Username and password format will become:

`94960ece-9513-4265-9cf2-67a4da330213:`

(Note: Make sure there is ":" character after server key!)

Encode these values into BASE 64 format, hence the result is:

`OTQ5NjBlY2UtOTUxMy00MjY1LTljZjItNjdhNGRhMzMwMjEzOg==`
  • To be authorized to use Veritrans API, you need to add the following HTTP(S) header:

Authorization: Basic OTQ5NjBlY2UtOTUxMy00MjY1LTljZjItNjdhNGRhMzMwMjEzOg==

Caution: NOTE:
The following Authorization Header is just an example.
Please use your own Sandbox / Production Server Key!)
