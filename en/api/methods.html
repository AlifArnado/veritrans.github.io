Veritrans API Methods

Endpoint HTTP Method Definition
BASE_URL/token POST Tokenize Credit Card information before being charged
BASE_URL/charge POST Perform a transaction with various available payment methods and features
BASE_URL/capture POST Charge a Pre-Authorized transaction for card payment
BASE_URL/query/approve POST Approve a transaction with certain order_id or transaction_id which gets challenge status from Fraud Detection System.
BASE_URL/query/cancel POST Cancel a transaction with certain order_id or transaction_id. Cancelation can only be done before settlement process.
BASE_URL/query/status GET Get information status of a transaction with certain order_id or transaction_id.



Get Token

Token ID is a unique value that is associated with the customer credit card information during a transaction.

Get Token method sends the credit card info via Veritrans.min.js to Veritrans server and returns token ID to the merchant be charged.

Info: Note: Get Token methods and features are only applicable for Credit Card Transaction


Veritrans Javascript Library

Veritrans Javascript library allows customer credit card to be securely sent to Veritrans server.
To utilize this library, please add the following line of code at your payment page inside the <head> tag.

<script src="https://api.veritrans.co.id/v2/assets/js/veritrans.min.js"></script>

The Get Token function from the Javascript Library has to be triggered when the customer click pay:

// Create the card object with the required fields
function card () {

    }
};

// Create the callback function according to Veritrans response
function callback(response) {
        // Handling of Veritrans Callback Response Object explained below
};

//Get token function to be triggered when click pay
Veritrans.token(card, callback);

function callback(response) {
    if (response.redirect_url) {
      // If 3Dsecure transaction. Open 3Dsecure dialog
      console.log('Open Dialog 3Dsecure');
      openDialog(response.redirect_url);

    } else if (response.status_code == '200') {
      // success 3d secure or success normal
      //close 3d secure dialog if any
      closeDialog();

      // store token data in input #token_id
      $("#token-id").val(response.token_id);

    } else {
      // failed request token
      //close 3d secure dialog if any
      closeDialog();
      $('#submit-button').removeAttr('disabled');
      // Show status message.
      $('#message').text(response.status_message);
      console.log(JSON.stringify(response));
    }
  }

BBM Money Response Handler

Merchant can generate BBM Money App link with permata_va_number and callback URL information through a seperate API call.

URL: bbmmoney://api/payment/imp?data=URL encoded JSON data

URL encoded JSON Object:

JSON Attribute Data Type Description
reference String The payment reference number. It is the permata_va_number from the charge response.
callback_url String Contains the callback URL described in the next section


Callback URL:

  • check_status : When the payment transaction is completed or failed, BBM Money will return to this URL. The merchant should always double check the transaction status using the API from payment gateway (or its own system). If the payment failed, the merchant may want to display information on how to retry the payment later.

  • before_payment_error : When there is an app error (like login failure, connection issue, etc) or there are problems with the bill before the actual payment, BBM Money will return to this URL. The merchant may want to display information on how to retry the payment later.

  • user_cancel : When the user pressed the “Cancel” button in the BBM Money payment screen, BBM Money will return to this URL. The merchant may want to display information on how to retry the payment later, or cancel the order immediately.

Info: Note:
The merchant may want to embed order information (e.g. order_id, etc) in the callback url for easy lookup. For example: http://example.com/checkstatus?order_id=12345
