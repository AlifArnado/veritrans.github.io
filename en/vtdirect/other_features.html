VT-Direct: Other Features

The tutorial below will assist you in implementing other features on VT-Direct. If further assistance is required, please contact us at support@veritrans.co.id or visit our support page.


Currently, VT-Direct has additional features for credit card payment method which are:

  1. Pre-Authorization & Capture
  2. Installment
  3. BIN Promo
  4. One Click
  5. Two Clicks
  6. Point Transaction

VT-Direct also offers additional Charge Feature, such as:

  1. Set Custom Fields
  2. Set Custom Expiration


The following is a tutorial to utilize these features as needed:

  • Pre-Authorization & Capture
  • Installment
  • BIN Promo
  • One Click
  • Two Clicks
  • Custom Fields
  • Custom Expiry
  • Point Transaction


Pre-Authorization & Capture

Pre-authorization is useful to check if a credit card is allowed to make a transaction for a certain value. When pre-authorization is completed, you can check the credit card status when the transaction has not occured yet. Transaction occurs when the capture operation is issued. Capture can only be completed when pre-authorization process is successful.

By default, charge operation in Veritrans is a combination of pre-authorization and capture. If you wish to do these separately, please see the following step.

Pre-Authorization Transaction

Generally, pre-authorization process is similar to normal charge transaction. The difference lies on the request token parameter and charge transaction. You have to add the 'type' attribute with the value 'authorize' on both of the processes.

Following the same example as credit card charge transaction, additional script on request token is as followed: 

<script type="text/javascript">

$(function(){
  
  ...

  var card = function(){
    return {
      'card_number'   : $("#card-number").val(),
      'card_exp_month'  : $("#card-expiry-month").val(),
      'card_exp_year'   : $("#card-expiry-year").val(),
      'card_cvv'      : $("#card-cvv").val(),
      'secure'       : true,
      'bank'       : 'bni',
      'gross_amount'   : $("#price").val(),

      // Ask for pre-authorization token id
      'type' : 'authorize',
    }
  };

  ...

</script>


The acquired token can be saved and used to perform pre-authorization during charge transaction. Parameters that is modified during charge transaction is as followed: 

<?php
include_once('Veritrans.php');

...

$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'token_id'  => $token_id,
    'bank'    => 'bni',

    // Authorize instead of normal charge operation
    'type' => 'authorize'
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details
  );

$result = Veritrans_VtDirect::charge($transaction_data);

?>


In the above example, pre-authorization response is saved in the variable $result. Structure and example of state $result when pre-authorization process is successful is as followed: 

$result = array(
    "status_code" => "200",
    "status_message" => "Success, Credit Card transaction is successful",
    "transaction_id" => "b505b192-f202-41ad-b82a-18678d0571a9",
    "order_id" => "201404211437",
    "payment_type" => "credit_card",
    "transaction_time" => "2014-04-21 14:35:32",
    "transaction_status" => "authorize",
    "fraud_status" => "accept",
    "masked_card" => "481111-1114",
    "bank" => "bni",
    "eci" => "05",
    "approval_code" => "140540",
    "gross_amount" => "150000.00"
);


The important attribute on pre-authorization response is transaction_id. You need to use this attribute value to perform the capture transaction.


Capture Transaction

To perform capture transaction, you simply have to prepare the attribute transaction_id and gross_amount. Both these attributes are related to the previous pre-authorization process. The gross_amount value cannot exceed the gross_amount on pre-authorization. If gross_amount attribute is not given during capture, the gross_amount value on pre-authorization will be used.

The following is the sample PHP code to perform capture transaction: 


<?php
include_once('Veritrans.php');

// TODO: change with your actual server_key that can be found on Merchant Administration Portal (MAP)
Veritrans_Config::$serverKey = "< Your Server Key >";
Veritrans_Config::$isProduction = false;

// Use transaction_id from Pre-authorization process
$transaction_id = 'b505b192-f202-41ad-b82a-18678d0571a9';
$amount = 150000;

$result = Veritrans_VtDirect::capture($transaction_id, $amount);
?>


On the above example, the capture response is saved in the variable $result. Structure and example of state $result when capture transaction process is successful is as followed: 

$result = array(
    "status_code" => "200",
    "status_message" => "Success, Credit Card capture transaction is successful",
    "transaction_id" => "b505b192-f202-41ad-b82a-18678d0571a9",
    "order_id" => "201404211437",
    "payment_type" => "credit_card",
    "transaction_time" => "2014-04-21 14:35:32",
    "transaction_status" => "capture",
    "fraud_status" => "noscore",
    "masked_card" => "481111-1114",
    "bank" => "bni",
    "gross_amount" => "150000.00"
);


Installment

Online Installment

To activate installment feature, merchant has to make an agreement with the acquiring bank. Acquiring bank will issue MID Installment which allow the transaction to be converted into installment automatically. Merchant also needs to inquire if the MID for installment is ready from a Veritrans sales representative. The installment feature can be performed with a normal token or 3DS token.

The installment feature implementation is similar to credit card charge transaction. The difference lies on the request token and charge transaction, you have to add a 'Installment’ attribute and installment_term. The installment attribute with the respective value can be used in Sandbox environment:

Attribute Data Type Required Explanation
bank String(50) Yes Acquiring Bank that provides installment. In the moment, the valid value is "bni" or "mandiri"
installment_term Array Integer Yes Installment period. In the moment, the valid value is 6 or 12


Using the same example in credit card charge transaction, script that is modified during request token is as followed: 

<script type="text/javascript">

$(function(){
  
  ...

  var card = function(){
    return {
      'card_number'   : $("#card-number").val(),
      'card_exp_month'  : $("#card-expiry-month").val(),
      'card_exp_year'   : $("#card-expiry-year").val(),
      'card_cvv'      : $("#card-cvv").val(),
      'secure'       : true,

      // Ask for 12 months installment token_id
      'installment' : true, 
      'installment_term' : 12,
      
      'bank'       : 'bni',
      'gross_amount'   : $("#price").val(),
    }
  };

  ...

</script>


The acquired token can be saved and then used to perform the charge transaction. Attribute that has to be added to perform installment is installment_term. Parameters that are modified during charge transaction is as followed:

Offline Installment

To allow installment feature with banks which doesn't issue MID Installment, merchant can use feature offline installment. With offline installment feature, the transaction will be charged in full amount and the transaction will be converted into installment later. For more information and support, merchant can contact Veritrans Sales Representative.

This installment feature implementation is similar with credit card charge online installment with combination with bin filter feature. The purpose of bin filter is to limit only certain cards will be allowed to do offline installment based on the agreement between merchant and issuing banks.

BIN Promo

BIN (Bank Identification Number) is the first 6 digit number of a credit card. The issuing bank for the credit card can be identified from these numbers. BIN promo is the feature that allows merchant to give out the deal based on the promotion given by the issuing bank. Merchant can use the first 1-8 digits of the credit card to utilize the BIN Promo feature.

To use this feature, you have to first list out the BIN that offers a promotion. The BIN list will become the transaction parameter bins. Transactions can only be performed by credit cards that are included in the BIN list. Use the BIN list as a value from the attribute bins during the charge transaction.

Following the same transaction in credit card integration, the sample PHP script that can be used is as followed: 

<?php
include_once('Veritrans.php');

...

$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'token_id'  => $token_id,
    'bank'    => 'bni',
    'bins' => array("48111111", "3111", "5")
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details,
  );
try{
  $result = Veritrans_VtDirect::charge($transaction_data);
} catch (Exception $e){
  echo $e->getMessage();
}
...
?>


On the above example, the BIN that is listed is 48111111, 3111, and 5. If a card with the BIN "411111" is used, charge transaction will fail.


One Click

The one click feature allows you to use one token on several transactions. The token represents the credit card number dan expiry date of the customer. This will make transactions easier since the credit card information needs to be filled only once and not recurrently.

Do note that token will be activated only after the initial transaction is performed. To use the one click feature, 3D Secure feature has to be used.

In general, there are two steps required to utilize one click feature:

  1. Initial Transaction
    • Customer fill the credit card number, CVV, expiry date
    • Customer perform 3D Secure authentication
    • Merchant receives token
    • Merchant uses token to charge transaction
    • When transaction successful, merchant saves customer’s token in the database. We name this token as saved_token.
  2. Following Transaction
    • Merchant retrieve customer saved_token from database
    • Merchant uses saved_token to charge transaction


Initial Transaction

To implement one click, you simply add the attribute save_token_id on initial transaction. Following the example on credit card charge transaction, the modified PHP script during charge transaction is as follows: 

<?php
include_once('Veritrans.php');

...

$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'token_id'  => $token_id,
    'bank'    => 'bni',
    'save_token_id' => true
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details,
  );

$result = Veritrans_VtDirect::charge($transaction_data);

...

?>


On the above example, one click response is saved in the variable $result. Structure and example of state $result when one click process is successful is shown below: 

array(
  "transaction_id" => "ca5e0ff5-4297-48ef-aa25-b3ab018d4ead",
  "order_id" => "order52",
  "gross_amount" => "200000.00",
  "payment_type" => "credit_card",
  "transaction_time" => "2014-03-27 17:20:53",
  "transaction_status" => "capture",
  "fraud_status" => "accept",
  "masked_card" => "481111-1114",
  "saved_token_id" => "4011117d16c884-2cc7-4624-b0a8-10273b7f6cc8",
  "saved_token_id_expired_at" => "2024-03-27 17:20:53",
  "status_code" => "200",
  "bank" => "bni",
  "eci" => "05",
  "approval_code" => "140540",

Two important attributes for a successful one click response is saved_token_id and saved_token_id_expired_at. saved_token_id is the token that you save when the customer makes a following transaction. This attribute is what we referred as saved_token on the previous explanation. The value from saved_token can be used repeatedly until saved_token_id_expired_at.


Following Transaction

On the following transaction, customer does not have to fill in credit card information anymore and charge transaction process can be performed using saved token.

The following example is the charge following the transaction using saved_token_id: 

<?php

$transaction_data = array(
  'payment_type'      => 'credit_card',

    'token_id'  => $saved_token_id,
    'bank'    => 'bni',
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details,
  );

$result = Veritrans_VtDirect::charge($transaction_data);

...

?>


Structure and example of state $result is the same with the standard charge transaction process.

Two Clicks

Token on two clicks represents the customer’s card number and expiry date. Your customer simply has to fill in the CVV and 3D Secure authentication on the following transaction. The difference between one click is that the two clicks feature can be used in both 3D Secure and non-3D Secure transaction.

In general, there are two steps required to utilise two clicks feature:

  1. Initial Transaction
    • Customer fills in the credit card number, CVV, expiry date
    • Customer performs 3D Secure authentication
    • Merchant receives token
    • Merchant uses token to charge transaction
    • When transaction successful, merchant save customer’s token in the database. We name this token as saved_token.
  2. Following Transaction
    • Merchant retrieves customer saved_token from database
    • Customer fills in CVV
    • Perform 3D Secure authentication if required
    • Merchant uses saved_token and CVV to acquire the new token
    • Merchant uses new token to charge transaction


Initial Transaction

Initial transaction implementation is similar to standard credit card charge transaction. You simply have to add the attribute saved_token_id during charge transaction. When transaction is successful, the token is saved for the following transaction. 

<?php
include_once('Veritrans.php');

...

$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'token_id'  => $token_id,
    'bank'    => 'bni',
    'save_token_id' => true
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details,
  );

$result = Veritrans_VtDirect::charge($transaction_data);
$status = $result->transaction_status;

if($status == 'capture'){
  // save 'token_id' for this customer in database
}
...

?>

This attribute is what we referred as saved_token on two clicks step. The value from saved_token can be used repeatedly until saved_token_id_expired_at.


Following Transaction

On the following transaction, we use the saved_token and the customer filled CVV to acquire new token. two_click attributes also need to be added when new request for token is performed. The new token will be used for the following transaction. The new token will be automatically filled in <input id="token-id" name="token-id" type="hidden" /> by veritrans.min.js.

Do note that you do not use saved_token to perform charge transaction. You use saved_token as one of the conditions to acquire a new token. You do not save new token from the customer to the database.

Field number and expiry date in the form can be removed because the customer simply needs to fill the CVV and perform 3D Secure authentication. Using the same example in credit card charge transaction, the following example is the new request token using the saved_token and CVV from the customer:

      'bank' : 'bni',

</script>

Set Custom Expiry

Set Custom Expiry feature enables merchant to set an expiry time of payment with pending status for every transaction.

The following payment is the list of payment with Pending Status:

  • Bank Transfer: Permata Virtual Account, Mandiri Bill Payment
  • Direct Debit: CIMB Clicks, e-pay BRI, BCA KlikPay
  • e-Wallet: XL Tunai, BBM Money

Following the test transaction on integration, the sample PHP code that we can make use of is:

<?php
include_once('Veritrans.php');

...

$transaction = array(
    'payment_type' => 'bank_transfer',
    'bank_transfer' => array(
    'bank' => "permata"
    ),

        ),
    'transaction_details' => array(
        'order_id' => 8033,
        'gross_amount' => 10000
        ),

    //Set Custom Expiry
    'custom_expiry' => array ( 
      "expiry_duration" => 30,
      "unit" => "minute"
  )
    );

$vtweb_url = Veritrans_Vtweb::getRedirectionUrl($transaction);

...

?>

Info: Info:
Please refer to our API Documentation - Set Custom Expiry for more info on the JSON attributes.

Point Transaction

Point transaction is a transaction in which payment can be made by the reward points on credit cards in full or partial. For now, point transaction can only be used with BNI acquiring and using BNI credit card (On Us).

Getting point information

Token request process

<script type="text/javascript">

$(function(){
  
  ...

  var card = function(){
    return {
      'card_number'   : $("#card-number").val(),
      'card_exp_month'  : $("#card-expiry-month").val(),
      'card_exp_year'   : $("#card-expiry-year").val(),
      'card_cvv'      : $("#card-cvv").val(),
      'bank'       : 'bni',
      'gross_amount'   : $("#price").val(), // no decimal allowed for creditcard
    }
  };

  ...

</script>

Result of the above token request is token_id used to obtain point information

Inquiry point process

GET https://api.veritrans.co.id/v2/point_inquiry/{token_id}

Success response

{
    "status_code": "200",
    "status_message": "Success, Credit Card Point transaction is successful",
    "transaction_time": "2015-05-11 16:28:17",
    "point_balance": 100000
}

The essentials of the above information is point_balance, stating that the customer has a number of points that can be used.


Failed response

{

Failed response can be invalid card number or Merchant TID is not supported for point transaction.

Using a point to make payment

Point balance that you have can be used to make payments, the parameters that need to be added when the charge process is point_redeem_amount. Script that is modified during charge process is as followed:

<?php
include_once('Veritrans.php');

...

$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'token_id'  => $token_id,
    'bank'    => 'bni',
    'point_redeem_amount' => 10000 // should not exceed the balance point
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details,
  );

$result = Veritrans_VtDirect::charge($transaction_data);

...

?>



Info: Notes:

* Point transaction are only available to BNI On Us transaction. That means issuer == acquirer.
* Parameter bank in credit_card or token_request are optional. If specified, the only possible value are BNI.
* Nominal payment using point should not exceed the balance point owned, If redeem points larger than nominal point balance, the transaction will fail.
