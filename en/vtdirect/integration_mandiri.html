VT-Direct Integration (Mandiri ClickPay)

Every payment method has different integration steps available. The objective of this documentation is to assist you in integrating with VT-Direct Mandiri Cickpay payment method. For more inquiries, please contact us at support@veritrans.co.id or visit our support web page.



Mandiri ClickPay is one of the internet banking payment method that is provided by Mandiri Bank. This method allows customer to perform transaction by filling in his 16 digits Mandiri Debit Card number and Mandiri PIN Token to validate transaction.

To activate Mandiri ClickPay payment method (Automatically active on Sandbox mode), you have to contact a Veritrans sales representative.

Transaction Process Flows



Integration Steps VT-Direct Mandiri Clickpay

  1. Create Form
  2. Perform Charge Transaction
  3. Check Transaction Status

  • Step 1Create Form
  • Step 2Perform Charge Transaction
  • Step 3Check Transaction Status


Create Form

On this step, we will try to create Mandiri ClickPay transaction form at the file form_vtdirect_mandiri.html. We have prepared some pictures to assist you to create the form, which can be downloaded here. Mandiri ClickPay form is required to have fields as followed:



Field Name Data Type Length Description
Debit Card Number String 16 Input at merchant website
Input-1 String 10 The last 10 digits of debit card number
Input-2 String 16 Total amount of the transaction
Input-3 String 5 Can be any random number
Token Response String 6 Generated by token


The following is a sample from the file form_vtdirect_mandiri.html: 

<html>
  <head>
    <title>VT-Direct Form Example</title>
    <script src="http://ajax.googleapis.com/ajax/libs/jquery/1.11.0/jquery.min.js"></script>
    <script type="text/javascript">
$(function(){
  
  updateInput1();

  $('#card-number').keyup(function(){
    updateInput1();
  });

  $('#input3').val(random5digit());

  function updateInput1(){
    var ccNumber = $('#card-number').val();
    if(ccNumber.length > 9) {
      $('#input1').val(ccNumber.substring(ccNumber.length, ccNumber.length-10));
    }
  }

  function paddy(n, p, c) {
    var pad_char = typeof c !== 'undefined' ? c : '0';
    var pad = new Array(1 + p).join(pad_char);
    return (pad + n).slice(-pad.length);
  }

  function random5digit(){
    return paddy(Math.floor(Math.random() * 99999), 5); 
  }

});
</script>
  </head>

The above form utilizes the jQuery library that can be modified as needed. When the form is first shown, field input 2 and 3 are automatically filled. Input 2 is the total transaction value. Input 3 is the 5 digit numeric that is generated randomly by Javascript.

Javascript also allows field input 1 to automatically be filled when your customer insert their Mandiri Debit card. Customer simply has to use the Mandiri Token to fill the challenge field and this information will be submitted to checkout_mandiri.php. This file provides the implementations of charging a transaction using Mandiri ClickPay.

To ease the transaction process, we recommend that the Mandiri ClickPay transaction details is filled comprehensively in the form page.


Perform Charge Transaction

To perform charge transaction using Mandiri ClickPay, the steps required are as followed:

  • payment_type parameter has to be set as mandiri_clickpay.

  • New JSON object has to be added, with the name mandiri_clickpay, and has the pairing name/value as followed:

    • "card_number" → debit card number
    • "input1" → last 10 digits of the debit card number
    • "input2" → total transaction value
    • "input3" → 5 digit random number
    • "token" → token that is filled by the customer

Information that can be accessed from the file form_vtdirect_mandiri.html are: Mandiri Debit card number, input 1, input 2, input 3, and token challenge.

To perform transaction charge in Mandiri ClickPay, we can use Veritrans PHP Library. The following is an example of charge transaction implementation in the file checkout_mandiri.php: 

<?php

include_once('Veritrans.php');

// TODO: change with your actual server_key that can be found on Merchant Administration Portal (MAP)
Veritrans_Config::$serverKey = "< Your Server Key>";
Veritrans_Config::$isProduction = false;

$card_number = $_POST['card-number'];
$input1 = $_POST['input1'];
$input2 = $_POST['input2'];
$input3 = $_POST['input3'];
$token = $_POST['token'];

$transaction_details = array(
  'order_id'    => rand(),
  'gross_amount'  => $input2
);

$customer_details = array(
    'first_name'    => "Andri",
    'last_name'     => "Litani",
    'email'         => "Andri@Litani.com",
    'phone'         => "081122334455"
    );

// Data that will be sent for charge transaction request with Mandiri ClickPay.
$transaction_data = array(
  'payment_type'      => 'mandiri_clickpay',
  'mandiri_clickpay' => array(
      'card_number' => $card_number,
      'input1' => $input1,
      'input2' => $input2,
      'input3' => $input3,
      'token' => $token
    ),
  'transaction_details' => $transaction_details,
  'customer_details' => $customer_details
  );

$result = Veritrans_VtDirect::charge($transaction_data);
?>


On the above example, we utilize the mandiri Clickpay, transaction, and customer data as the charge transaction parameter. You can add additional data during the charge transaction, such as: Customer address, Delivery address, and item information. To add this data, you can check API Reference - API Methods.

When the transaction is completed, customer can proceed back to your website. To do this, you have to fill in the callback URL configuration in MAP (Merchant Administration Portal). For more information, please refer to the Getting Started menu.

Error During Charge Transaction

Under a certain case, charge transaction on VT-Direct may fail. Veritrans will return an error message if failure occurrs, the reason for failure can be inspected from the status code displayed on the error message. The status codes that generally occur are:

Error Code Explanation
400 Validation error Format or content of the transaction data is invalid; Example: Wrong variable type, invalid payment method.
401 Access Denied Invalid client or server key.
402 Access Denied Merchant does not have access to utilise a certain method of payment.
406 Duplicate Order ID Order ID has previously been used and paid.
410 Account Inactive Merchant account has been inactive for a long period of time.


The complete list of status code can be found at API Reference - Status Code.

When charge transaction process is completed, you can check the transaction status. The steps to examine transaction status will be discussed on the next step.


Transaction Status

Veritrans server will return a response for every charged transaction in JSON form. Veritrans PHP Library will convert JSON into a PHP object automatically. The sample JSON script that is returned by Veritrans is as followed: 

$result = array(
    "transaction_id" => "cddd6c1f-3b03-4c89-b0a8-0ea7dccf5217",
    "order_id" => "201404031721",
    "gross_amount" => "10000.00",
    "payment_type" => "mandiri_clickpay",
    "transaction_time" => "2014-04-03 17:21:46",
    "transaction_status" => "settlement",
    "status_code" => "200",
    "approval_code" => "1405402217483",
    "masked_card":"411111-1111"
    "status_message" => "Success, Mandiri Clickpay transaction is successful"
);


One of the most important information in the JSON response is the transaction status. There are two types of transaction status for Mandiri ClickPay charge transaction response:

  1. settlement: Transaction successful.
  2. deny: Transaction rejected by bank.

You can add the following PHP code at the last line of the file checkout_mandiri.php to send transaction status to your customer: 

<?php
include_once('Veritrans.php');

...

$result = Veritrans_VtDirect::charge($transaction_data);

if($result->transaction_status == "settlement")
{
  //success
  echo "Mandiri ClickPay Transaction Successful. <br />";
  echo "Transaction status for order id ".$result->order_id.": ".$result->transaction_status;
  echo "<h3>Transaction details:</h3>";
  var_dump($result);
}
else if ($result->transaction_status == "deny")
{
  //deny
  echo "Mandiri ClickPay Transaction Failed. <br />";
  echo "Transaction status for order id ".$result->order_id.": ".$result->transaction_status;
  echo "<h3>Transaction details:</h3>";
  var_dump($result);
} else {
  //error
  echo "Errors occurs on the sent transaction data.<br />";
  echo "<h3>Result:</h3>";
  var_dump($result);
}
?>


At this point, the payment process of a Mandiri ClickPay transaction is completed. If transaction_status value is settlement, the transaction value has been settled to your account. In addition to sending the transaction status to the customer, you may also use the code above to update the respective database. Example, to mark if the respective order_id has been paid.

Transaction status can also be found in Merchant Administration Portal (MAP). You have to login to MAP to see the details of the transaction. The other alternative to check on the transaction status is to use the get status command.
