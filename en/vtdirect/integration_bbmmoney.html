VT-Direct Integration (BBM Money)

Every payment method has different integration steps available. The objective of this documentation is to assist you in integrating with VT-Direct BBM Money payment method. For more inquiries, please contact us at support@veritrans.co.id or visit our support web page.



One of the payment method offered by Veritrans is BBM Money. By using this service, your customer will have an option to make a payment via BBM Money to either BBM Money or Permata Bank Virtual Account Number.

To activate BBM Money method of payment (automatically active on Sandbox mode), you have to contact a Veritrans sales representative.

Info: Note:
- Refer to Veritrans API Reference for more information on BBM Money Attributes.

Integration VT-Direct BBM Money

  1. Create Form
  2. Charge Transaction
  3. Create BBM Money Link
  4. Transaction Status

Customer has to pay with BBM Money to complete the transaction.

  • Step 1Create Form
  • Step 2Charge Transaction
  • Step 3Create BBM Money Link
  • Step 4Transaction Status


Create Form

BBM Money payment form will be made in the file form_vtdirect_bbmmoney.html. Because most of the transaction process will not be made in the merchant's page, the form that we made will only show the steps of payment and the payment button of BBM Money. When customer clicks this payment button, charge transaction process will be carried on the next step.

The following is a sample from the file form_vtdirect_bbmmoney.html: 

<head>
	<title>Mobile Payment</title>
</head>
<body>
	<h1>BBM Money</h1>
	<b>I want to pay with BBM Money</b>
	<form action="checkout_bbmmoney.php" method="POST" id="payment-form">
      <button id="submit-button" type="submit">Pay</button>
    </form>
</body>


Charge Transaction

To perform charge transaction using BBM Money, the following steps are required:

  • payment_type parameter has to be set as bbm_money.

This step describes the BBM Money charge transaction in the file checkout_bbmmoney.php. To perform charge transaction, we will use Veritrans PHP library. The following is an example of charge transaction implementation in the file checkout_bbmmoney.php: 

<?php
include_once('Veritrans.php');

// TODO: change with your actual server_key that can be found on Merchant Administration Portal (MAP)
Veritrans_Config::$serverKey = "< Your Server Key >";
Veritrans_Config::$isProduction = false;

$transaction_details = array(
  'order_id'    => rand(),
  'gross_amount'  => 145000,
);

$customer_details = array(
    'first_name'    => "Andri",
    'last_name'     => "Litani",
    'email'         => "andri@litani.com",
    'phone'         => "081122334455"
    );

// Data that will be sent for charge transaction request with BBM Money.
$transaction_data = array(
  'payment_type' => 'bbm_money',
  'transaction_details' => $transaction_details,
  'customer_details' => $customer_details
  );

$result = Veritrans_VtDirect::charge($transaction_data);
?>


On the above example, we utilize the transaction data and customer data as the charge transaction parameter. You can add additional data during the charge transaction, such as: Customer address, delivery address, and item information. To add this data, you can check API Reference - API Methods.


Create BBM Money Link

Veritrans' server will return a response for every charge transaction in JSON form. Veritrans' PHP Library will convert JSON into a PHP object automatically. The sample JSON script that is returned by Veritrans is as followed: 

$result = array(
  "status_code" => "201",
  "status_message" => "Success, BBM Money transaction is successful",
  "transaction_id" => "ff05337c-6c94-4f70-8e81-35acd89b688e",
  "order_id" => "201404141421",
  "payment_type" => "bbm_money",
  "transaction_time" => "2014-04-14 16:03:51",
  "transaction_status" => "pending",
  "permata_va_number => "8778999933221111",
  "gross_amount" => "145000.00"
);


Two important information details in the JSON response above are the status_code and permata_va_number. If the value of status_code is 201, the charge transaction process is successfully saved in Veritrans server with pending status. You should create BBM Money link with permata VA number previously obtained from Veritrans.

Wrapping BBM Money

  • BBM Money API Format

    bbmmoney://api/payment/imp?data=URL encoded JSON data


  • Create URL encode JSON data

    1. Create JSON data

       {
   "reference":"8778999933221111", // permata_va_number obtain from Veritrans 
   "callback_url": {
     "check_status":"https://example.com/order/check_status.html",
     "before_payment_error":"https://example.com/order/unfinish.html",
     "user_cancel":"https://example.com/order/cancel.html"
     }
 }

    2. URL Encodes the JSON

       %7B%22reference%22%3A%228778999933221111%22%2C%22callback_url%22%3A%7B%22check_status%22%3A%22https%3A%2F%2Fexample.com%2Forder%2Fcheck_status.html%22%2C%22before_payment_error%22%3A%22https%3A%2F%2Fexample.com%2Forder%2Funfinish.html%22%2C%22user_cancel%22%3A%22https%3A%2F%2Fexample.com%2Forder%2Fcancel.html%22%7D%7D

    3. Final URL

       bbmmoney://api/payment/imp?data=%7B%22reference%22%3A%228778999933221111%22%2C%22callback_url%22%3A%7B%22check_status%22%3A%22https%3A%2F%2Fexample.com%2Forder%2Fcheck_status.html%22%2C%22before_payment_error%22%3A%22https%3A%2F%2Fexample.com%2Forder%2Funfinish.html%22%2C%22user_cancel%22%3A%22https%3A%2F%2Fexample.com%2Forder%2Fcancel.html%22%7D%7D

Transaction Status

When charge transaction is successful, the transaction status is still pending. Customer has to complete the transaction. When this process is completed, the transaction status will be changed into success. You can check the transaction status change through MAP.

HTTP(S) POST

Veritrans' server will return a response to your server when customer has complete the payment. The sample JSON script that is returned by Veritrans is as followed: 

{
  "status_code": "200",
  "status_message": "Veritrans payment notification",
  "transaction_id": "9b48d9da-7b35-43f8-b375-026bf0b05538",
  "order_id": "201404141421",
  "payment_type": "bbm_money",
  "transaction_time": "2015-02-12 16:32:41",
  "transaction_status": "settlement",
  "signature_key": "ac9d80e274499386400244bd3737ec0a378b05ccb2b1c0a244becdf1c74b9139cf1172c39d28025e9730f21ddb065388a9008164d717b13f76a95aee1c467aef",
  "gross_amount": "145000.00"
