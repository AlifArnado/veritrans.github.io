Dashboard Operation

The tutorial below will assist you in operating Veritrans Merchant Admin Portal (MAP). If further assistance is required, please contact us at support@veritrans.co.id or visit our support page.

Merchant Admin Portal (MAP) is Veritrans dashboard for merchants to configure payment settings and manage transaction from customers.

Veritrans dashboard provides various utilities through MAP.

Sandbox vs. Production

Veritrans dashboard is available for development and production environment:

Sandbox

In the beginning, merchant is prompted to register in development environment (Sandbox).

URL: https://my.sandbox.veritrans.co.id.

You can test Veritrans' various payment methods available without triggering actual transaction on this environment. Testing credentials in sandbox environment can be found here.

Production

Our production environment is available once merchant has completed both technical integration in development environment and also legal documents requirement.

URL: https://my.veritrans.co.id.

Merchant can fully utilize Veritrans service and start accepting payment from customer in production environment.

Info: Note:
Please contact our support team at support@veritrans.co.id for assistance in migration from Sandbox to Production environment.


Checking Transaction Status

Transaction from every payment method will generate a transaction status. The figure below describes the various transaction status that we have.

Transaction Status Definition
Authorize Transaction is Authorized for a specific fund for the credit card holder.
Success Transaction is accepted and will go to settlement.
Challenge Credit card transaction is marked as suspicious transaction by Veritrans Fraud Detection System.Merchant has to manually Accept or Deny the transaction through Veritrans Dashboard.
See below to understand more about [Fraud Management].
Failure Failure transaction can be caused by various reasons:
1. Transaction is denied by the Bank.
2. Credit card transaction is rejected by Veritrans Fraud Detection System.
3. Pending Transaction has not been paid and is expired.
Pending Transaction that is asynchronous - Payment has to be completed outside Veritrans domain - will generate Pending status.
Once customer complete the transaction, the status will turn into 'success'.
Cancel Cancelled transaction can be caused by various reasons:
1. Successful transaction is cancelled before Settlement.
2. Challenge Transaction is denied by merchant.
Settlement Flow of funds from customer's account to merchant's account.
See below to know more about Settlement.


Info: Note:
Merchant can check the transaction status in details by clicking on the specific Order ID.


Settlement

Settlement is the moment when the funds flows from the customer's account to the merchant's account.
Settlement is different for every payment methods:

  • Non-Credit Card transaction:
    Successful transaction will be settled directly.
  • Credit Card transaction:
    Successful transaction from 00:00 - 23:59 will be settled as a batch on the next day at 16:00.


Cancel a Transaction

Merchant can cancel a transaction before the payment has been settled or still on pending status, since the funds has not been moved to the merchant's account.

The following payment is the list of payment that can be cancelled before settlement:

Platform Payment Channel
Card Payment Credit Card
Bank Transfer Permata Virtual Account, Mandiri Bill Payment
Direct Debit CIMB Clicks, e-pay BRI, BCA KlikPay
e-Wallet XL Tunai, BBM Money


To cancel a transaction that is not settled:

  1. Go to Transaction tab.
  2. Click on the 'Order ID' of the transaction.
  3. On the upper right corner, click on 'Action: Cancel'.

Info: Note:
Transaction that has been settled cannot be cancelled. Instead, you can request a refund by contacting support@veritrans.co.id.


Understand Fraud Management

Credit Card payment in Veritrans will be filtered by Aegis - Veritrans Fraud Detection System.
Aegis will then generate some recommendation:

  1. Accept - Payment passes Aegis fraud detection rules.
  2. Challenge - Payment is marked as suspicious by Aegis.
  3. Deny - Payment does not pass Aegis Fraud detection rules.

Challenge transaction can be managed manually by the merchant by:

  1. Go to Transaction tab.
  2. Click on the 'challenge' payment status.
  3. Merchant can pick on 'Accept' or 'Deny' the transaction.


Get Transaction Report

Merchant can get a transaction report in Excel or CSV.

Follow the steps below to get your transaction report:

  1. Go to Transactions tab.
  2. On the top right you will find a link to 'Download as Excel or CSV'.
  3. Choose which data you want to include in your report.
  4. Click 'Download CSV' or 'Download Excel'.

Merchant can also get daily transaction report via email by:

  1. Go to Settings tab.
  2. Under 'daily report', set the flag and enter the email.

Merchant can also filter transaction by clicking 'Search' on the top left in the Transaction tab.


Withdraw Fund from Veritrans

Veritrans merchant will be able to withdraw funds 7 days after the transaction is performed.
Merchant can choose when will be the right time to transfer their funds.

Follow the steps below to get your fund transferred:

  1. Log in to MAP Production.
  2. Go to Settings tab.
    Under 'billings', enter the bank account for the funds to be transferred.
  3. Go to Billings tab.
    Merchant will find all information regarding the pay out.
  4. Click 'Transfer'.
  5. Merchant will get the money in the same day if it is requested before 11 AM.
    Merchant will get the money on the next day if it is requested at 11 AM or after.
    There might be a delay on holiday time. Please contact us if you haven't get your money in 3 days.

Caution: Caution!
Your Bank Account has to be the same with the account signed on Veritrans Agreement.


Dashboard User Management

Follow the steps below to add your team member(s) Veritrans Dashboard account:

  1. Login to your Veritrans dashboard.
  2. Go to Account tab, under User Management.

Merchant can add user(s) and set the roles, for example:

  • Administrator (access to all MAP menu, usually reserved for business owner/ CEO/ GM).
  • Operations (access to transaction information and status. Can cancel (void), accept, and deny transactions).
  • Finance (access to billing related information. Can do pay outs and edit Bank account info).
  • Developer/ IT Staff (access to VT-Web/ VT- Direct setting and integration).
  • Customer Service (access to transaction information and status).

For security reason, we recommend you to set the roles carefully, especially in Production environment.

You can also check the user activity of the dashboard under 'Activity Log' or set a dedicated IP address to login to Veritrans Dashboard under 'IP Whitelist'.


Set Custom Fields Convention

Veritrans provides merchant with 3 custom fields that can be used for various utility.

Follow the steps below to set your custom field:

  1. Go to Configuration tab, under General Settings, set the custom fields label.
  2. Request Charge transaction with the custom fields value.
  3. These custom fields label and value can be checked at the Order ID of a transaction.
  4. Veritrans will include these custom fields when the HTTP POST Notification is sent.
