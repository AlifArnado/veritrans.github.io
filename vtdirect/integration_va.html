VT-Direct Integration (Virtual Account)

Every payment method has different integration steps available. The objective of this documentation is to assist you in integrating with VT-Direct with Virtual Account payment method. For more inquiries, please contact us at support@veritrans.co.id or visit our support web page.

One of the payment method offered by Veritrans is Virtual Account. Currently Veritrans has already integrated with Permata Bank for Virtual Account system. By using this service, your customer will have an option to make a payment via bank transfer to a Permata Bank Virtual Account Number.

To activate Virtual Account method of payment (automatically active on Sandbox mode), you have to contact Veritrans sales representative.

Integration VT-Direct Virtual Account

  1. Create Form
  2. Perform Charge Transaction
  3. Display Virtual Account
  4. Check Transaction Status

Unlike the other payment methods (Credit Card, Mandiri ClickPay and CIMB Clicks), Virtual Account payment process is not executed in your payment page. Customer has to pay with transfer method to complete transaction.

  • Step 1Create Form
  • Step 2Charge Transaction
  • Step 3Display Virtual Account
  • Step 4Transaction Status

Create Form

Virtual Account payment form will be made in the file form_vtdirect_va.html. Because most of the transaction process will not be made in the merchant's page, the form that we made will only show the steps of payment and the payment button of Virtual Account. When customer clicks this payment button, charge transaction process will be carried on the next step.

The following is a sample from the file form_vtdirect_va.html:

<head>
	<title>Bank Transfer</title>
</head>
<body>
	<h1>Bank Transfer</h1>
	<b>I want to pay with Bank Transfer</b>
	<form action="checkout_va.php" method="POST" id="payment-form">
      <button id="submit-button" type="submit">Pay</button>
    </form>
</body>

Charge Transaction

To perform charge transaction using Virtual Account, the following steps are required:

  • payment_type parameter has to be set as bank_transfer.

  • New JSON object has to be added, with the name bank_transfer, and has the pairing name/value as follows:

    • bank → Descriptions bank will use for Virtual Account

This step describes the Virtual Account charge transaction in the file checkout_va.php. To perform charge transaction, we will use Veritrans PHP library. The following is an example of charge transaction implementation in the file checkout_va.php:

<?php
include_once('Veritrans.php');

// TODO: change with your actual server_key that can be found on Merchant Administration Portal (MAP)
Veritrans_Config::$serverKey = "< Your Server Key >";
Veritrans_Config::$isProduction = false;

$transaction_details = array(
  'order_id'    => rand(),
  'gross_amount'  => 145000,
);

$customer_details = array(
    'first_name'    => "Andri",
    'last_name'     => "Litani",
    'email'         => "andri@litani.com",
    'phone'         => "081122334455"
    );

// Data yang akan dikirim untuk request charge transaction dengan credit card.
$transaction_data = array(
  'payment_type' => 'bank_transfer',
  'bank_transfer' => array(
      'bank' => "permata",
    ),
  'transaction_details' => $transaction_details,
  'customer_details' => $customer_details
  );

$result = Veritrans_VtDirect::charge($transaction_data);
?>

On the above example, we utilise the Virtual Account data, transaction data, and customer data as the charge transaction parameter. You can add additional data during charge transaction, such as: Customer address, delivery address, and item information. To add this data, you can check API Reference - Charge Transaksi.

Display Virtual Account

Veritrans server will return a response for every charge transaction in JSON form. Veritrans PHP Library will convert JSON into PHP object automatically. The sample JSON script that is returned by Veritrans is as follows:

$result = array(
  "status_code" => "201",
  "status_message" => "Success, PERMATA transaction is successful",
  "transaction_id" => "ff05337c-6c94-4f70-8e81-35acd89b688e",
  "order_id" => "201404141421",
  "payment_type" => "bank_transfer",
  "transaction_time" => "2014-04-14 16:03:51",
  "transaction_status" => "pending",
  "permata_va_number => "8778999933221111",
  "gross_amount" => "145000"
);

Two important information in the JSON response above is the status_code and permata_va_number. If the value of status_code is 201, the charge transaction process is successfully saved in Veritrans server with pending status. Merchant is required to show the permata_va_number to the customer so they can finish the Virtual Account transaction. You can add the following PHP code at the last line of the file checkout_va.php to do so:

<?php
include_once('Veritrans.php');

...

$result = Veritrans_VtDirect::charge($transaction_data);

if($result->status_code == "201")
{
  //success
  echo "<b>Bank Transfer</b> <br /><br />";
  echo "<b>Please complete payment with transfer method to this Permata bank virtual account number:</b> ".$result->permata_va_number;
  echo "<br /><br />";
  echo "1. You have 2 hours to complete payment before this transaction expired <br />";
  echo "2. If you need help, please contact 021-22334456";

} else {
  //error
  echo "Have error on transaction.<br />";
  echo "<h3>Result:</h3>";
  var_dump($result);
}
?>

Up to this point, the transaction is stored in MAP with pending status. Transaction status will change to success when your customer completes the payment.

To facilitate the payment process, we provide the payment simulator in sandbox Environment. Enter the account number previously obtained to check whether there is a transaction with the virtual account number.

If there is a transaction with virtual account numbers you provide, you can do a payment simulation.

Error During Charge Transaction

Under a certain case, charge transaction on VT-Direct can fail. This will prevents you in getting the virtual account number. Veritrans will return an error message if failure occurred, The reason of failure can be inspected from the status code displayed on the error message. The status code that generally occurs are:

Error Code Explanation
400 Validation error Format or content of the transaction data is invalid; Example: Wrong variable type, invalid payment method.
401 Access Denied Invalid client or server key.
402 Access Denied Merchant does not have access to utilise a certain method of payment.
406 Duplicate Order ID Order ID has previously been used and paid.
410 Account Inactive Merchant account has been inactive for a long period of time.

The complete list of status code can be found at API Reference - Status Code. If you have any difficulties, please contact us.

When charge transaction process is completed, you can check the transaction status. The steps to examine transaction status will be discussed on the next steps.

Transaction Status

When charge transaction is successful, the transaction status is still pending. Customer has to complete the transaction. When this process is completed, the transaction status will be changed into success. You can check the transaction status change through MAP.
