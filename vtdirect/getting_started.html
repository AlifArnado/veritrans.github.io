Getting Started

Target: Merchant understands what to prepare before integration with VT-Direct

There are a few things that need to be prepared before integrating with VT-Direct:

  1. Create a Veritrans Sandbox account.

    Utilized for development. Data and transaction made on this account is dummy in nature. During the development period, merchant has to use sandbox account. Sandbox account can be created here.

  2. Fill in your basic information in MAP (Merchant Administration Portal). This can be found in menu Settings - General Settings.



  3. Download Veritrans-PHP library [ Github | Zip ]

  4. Take note of the server key and client key of your account. Both keys can be obtained by

    • Login Sandbox MAP (Merchant Administration Portal).

    • Choose menu Settings and Access keys



    • Caution!!! Be discreet on your server key.

  5. Manage redirected URL

    The last step is to decide on how your customer will return to your website by configuring Finish URL, Unfinish URL, and Error URL. On VT-Direct, the most important field is finish URL. Customer will be redirected to finish URL when transaction is successful. This step is exclusively for the CIMB Clicks payment method.

    Unfinish URL and Error URL are the fields that are more applicable on VT-Web than VT-Direct. However, because all fields must be filled, please use your website address for Unfinish URL and Error URL. You can configure these URL in MAP.



    Explanation :

    • Fill Finish Redirect URL field with the callback URL.
    • Fill Unfinish Redirect URL and Error Redirect URL field with your website address.

When all steps above are completed, VT-Direct integration can be started.
