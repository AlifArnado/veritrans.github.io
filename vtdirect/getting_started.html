Getting Started

Authentication

Authentication is required to utilize the Payment API. We utilize BASIC AUTH http, whereby merchant is required to send us the server_key.

Server Key dan Client Key

Veritrans provides a server_key and client_key to allow merchant to communicate with the Payment API. Merchants can find the server_key and client_key in the MAP (Merchant Administration Portal).

  • server_key, is a key for merchant’s authentication when transactioning. It is a private key that needs to be kept secret by the merchant. .
  • client_key, is a key to generate token (only required in Credit Cards). it is a public key.

Basic Auth HTTP Header

To pass the authentication process, merchant needs to send server_key in the header of “Authorization”, as a standard specification BASIC AUTH. http://en.wikipedia.org/wiki/Basic_access_authentication

Basic authentication must use the following format :

username:password

Since there is no password for the Server Key, we will leave the password blank. Therefore, if our Server Key is :

94960ece-9513-4265-9cf2-67a4da330213

it will become :

94960ece-9513-4265-9cf2-67a4da330213:

(Note: make sure that there is a colon “:” after the Server Key)

Merchant then needs to convert it into Base 64 to get:

OTQ5NjBlY2UtOTUxMy00MjY1LTljZjItNjdhNGRhMzMwMjEzOg==

Therefore, the header Authorization will be:

Authorization: Basic OTQ5NjBlY2UtOTUxMy00MjY1LTljZjItNjdhNGRhMzMwMjEzOg==

Make sure the merchant put the string "Basic " before BASE 64 server_key.

HTTP Header

Given that the Payment API uses JSON to communicate with the merchant, there are several header that needs to be sent by the merchant for every transaction with the Payment API.

Below is the required header for every http request sent to the Payment API.

Header Value
Accept application/json
Content-Type application/json
Authorization Basic base64(server_key:)

URL Endpoint

URL endpoint is the URL address that will receive the request charge transaction. You will need the URL endpoint for next step. Veritrans provides separate endpoints for testing and production environments:

Testing / Development / Sandbox 	: https://api.sandbox.veritrans.co.id
Production 				: https://api.veritrans.co.id

During the development phase, please use Testing / Development / Sandbox endpoint. Once you are ready to “go live”, the merchant is required to change the URL endpoint to that for production.

Callback URL

Before conducting transactions using VT-Direct, merchant must set the callback URL via MAP (Merchant Administration Portal). Callback URL Settings can be found on the menu Settings → Configuration.
