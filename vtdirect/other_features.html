Other Features

The tutorial below will assist you in implementing other features on VT-Direct. If further assistance is required, please contact us at support@veritrans.co.id or visit our support page.


Currently, VT-Direct has additional features for credit card payment method which are:

  1. Pre-Authorization & Capture
  2. Installment
  3. BIN Promo
  4. One Click
  5. Two Clicks

The following is a tutorial to utilize these features as needed.

  • Pre-Authorization & Capture
  • Installment
  • BIN Promo
  • One Click
  • Two Clicks


Pre-Authorization & Capture

Pre-authorization is useful to check if a credit card is allowed to make a transaction for a certain value. When pre-authorization is completed, you can check the credit card status when the transaction has not occured yet. Transaction occurs when the capture operation is issued. Capture can only be completed when pre-authorization process is successful.

By default, operation charge transaction in Veritrans is a combination of pre-authorization and capture. If you wish to do these separately, please see the following step.

Pre-Authorization Transaction

Generally, pre-authorization process is similar to charge transaction. The difference lies on the request token parameter and charge transaction. You have to add the 'type' attribute with the value 'authorize' on both of the processes.

Following the same example as charge transaction, additional script on request token is as followed: 

<script type="text/javascript">

$(function(){
  
  ...

  var card = function(){
    return {
      'card_number'   : $("#card-number").val(),
      'card_exp_month'  : $("#card-expiry-month").val(),
      'card_exp_year'   : $("#card-expiry-year").val(),
      'card_cvv'      : $("#card-cvv").val(),
      'secure'       : true,
      'bank'       : 'bni',
      'gross_amount'   : $("#price").val(),

      // Ask for pre-authorization token id
      'type' : 'authorize',
    }
  };

  ...

</script>


The acquired token can be saved and used to perform pre-authorization during charge transaction. Parameters that is modified during charge transaction is as followed: 

<?php
include_once('Veritrans.php');

...

$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'token_id'  => $token_id,
    'bank'    => 'bni',

    // Authorize instead of normal charge operation
    'type' => 'authorize'
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details
  );

$result = Veritrans_VtDirect::charge($transaction_data);

?>


In the above example, pre-authorization response is saved in the variable $result. Structure and example of state $result when pre-authorization process is successful is as followed: 

$result = array(
    "status_code" => "200",
    "status_message" => "Success, Credit Card transaction is successful",
    "transaction_id" => "b505b192-f202-41ad-b82a-18678d0571a9",
    "order_id" => "201404211437",
    "payment_type" => "credit_card",
    "transaction_time" => "2014-04-21 14:35:32",
    "transaction_status" => "authorize",
    "fraud_status" => "noscore",
    "masked_card" => "481111-1114",
    "bank" => "bni",
    "gross_amount" => "150000.00"
);


The important attribute on pre-authorization response is transaction_id. You need to use this attribute value to perform the capture transaction.


Capture Transaction

To perform capture transaction, you simply have to prepare the attribute transaction_id and gross_amount. Both these attributes is related to the previous pre-authorization process. The gross_amount value cannot exceed the gross_amount on pre-authorization. If gross_amount attribute is not given during capture, the gross_amount value on pre-authorization will be used.

The following is the sample PHP code to perform capture transaction: 


<?php
include_once('Veritrans.php');

// TODO: change with your actual server_key that can be found on Merchant Administration Portal (MAP)
Veritrans_Config::$serverKey = "< Your Server Key >";
Veritrans_Config::$isProduction = false;

// Use transaction_id from Pre-authorization process
$transaction_id = 'b505b192-f202-41ad-b82a-18678d0571a9';
$amount = 150000;

$result = Veritrans_VtDirect::capture($transaction_id, $amount);
?>


On the above example, the capture response is saved in the variable $result. Structure and example of state $result when pre-authorization process is successful is as followed: 

$result = array(
    "status_code" => "200",
    "status_message" => "Success, Credit Card capture transaction is successful",
    "transaction_id" => "b505b192-f202-41ad-b82a-18678d0571a9",
    "order_id" => "201404211437",
    "payment_type" => "credit_card",
    "transaction_time" => "2014-04-21 14:35:32",
    "transaction_status" => "capture",
    "fraud_status" => "noscore",
    "masked_card" => "481111-1114",
    "bank" => "bni",
    "approval_code" => "140540",
    "gross_amount" => "150000.00"
);


Installment

To activate installment feature, merchant has to make an agreement with the acquiring bank. Merchant also needs to inquire if the MID for installment is ready from a Veritrans sales representative. The installment feature can be performed with a normal token or 3DS token.

The installment feature implementation is similar to charge transaksi. The difference lies on the request token and charge transaction. You have to add a 'Installment’ attribute and installment_term. The installment attribute with the respective value can be used in Sandbox environment:

Attribute Data Type Required Explanation
bank String(50) Yes Acquiring Bank that provides installment. In the moment, the valid value is "bni" or "mandiri"
installment_term Array Integer Yes Installment period. In the moment, the valid value is 6 or 12


Using the same example in charge transaction, script that is modified during request token is as followed: 

<script type="text/javascript">

$(function(){
  
  ...

  var card = function(){
    return {
      'card_number'   : $("#card-number").val(),
      'card_exp_month'  : $("#card-expiry-month").val(),
      'card_exp_year'   : $("#card-expiry-year").val(),
      'card_cvv'      : $("#card-cvv").val(),
      'secure'       : true,

      // Ask for 12 months installment token_id
      'installment' : true, 
      'installment_term' : 12,
      
      'bank'       : 'bni',
      'gross_amount'   : $("#price").val(),
    }
  };

  ...

</script>


The acquired token can be saved and then used to perform the charge transaction. Attribute that has to be added to perform installment is installment_term. Parameters that are modified during charge transaction is as followed: 

<?php
include_once('Veritrans.php');

...

$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'token_id'  => $token_id,
    'bank'    => 'bni',
    'installment_term' => 12
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details,
  );

$result = Veritrans_VtDirect::charge($transaction_data);

...

?>


In the above example, the installment response is saved in the variable $result. Structure and example of state $result when installment process is successful is as followed: 

$result = array(
    "status_code" => "200",
    "status_message" => "Success, Credit Card Installment transaction is successful",
    "transaction_id" => "2a913000-4ed6-4c91-aff1-94e5ffb1b737",
    "order_id" => "201404231811",
    "payment_type" => "credit_card",
    "transaction_time" => "2014-04-23 18:11:44",
    "transaction_status" => "capture",
    "masked_card" => "481111-1114",
    "bank" => "bni",
    "approval_code" => "140540",
    "gross_amount" => "10000.00"
    );


BIN Promo

BIN (Bank Identification Number) is the first 6 digit number of a credit card. The issuing bank for the credit card can be identified from these numbers. BIN promo is the feature that allows merchant to give out the deal based on the promotion given by the issuing bank. Merchant can use the first 1-8 digits of the credit card to utilize the BIN Promo feature.

To use this feature, you have to first list out the BIN that offers a promotion. The BIN list will become the transaction parameter bins. Transactions can only be performed by credit cards that are included in the BIN list. Use the BIN list as a value from the attribute bins during the charge transaction.

Following the same transaction in integration, the sample PHP script that can be used is as followed: 

<?php
include_once('Veritrans.php');

...

$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'token_id'  => $token_id,
    'bank'    => 'bni',
    'bins' => array("48111111", "3111", "5")
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details,
  );
try{
  $result = Veritrans_VtDirect::charge($transaction_data);
} catch (Exception $e){
  echo $e->getMessage();
}
...
?>


On the above example, the BIN that is listed is 48111111, 3111, and 5. If a card with the BIN "411111" is used, charge transaction will fail.


One Click

The one click feature allows you to use one token on several transactions. The token represents the credit card number, expiry date, CVV, and 3D Secure authentication of the customer. This will make transactions easier since the credit card information needs to be filled only once and not recurrently.

Do note that token will be activated only after the initial transaction is performed. To use the one click feature, 3D Secure feature has to be used.

In general, there are two steps required to utilize one click feature:

  1. Initial Transaction
    • Customer fill the credit card number, CVV, expiry date
    • Customer perform 3D Secure authentication
    • Merchant receives token
    • Merchant uses token to charge transaction
    • When transaction successful, merchant saves customer’s token in the database. We name this token as saved_token.
  2. Following Transaction
    • Merchant retrieve customer saved_token from database
    • Merchant uses saved_token to charge transaction


Initial Transaction

To implement one click, you simply add the attribute save_token_id on initial transaction. Following the example on charge transaction, the modified PHP script during charge transaction is as follows: 

<?php
include_once('Veritrans.php');

...

$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'token_id'  => $token_id,
    'bank'    => 'bni',
    'save_token_id' => true
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details,
  );

$result = Veritrans_VtDirect::charge($transaction_data);

...

?>


On the above example, one click response is saved in the variable $result. Structure and example of state $result when one click process is successful is shown below: 

array(
  "transaction_id" => "ca5e0ff5-4297-48ef-aa25-b3ab018d4ead",
  "order_id" => "order52",
  "gross_amount" => "200000.00",
  "payment_type" => "credit_card",
  "transaction_time" => "2014-03-27 17:20:53",
  "transaction_status" => "capture",
  "fraud_status" => "accept",
  "masked_card" => "481111-1114",
  "saved_token_id" => "4011117d16c884-2cc7-4624-b0a8-10273b7f6cc8",
  "saved_token_id_expired_at" => "2024-03-27 17:20:53",
  "status_code" => "200",
  "bank" => "bni",
  "approval_code" => "140540",
  "status_message" => "Success, Credit Card 3D Secure transaction is successful"
);


Two important attributes for a successful one click response is saved_token_id and saved_token_id_expired. saved_token_id is the token that you save when the customer makes a following transaction. This attribute is what we referred as saved_token on the previous explanation. The value from saved_token can be used repeatedly until saved_token_id_expired.


Following Transaction

On the following transaction, customer does not have to fill in credit card information anymore and charge transaction process can be performed using saved token.

The following example is the charge following the transaction using saved_token_id: 

<?php
include_once('Veritrans.php');

...

//Retrieve saved token id for this customer from database
$saved_token_id = "<saved token id>";

$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'token_id'  => $saved_token_id,
    'bank'    => 'bni',
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details,
  );

$result = Veritrans_VtDirect::charge($transaction_data);

...

?>


Structure and example of state $result is the same with the standard charge transaction process. 

array(
    "transaction_id" => "05dfbd23-b767-4748-8fe9-7e30769e1857",
    "order_id" => "order53",
    "gross_amount" => "200000.00",
    "payment_type" => "credit_card",
    "transaction_time" => "2014-03-27 17:25:12",
    "transaction_status" => "capture",
    "status_code" => "200",
    "bank" => "bni",
    "approval_code" => "140540",
    "status_message" => "Success, Credit Card transaction is successful"
);


Two Clicks

Token on two clicks represents the customer’s card number and expiry date. Your customer simply has to fill in the CVV and 3D Secure authentication on the following transaction. The difference between one click is that the two clicks feature can be used in both 3D Secure and non-3D Secure transaction.

In general, there are two steps required to utilise two clicks feature:

  1. Initial Transaction
    • Customer fills in the credit card number, CVV, expiry date
    • Customer performs 3D Secure authentication
    • Merchant receives token
    • Merchant uses token to charge transaction
    • When transaction successful, merchant save customer’s token in the database. We name this token as saved_token.
  2. Following Transaction
    • Merchant retrieves customer saved_token from database
    • Customer fills in CVV
    • Perform 3D Secure authentication if required
    • Merchant uses saved_token and CVV to acquire the new token
    • Merchant uses new token to charge transaction


Initial Transaction

Initial transaction implementation is similar to standard charge transaction. You simply have to add the attribute saved_token_id during charge transaction. When transaction is successful, the token is saved for the following transaction. 

<?php
include_once('Veritrans.php');

...

$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'token_id'  => $token_id,
    'bank'    => 'bni',
    'save_token_id' => true
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details,
  );

$result = Veritrans_VtDirect::charge($transaction_data);
$status = $result->transaction_status;

if($status == 'capture'){
  // save 'token_id' for this customer in database
}
...

?>


On the above example, two clicks response is saved in the variable $result. Structure and example of state $result when two clicks process is successful is as followed: 

array(
  "transaction_id" => "ca5e0ff5-4297-48ef-aa25-b3ab018d4ead",
  "order_id" => "order52",
  "gross_amount" => "200000.00",
  "payment_type" => "credit_card",
  "transaction_time" => "2014-03-27 17:20:53",
  "transaction_status" => "capture",
  "fraud_status" => "accept",
  "masked_card" => "481111-1114",
  "saved_token_id" => "4011117d16c884-2cc7-4624-b0a8-10273b7f6cc8",
  "saved_token_id_expired_at" => "2024-03-27 17:20:53",
  "status_code" => "200",
  "bank" => "bni",
  "approval_code" => "140540",
  "status_message" => "Success, Credit Card 3D Secure transaction is successful"
);


With a successful two clicks response, the saved_token_id along with saved_token_id_expired. saved_token_id is saved for the customer's conveniance in a following transaction. This attribute is what we referred as saved_token on two clicks step. The value from saved_token can be used repeatedly until saved_token_id_expired.


Following Transaction

On the following transaction, we use the saved_token and the customer filled CVV to acquire new token. two_click attributes also need to be added when new request for token is performed. The new token will be used for the following transaction. The new token will be automatically filled in <input id="token-id" name="token-id" type="hidden" /> by veritrans.min.js.

Do note that you do not use saved_token to perform charge transaction. You use saved_token as one of the conditions to acquire a new token. You do not save new token from the customer to the database.

Field number and expiry date in the form can be removed because the customer simply needs to fill the CVV and perform 3D Secure authentication. Using the same example in charge transaction, the following example is the new request token using the saved_token and CVV from the customer: 

<script type="text/javascript">

$(function(){
  
  ...

  var card = function(){
    return {
      'card_cvv' : $("#card-cvv").val(),
      'token_id' : '<retrieve this customer saved_token from your database>',


      'two_click' : true,
      'secure' : true,
      'bank' : 'bni',
      'gross_amount' : $("#price").val(),
    }
  };

  ...

</script>


The following charge transaction in two steps is the same with the standard process of charge transaction. You simply use the new token as the token_id. 

<?php
include_once('Veritrans.php');

...

$token_id = $_POST['token-id'];

$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'token_id'  => $token_id,
    'bank'    => 'bni',
    ),
  'transaction_details'   => $transaction_details,
  'customer_details' => $customer_details,
  );

$result = Veritrans_VtDirect::charge($transaction_data);

...

?>


Structure and example of state $result is the same with the standard charge transaction process. 

array(
    "transaction_id" => "05dfbd23-b767-4748-8fe9-7e30769e1857",
    "order_id" => "order53",
    "gross_amount" => "200000.00",
    "payment_type" => "credit_card",
    "transaction_time" => "2014-03-27 17:25:12",
    "transaction_status" => "capture",
    "status_code" => "200",
    "bank" => "bni",
    "approval_code" => "140540",
    "status_message" => "Success, Credit Card transaction is successful"
