Other Features

BIN Promo

The BIN promo feature enables a merchant to offer promotions based on the customer's credit card BIN (Bank Identification Number). BIN is the first six digits in the credit card number that helps to identify which particular bank issues the card. This feature allows a merchant to cooperate with a particular bank for any marketing offers/programs.

BIN promo works by the whitelist method. When the merchant's server sends a customer's transaction with the discounted price to Veritrans, it also sends along a list of credit card BIN numbers that are allowed to make the transaction. Veritrans will check if the credit card number used by the customer is in the list of allowed BIN. If not, the customer will not be able to finish the transaction.

For BIN promo transaction, merchant only need to add a JSON parameter representing the BIN card numbers allowed.

Endpoint
Endpoint : https://payments.veritrans.co.id/vtdirect/v1/charges
HTTP Method : POST
Header
Accept : application/json
Content-Type : application/json
Authorization : Basic Base64 (server-key:)

Note: Endpoint, HTTP Method, and Header for installment transactions are exactly the same with the ones for a normal charge transactions

3D Secure Transaction

The 3-D Secure protocol is a technical platform that includes technical specifications and requirements for issuers, acquirers, and merchants. In addition to utilizing the widely supported Internet technology Secure Sockets Layer (SSL) encryption to protect payment card information during transmission over the Internet, 3-D Secure uses cardholder authentication to verify the parties involved in the transaction.

For 3D Secure Transaction, merchant only need add some flag in Normal Credit Card Transaction request. Please refer to the sample code below for the additional JSON parameters that you need to add for a 3D secure transaction.

Endpoint
Endpoint : https://payments.veritrans.co.id/vtdirect/v1/charges
HTTP Method : POST
Header
Accept : application/json
Content-Type : application/json
Authorization : Basic Base64 (server-key:)

Note: Endpoint, HTTP Method, and Header for 3D secure transactions are exactly the same with the ones for a normal charge transactions.

3D Secure Callback Response

After transaction finished by customer, veritrans will send a HTTP POST message back to the merchant’s 3dsecure_callback_url with a POST parameter called "vtdirect":

$data = $_POST[‘vtdirect’];

$data will include a JSON String with the following information:

{
  "data": {
    "order_id": "768798",
    "transaction_status": "authorize" 
    },
  "code": "VD00",
  "status": "success",
  "message": "Success get status transaction"
}
3D Secure Notification Response

Other than the callback, veritrans will send a notification to the 3dsecure_notification_url set by the merchant. For security reasons, we suggest that you utilize the data in the 3dsecure_notification_url instead of the 3dsecure_callback_url to update the status of a transaction.

Below is the data that we will send to your notification_url:

{
  "data": {
    "order_id": "77897982",
    "transaction_status": "authorize",
    "server_key": "..."
    },
  "code": "VD00",
  "status": "success",
  "message": "Success get status transaction"
}

IMPORTANT: For security purposes, please make sure that the server_key included in the notification is the same as what you have on your server.

Point Transaction

You can also accept payments via credit card points that the credit card holder has accumulated. Points can be used in full or for partial payment of the purchase amount.

Check point balance

Before your customer conduct a transaction using points, you will need to show to the customer how many points he or she has on her card. The command to check the point balance is below:

Endpoint:
Endpoint : https://payments.veritrans.co.id/vtdirect/v1/balance
HTTP Method : POST
Header:
Accept : application/json
Content-Type : application/json
Authorization : Basic Base64 (server-key:) (Please look at the section on Charge transaction)
Body Content:
{
  "token_id" : "e0fcc426-4a7c-4902-a332-3a3e563ec86e-410505-1244", 
  "bank" : "bni"
}
Parameter Description Example
Token_id String. Token e0fcc426-4a7c-4902- a332-3a3e563ec86e- 410505-1244
Bank Name of bank BNI, CIMB
Sample Response:
{
  "data": {
    "card_number": "4105XXXXXXXXXX44",
    "balance": 4907950
  },
  "code": "VD00",
  "status": "success",
  "message": "Success get balance inquiry"
}

Installment Transaction

To accept installment transactions, merchants are required to sign a separate agreement with the acquiring bank. Conducting an installment transactions with VT-direct is very similar to a normal credit transactions, but with the addition of several JSON parameters.

Note: Endpoint, HTTP Method, and Header for installment transactions are exactly the same with the ones for a normal charge transactions

Endpoint
Endpoint : https://payments.veritrans.co.id/vtdirect/v1/charges
HTTP Method : POST
Header
Accept : application/json
Content-Type : application/json
Authorization : Basic Base64 (server-key:)
Body Content
