Fitur Lain

Pre-Authorization

Pre-Authorization Transaction

Untuk melakukan pre-authorization charge transaksi, "type": authorize harus ditambahkan di JSON request dalam bagian "credit_card". Contoh dalam PHP:

// Data yang akan dikirim untuk request charge transaction dengan credit card.
$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'type'          => authorize,
           'token_id'   => $token_id,
    'bank'    => 'bni'
  ),
  'transaction_details'   => $transaction_details,
  'item_details'      => $items,
  'customer_details'    => $customer_details
);

Pre-Authorization Response

Response yang akan di dapat kalau charge transaksi berhasil:

{
    "status_code": "200",
    "status_message": "OK, Credit Card transaction is successful",
    "transaction_id": "948996b4-9323-4ac3-b3ea-bd7035f9bf2b",
    "order_id": "201404211437",
    "payment_type": "credit_card",
    "transaction_time": "2014-04-21 14:35:32",
    "transaction_status": "authorize",
    "fraud_status": "noscore",
    "masked_card": "411111-1111",
    "gross_amount": "1000000.00"
}

Capture Transaction

Untuk melakukan capture transaksi, URL endpoint harus memakai /v2/capture. Contoh JSON request seperti berikut:

{
  "transaction_id" : "77e137c0-019b-40c9-95b2-76237773e449",
  "gross_amount" : 1000000
}

"gross_amount" elemen tidak boleh melebihi jumlah yang dilakukan pada saat melakukan authorize charge transaksi. Jika "gross_amount" elemen tidak tertera, jumlah total akan disesuaikan dengan jumlah pada saat melakukan authorize charge transaksi. Hasil contoh capture response yang sukses sebagai berikut:

{
    "status_code": "200",
    "status_message": "OK, Credit Card capture transaction is successful",
    "transaction_id": "948996b4-9323-4ac3-b3ea-bd7035f9bf2b",
    "order_id": "201404211437",
    "payment_type": "credit_card",
    "transaction_time": "2014-04-21 14:35:32",
    "transaction_status": "capture",
    "fraud_status": "noscore",
    "masked_card": "411111-1111",
    "gross_amount": "1000000.00"
}

Jika "gross_amount" elemen tertera dengan jumlah yang lebih kecil daripada saat melakukan authorize charge transaksi, maka dalam saat melakukan capture, jumlah tersebut akan digunakan dan response yang didapat akan disesuaikan. Contoh capture transaksi dan response nya sebagai berikut:

{
  "transaction_id" : "77e137c0-019b-40c9-95b2-76237773e449",
  "gross_amount" : 100000
}

{
    "status_code": "200",
    "status_message": "OK, Credit Card capture transaction is successful",
    "transaction_id": "948996b4-9323-4ac3-b3ea-bd7035f9bf2b",
    "order_id": "201404211437",
    "payment_type": "credit_card",
    "transaction_time": "2014-04-21 14:35:32",
    "transaction_status": "capture",
    "fraud_status": "noscore",
    "masked_card": "411111-1111",
    "gross_amount": "100000.00"
}

Installment

Charge Transaction

Installment transaksi dapat dilakukan dengan token biasa atau token 3DS.

Untuk melakukan installment charge transaksi, "installment_term": < angka > harus ditambahkan di JSON request dalam bagian "credit_card". Angka untuk "installment_term" harus lebih besar dari satu.

Contoh dalam PHP:

// Data yang akan dikirim untuk request charge transaction dengan credit card.
$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'installment_term'   => 3,
           'token_id'   => $token_id,
    'bank'    => 'bni'
  ),
  'transaction_details'   => $transaction_details,
  'item_details'      => $items,
  'customer_details'    => $customer_details
);

Charge Response

Response yang akan di dapat kalau charge transaksi berhasil:

{
    "status_code": "200",
    "status_message": "OK, Credit Card Installment transaction is successful",
    "transaction_id": "2a913000-4ed6-4c91-aff1-94e5ffb1b737",
    "order_id": "201404231811",
    "payment_type": "credit_card",
    "transaction_time": "2014-04-23 18:11:44",
    "transaction_status": "capture",
    "masked_card": "411111-1236",
    "gross_amount": "10000.00"
}

One Click

Mendapatkan Token 3DSecure

Token 3DSecure harus diaktifkan untuk menggunakan fitur One Click. Silahkan lihat bagian Charge Transaction - Mengaktifkan fitur VT-Direct 3DSecure untuk request parameter yang dibutuhkan oleh token 3DSecure.

Charge Request Pertama

Untuk One Click charge request pertama, "save_token_id": true harus di masukan di JSON request dalam bagian "credit_card". Dengan demikian, token akan disimpan untuk pembayaran berikutnya. Contoh dalam PHP:

// Data yang akan dikirim untuk request charge transaction dengan credit card.
$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'save_token_id' => true,
           'token_id'   => $token_id,
    'bank'    => 'bni'
  ),
  'transaction_details'   => $transaction_details,
  'item_details'      => $items,
  'customer_details'    => $customer_details
);

Response yang akan dikembalikan jika transaksi sukses seperti berikut ini:

{
    "transaction_id": "ca5e0ff5-4297-48ef-aa25-b3ab018d4ead",
    "order_id": "order52",
    "gross_amount": "200000",
    "payment_type": "credit_card",
    "transaction_time": "2014-03-27 17:20:53",
    "transaction_status": "capture",
    "fraud_status": "accept",
    "masked_card": "411111-1111",
    "saved_token_id": "4111117d16c884-2cc7-4624-b0a8-10273b7f6cc8",
    "saved_token_id_expired_at": "2024-03-27 17:20:53",
    "status_code": "200",
    "status_message": "OK, Credit Card 3D Secure transaction is successful"
}

Dari response di atas, hasil "save_token_id" perlu disimpan oleh merchant, misalnya di database, sehingga dapat digunakan untuk pembayaran berikutnya.

Charge Request Berikutnya

Untuk charge request berikutnya, merchant harus menggunakan token id yang didapat dari response yang pertama, "save_token_id". Token id tersebut diletakkan di bagian "credit_card".

"credit_card": {
    "token_id": "save_token_id",
    "bank": "bni"
}

JSON request format untuk charge transaksi sama seperti biasanya. Silahkan lihat bagian Charge Transaction untuk format selengkapnya. Jika response sukses akan mendapatkan hasil sebagai berikut:

{
    "transaction_id": "05dfbd23-b767-4748-8fe9-7e30769e1857",
    "order_id": "order53",
    "gross_amount": "200000",
    "payment_type": "credit_card",
    "transaction_time": "2014-03-27 17:25:12",
    "transaction_status": "capture",
    "status_code": "200",
    "status_message": "OK, Credit Card transaction is successful"
}
