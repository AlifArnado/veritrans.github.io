Fitur Lain

Pre-Authorization

Pre-Authorization Transaction

To conduct pre-authorization charge transaction "type": authorize should be added in JSON request in "credit_card" part. Sample in PHP:

// Data yang akan dikirim untuk request charge transaction dengan credit card.
$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'type'          => authorize,
           'token_id'   => $token_id,
    'bank'    => 'bni'
  ),
  'transaction_details'   => $transaction_details,
  'item_details'      => $items,
  'customer_details'    => $customer_details
);

Pre-Authorization Response

Obtained response if charge transaction success:

{
    "status_code": "200",
    "status_message": "OK, Credit Card transaction is successful",
    "transaction_id": "948996b4-9323-4ac3-b3ea-bd7035f9bf2b",
    "order_id": "201404211437",
    "payment_type": "credit_card",
    "transaction_time": "2014-04-21 14:35:32",
    "transaction_status": "authorize",
    "fraud_status": "noscore",
    "masked_card": "411111-1111",
    "gross_amount": "1000000.00"
}

Capture Transaction

To capture transaction, URL endpoint should use /v2/capture. Sample JSON request:

{
  "transaction_id" : "77e137c0-019b-40c9-95b2-76237773e449",
  "gross_amount" : 1000000
}

"gross_amount" element should not exceed the amount at authorize charge transactions. If "gross_amount" element is not listed, the total amount will be adjusted by the amount at authorize charge transactions. Hasil contoh capture response yang sukses sebagai berikut:

{
    "status_code": "200",
    "status_message": "OK, Credit Card capture transaction is successful",
    "transaction_id": "948996b4-9323-4ac3-b3ea-bd7035f9bf2b",
    "order_id": "201404211437",
    "payment_type": "credit_card",
    "transaction_time": "2014-04-21 14:35:32",
    "transaction_status": "capture",
    "fraud_status": "noscore",
    "masked_card": "411111-1111",
    "gross_amount": "1000000.00"
}

If "gross_amount" element is listed by a smaller amount than at authorize charge transaction, so when capture transaction, the amount will be used and obtained response will be adjusted. Examples of capture transactions and its response as follows:

{
  "transaction_id" : "77e137c0-019b-40c9-95b2-76237773e449",
  "gross_amount" : 100000
}

{
    "status_code": "200",
    "status_message": "OK, Credit Card capture transaction is successful",
    "transaction_id": "948996b4-9323-4ac3-b3ea-bd7035f9bf2b",
    "order_id": "201404211437",
    "payment_type": "credit_card",
    "transaction_time": "2014-04-21 14:35:32",
    "transaction_status": "capture",
    "fraud_status": "noscore",
    "masked_card": "411111-1111",
    "gross_amount": "100000.00"
}

Installment

Charge Transaction

Transaction installment can be done by use usual token or 3DS token.

To conduct installment charge transaction, "installment_term": < angka > should be added in JSON request in "credit_card part. Number for "installment_term" must be greater than one.

Sample in PHP:

// Data yang akan dikirim untuk request charge transaction dengan credit card.
$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'installment_term'   => 3,
           'token_id'   => $token_id,
    'bank'    => 'bni'
  ),
  'transaction_details'   => $transaction_details,
  'item_details'      => $items,
  'customer_details'    => $customer_details
);

Charge Response

Obtained response if charge transaction success:

{
    "status_code": "200",
    "status_message": "OK, Credit Card Installment transaction is successful",
    "transaction_id": "2a913000-4ed6-4c91-aff1-94e5ffb1b737",
    "order_id": "201404231811",
    "payment_type": "credit_card",
    "transaction_time": "2014-04-23 18:11:44",
    "transaction_status": "capture",
    "masked_card": "411111-1236",
    "gross_amount": "10000.00"
}

One Click

Obtaining 3DSecure Token

3DSecure token must be activated to use One Click feature. See Step 2: Charge Transaction - VT-Direct Integration to request parameter that required for 3DSecure token.

First Charge Request

For first charge request of One Click, "save_token_id": true should be added in JSON request in "credit_card" part. So the token will be saved for future payments. Sample in PHP:

// Data yang akan dikirim untuk request charge transaction dengan credit card.
$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'save_token_id' => true,
           'token_id'   => $token_id,
    'bank'    => 'bni'
  ),
  'transaction_details'   => $transaction_details,
  'item_details'      => $items,
  'customer_details'    => $customer_details
);

Response will be returned if transaction succeed as follows:

{
    "transaction_id": "ca5e0ff5-4297-48ef-aa25-b3ab018d4ead",
    "order_id": "order52",
    "gross_amount": "200000",
    "payment_type": "credit_card",
    "transaction_time": "2014-03-27 17:20:53",
    "transaction_status": "capture",
    "fraud_status": "accept",
    "masked_card": "411111-1111",
    "saved_token_id": "4111117d16c884-2cc7-4624-b0a8-10273b7f6cc8",
    "saved_token_id_expired_at": "2024-03-27 17:20:53",
    "status_code": "200",
    "status_message": "OK, Credit Card 3D Secure transaction is successful"
}

Based on response above, "save_token_id" result needs to be stored by the merchant, for example in the database, so it can be used for future payments.

Next Charge Request

For next charge request, merchant have to use token id that obtained from first response, "save_token_id. The token id is placed in "credit_card" part.

"credit_card": {
    "token_id": "save_token_id",
    "bank": "bni"
}

JSON request format for charge transaction are same as usual. See [Charge Transaction] (/ vtdirect / integration.html) for more formats. If the response successfully will get the following results:

{
    "transaction_id": "05dfbd23-b767-4748-8fe9-7e30769e1857",
    "order_id": "order53",
    "gross_amount": "200000",
    "payment_type": "credit_card",
    "transaction_time": "2014-03-27 17:25:12",
    "transaction_status": "capture",
    "status_code": "200",
    "status_message": "OK, Credit Card transaction is successful"
}

Two Clicks

Obtaining 3DSecure Token

Same as One Click, 3D Secure token must be activated to use Two Clicks feature. See Activating 3DSecure to request parameter that required for 3DSecure token.

First Charge Request

First charge request for Two Clicks have exactly same format with the first charge on One Click. On first charge request, "save_token_id": true must be added on JSON request in "credit_card" part. So, the token will be saved for future payments. Sample in PHP:

// Data yang akan dikirim untuk request charge transaction dengan credit card.
$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'save_token_id' => true,
    'token_id'   => $token_id,
    'bank'    => 'cimb'
  ),
  'transaction_details'   => $transaction_details,
  'item_details'      => $items,
  'customer_details'    => $customer_details
);

Response will be returned if transaction succeed as follows:

{
  "transaction_id": "ca5e0ff5-4297-48ef-aa25-b3ab018d4ead",
  "order_id": "order52",
  "gross_amount": "200000",
  "payment_type": "credit_card",
  "transaction_time": "2014-03-27 17:20:53",
  "transaction_status": "capture",
  "fraud_status": "accept",
  "masked_card": "411111-1111",
  "saved_token_id": "4111117d16c884-2cc7-4624-b0a8-10273b7f6cc8",
  "saved_token_id_expired_at": "2024-03-27 17:20:53",
  "status_code": "200",
  "status_message": "OK, Credit Card 3D Secure transaction is successful"
}

From response above, "saved_token_id" result needs to be stored by the merchant, for example in the database, so it can be used for future payments.

Next Charge Request

For next charge request, ask the customer to enter the credit card CVV and get "token_cvv_id" from the CVV. In addition, the merchant must also add a field to send the token to process the transaction. Examples of fields that can be applied in the HTML code is as below.

<input name="token_cvv_id" id="token_cvv_id" type="hidden">

To get "token_cvv_id", the merchant must add parameter "two_click": true on the function card used in the example script below. Then the credit card details sent to our endpoint together with the client-key merchant on Veritrans.client_key . After receiving the response, field "token_cvv_id" was able to be filled with values ​​in response.token_id. Example javascript code to get "token_cvv_id"</ code> is as below.

<script type="text/javascript">
  $(function(){
   //endpoint
   Veritrans.url = "https://api.sandbox.veritrans.co.id/v2/token";
   //client_key
   Veritrans.client_key = "bb558bb6-371a-415b-b7ee-bf285f33868f";
   var card = function(){
   return {
       'card_cvv'   : $(".card-cvv").val(),
       'two_click'  : true
     }
   };

   // handler when user click the 'Pay' button.
   $('.submit-button').click(function(event){
     event.preventDefault();
     $(this).attr("disabled", "disabled");
     Veritrans.token(card, callback);
     return false;
   });

   function callback(response) {
     if (response.status_code == '200') {
      // success
       $("#token_cvv_id").val(response.token_id);
       $("#payment-form").submit();
     } else {
       // failed request token
       $('.submit-button').removeAttr('disabled');
       // Show status message.
       $('#message').text(response.status_message);
       console.log(JSON.stringify(response));
     }
   }
  });
</script>
