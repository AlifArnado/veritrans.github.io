VT-Direct Integration (e-Pay BRI)

Every payment method has different integration steps available. The objective of this documentation is to assist you in integrating with VT-Direct with e-Pay BRI payment method. For more inquiries, please contact us at support@veritrans.co.id or visit our support web page.



e-Pay BRI is one of the internet banking payment method that is provided by BRI Bank. This method allows customer to perform transaction by redirecting the customer to the e-Pay BRI payment page. For more information, please visit e-Pay BRI website.

To activate e-Pay BRI payment method (Automatically active on Sandbox mode), you have to contact a Veritrans sales representative.

Integration Steps VT-Direct e-Pay BRI

  1. Create Form
  2. Perform Charge Transaction
  3. Redirect
  4. Check Transaction Status

Unlike the other payment method (Credit Card and Mandiri ClickPay), e-Pay BRI payment process are not executed in your payment page. Customer will be redirected to e-Pay BRI page to complete the transaction.

  • Step 1Create Form
  • Step 2Perform Charge Transaction
  • Step 3Redirect
  • Step 4Check Transaction Status


Create Form

e-Pay BRI payment form will be made in the file form_vtdirect_bri.php. Because most of the transaction process will be made in e-Pay BRI page, the form that we made will only shows the steps of payment and the payment button of e-Pay BRI. When your customer click this payment button, charge transaction process will be carried to the next step.

The following is a sample from the file form_vtdirect_bri.php: 

<html>
  <head>
    <title>VT-Direct e-Pay BRI Form Example</title>
  </head>
<body>
    <h1>e-Pay BRI Transaction</h1>
    <ol>
      <li>Please make sure that you have a User ID for BRI e-Pay and have registered for mTOKEN before going through with the payment </li>
      <li>To obtain a User ID for BRI e-Pay, please register at the nearest BRI ATM.</li>
      <li>Please visit the nearest BRI branch office to register the mTOKEN.</li>
      <li>Payment via BRI e-Pay will be processed online and your BRI bank account balance will be deducted automatically based on your total amount.</li>
      <li>Your transaction will be cancelled if payment is not completed within 2 hours.</li>
    </ol>
    <form action="checkout_bri.php" method="POST" id="payment-form">
      <button id="submit-button" type="submit">Pay</button>
    </form>
  </body>


Charge Transaction

To perform the charge transaction using e-Pay BRI, the steps required are as followed:

  • payment_type parameter has to be set as bri_epay.

This step describes the charge transaction e-Pay BRI in the file checkout_bri.php. To perform the charge transaction, we will use Veritrans' PHP library. The following is an example of charge transaction implementation in the file checkout_bri.php: 

<?php
include_once('Veritrans.php');

// TODO: change with your actual server_key that can be found on Merchant Administration Portal (MAP)
Veritrans_Config::$serverKey = "< Your Server Key >";
Veritrans_Config::$isProduction = false;

$transaction_details = array(
  'order_id'    => rand(),
  'gross_amount'  => 145000,
);

$customer_details = array(
    'first_name'    => "Andri",
    'last_name'     => "Litani",
    'email'         => "andri@litani.com",
    'phone'         => "081122334455"
    );

// Data that will be sent for charge transaction request with e-Pay BRI.
$transaction_data = array(
  'payment_type' => 'bri_epay',
  'transaction_details' => $transaction_details,
  'customer_details' => $customer_details
  );

$result = Veritrans_VtDirect::charge($transaction_data);
?>


On the above example, we utilize the e-Pay BRI, transaction, and customer data as the charge transaction parameter. You can add additional data during charge transaction such as customer address, delivery address, and item information. To add this data, you can check API Reference - Charge Transaction.


Redirect

Veritrans' server will return a response for every charge transaction in JSON form. Veritrans' PHP Library will convert JSON into a PHP object automatically. The sample JSON script that is returned by Veritrans is as followed: 

$result = array(
  "status_code" => "201",
  "status_message" => "Success, BRI E-Pay transaction is successful",
  "transaction_id" => "cf0ec2c0-3e52-4f20-94fc-a6bc8ca56e36",
  "order_id" => "2014111702",
  "payment_type" => "bri_epay",
  "transaction_time" => "2014-11-17 12:27:40",
  "transaction_status" => "pending",
  "redirect_url" => "https://api.sandbox.veritrans.co.id/v2/redirect/cf0ec2c0-3e52-4f20-94fc-a6bc8ca56e36",
  "gross_amount" => "145000"
);


Two important information details in the JSON response are the status_code and redirect_url. If the value of status_code is 201, the charge transaction process is successfully saved in Veritrans server with pending status. That being so, you have to redirect your customer to redirect_url to finish the e-Pay BRI transaction. You can add the following PHP code at the last line of the file checkout.php to do so: 

<?php
include_once('Veritrans.php');

...

$result = Veritrans_VtDirect::charge($transaction_data);

if($result->status_code == "201")
{
  //success
  echo "Redirect e-Pay BRI. <br />";
  var_dump($result);
  header('Location: ' . $result->redirect_url);
} else {
  //error
  echo "There is an error on the transaction.<br />";
  echo "<h3>Result:</h3>";
  var_dump($result);
}
?>


Information that is related to transaction value, order ID and transaction information will be displayed on the redirected URL page. In the sandbox environment, the e-Pay BRI payment page is a dummy page. Customer simply has to click the payment button to complete the transaction.

When the transaction is completed, customer can proceed back to your website. To do this, you have to fill in the callback URL configuration in MAP (Merchant Administration Portal). Error During Charge Transaction

Under a certain case, charge transaction on VT-Direct may fail. This will prevent you from getting the redirected URL to e-Pay BRI page. Veritrans will return an error message if failure occurrs. The reason of failure can be inspected from the status code displayed on the error message. The status codes that generally occur are:

Error Code Explanation
400 Validation error Format or content of the transaction data is invalid; Example: Wrong variable type, invalid payment method.
401 Access Denied Invalid client or server key.
402 Access Denied Merchant does not have access to utilise a certain method of payment.
406 Duplicate Order ID Order ID has previously been used and paid.
410 Account Inactive Merchant account has been inactive for a long period of time.


The complete list of status code can be found at API Reference - Status Code. If you have any difficulties, please contact us.

When charge transaction process is completed, you can check the transaction status. The steps to examine transaction status will be discussed on the next steps.


Transaction Status

When charge transaction is successful, the transaction status will still be pending. Customer has to complete the transaction in the e-Pay BRI page. When this process is completed, the transaction status will be changed into success. You can check the transaction status change through MAP.
