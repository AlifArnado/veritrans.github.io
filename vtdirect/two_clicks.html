Other Feature

Currently Veritrans has other features for credit card, namely :

  1. Pre-Authorization
  2. Installment
  3. One Click
  4. Two Click

Here is a tutorial for doing integration using the features to suit your needs.

Two Clicks

Obtaining 3DSecure Token

Same as One Click, 3D Secure token must be activated to use Two Clicks feature. See Activating 3DSecure to request parameter that required for 3DSecure token.

First Charge Request

First charge request for Two Clicks have exactly same format with the first charge on One Click. On first charge request, "save_token_id": true must be added on JSON request in "credit_card" part. So, the token will be saved for future payments. Sample in PHP:

// Data yang akan dikirim untuk request charge transaction dengan credit card.
$transaction_data = array(
  'payment_type'      => 'credit_card', 
  'credit_card'       => array(
    'save_token_id'   => true,
    'token_id'        => $token_id,
    'bank'            => 'cimb'
  ),
  'transaction_details'   => $transaction_details,
  'item_details'          => $items,
  'customer_details'      => $customer_details
);

Response will be returned if transaction succeed as follows:

{
  "transaction_id": "ca5e0ff5-4297-48ef-aa25-b3ab018d4ead",
  "order_id": "order52",
  "gross_amount": "200000",
  "payment_type": "credit_card",
  "transaction_time": "2014-03-27 17:20:53",
  "transaction_status": "capture",
  "fraud_status": "accept",
  "masked_card": "401111-1112",
  "saved_token_id": "4011117d16c884-2cc7-4624-b0a8-10273b7f6cc8",
  "saved_token_id_expired_at": "2024-03-27 17:20:53",
  "status_code": "200",
  "status_message": "OK, Credit Card 3D Secure transaction is successful"
}

From response above, "saved_token_id" result needs to be stored by the merchant, for example in the database, so it can be used for future payments.

Next Charge Request (Normal)

For next charge request, ask the customer to enter the credit card CVV and get "token_id" which is derived from the CVV and saved_token_id.

<input id="token_id" name="token_id" type="hidden">

To get "token_id", the merchant must add parameter "two_click": true on the function card used in the example script below. Then the credit card details sent to our endpoint together with the client-key merchant on Veritrans.client_key . After receiving the response, field "token_id" was able to be filled with values ​​in response.token_id. Example javascript code to get "token_id"</ code> is as below.

<script type="text/javascript">
  $(function(){
   //endpoint
   Veritrans.url = "https://api.sandbox.veritrans.co.id/v2/token";
   //client_key
   Veritrans.client_key = "bb558bb6-371a-415b-b7ee-bf285f33868f";
   var card = function(){
   return {
       'card_cvv'   : $(".card-cvv").val(),
       'token_id'   : $(".saved_token_id").val(), // from database
       'two_click'  : true
     }
   };

   // handler when user click the 'Pay' button.
   $('.submit-button').click(function(event){
     event.preventDefault();
     $(this).attr("disabled", "disabled");
     Veritrans.token(card, callback);
     return false;
   });

   function callback(response) {
     if (response.status_code == '200') {
      // success
       $("#token_id").val(response.token_id);
       $("#payment-form").submit();
     } else {
       // failed request token
       $('.submit-button').removeAttr('disabled');
       // Show status message.
       $('#message').text(response.status_message);
       console.log(JSON.stringify(response));
     }
   }
  });
</script>

Next Charge Request (3D Secure)

For next charge request using 3D Secure, ask the customer to enter the credit card CVV and get "token_id" which is derived from the CVV and saved_token_id.

<input id="token_id" name="token_id" type="hidden">

To get "token_id", the merchant must add parameter "two_click": true , "secure": true, "gross_amount" on the function card used in the example script below. Then the credit card details sent to our endpoint together with the client-key merchant on Veritrans.client_key . After receiving the response, field "redirect_URL" was able to be filled with values ​​in response.redirect_url will open 3D Secure dialogue.

After 3D Secure process done, use field "token_id" was able to be filled with values ​​in response.token_id. Example javascript code to get "token_id" and "redirect_url" is as below.

<script type="text/javascript">
  $(function(){
   //endpoint
   Veritrans.url = "https://api.sandbox.veritrans.co.id/v2/token";
   //client_key
   Veritrans.client_key = "bb558bb6-371a-415b-b7ee-bf285f33868f";
   var card = function(){
   return {
       'card_cvv'   : $(".card-cvv").val(),
       'token_id'   : $(".saved_token_id").val(), // from database
       'two_click'  : true,
       'secure'     : true,
       'bank'       : $(".bank").val(), //optional (bni | mandiri | cimb)
       'gross_amount' : $(".gross-amount").val()
     }
   };

   // handler when user click the 'Pay' button.
   $('.submit-button').click(function(event){
     event.preventDefault();
     $(this).attr("disabled", "disabled");
     Veritrans.token(card, callback);
     return false;
   });

   function callback(response) {
      if (response.redirect_url) {
            // 3Dsecure transaction. Open 3Dsecure dialog
            console.log('Open Dialog 3Dsecure');
            openDialog(response.redirect_url);

      } else if (response.status_code == '200') {
            // success 3d secure or success normal
             //close 3d secure dialog if any
      closeDialog();

            // store token data in input #token_id and then submit form to merchant server
            $("#token_id").val(response.token_id);
            $("#payment-form").submit();
      } else {
            // failed request token
            //close 3d secure dialog if any
            closeDialog();
            $('.submit-button').removeAttr('disabled');
            // Show status message.
            $('#message').text(response.status_message);
            console.log(JSON.stringify(response));
    }
  }

  // Open 3DSecure dialog box
  function openDialog(url) {
      $.fancybox.open({
          href: url,
          type: 'iframe',
          autoSize: false,
          width: 700,
          height: 500,
          closeBtn: false,
          modal: true
      });
  }

  // Close 3DSecure dialog box
  function closeDialog() {
        $.fancybox.close();
  }
});

</script>
