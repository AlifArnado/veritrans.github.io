Getting Started

Authentication

Authentication is required to utilize the Payment API. We utilize BASIC AUTH http, whereby merchant is required to send us the server_key.

Server Key

Veritrans provides a server_key and client_key to allow merchant to communicate with the Payment API. Merchants can find the server_key in the MAP (Merchant Administration Portal).

  • server_key, is a private key that needs to be kept secret by the merchant. The Server Key is utilized by the merchant to send a transaction request to Veritrans.

Basic Auth HTTP Header

The header for authorization can be generated by obtaining the Base 64 of the Server Key. This is an implementation of Basic Authentication. http://en.wikipedia.org/wiki/Basic_access_authentication

Basic authentication must use the following format:

username:password

Since there is no password for the Server Key, we will leave the password blank. Therefore, if our Server Key is

94960ece-9513-4265-9cf2-67a4da330213

it will become:

94960ece-9513-4265-9cf2-67a4da330213:

(Note: make sure that there is a colon “:” after the Server Key)

Merchant then needs to convert it into Base 64 to get:

OTQ5NjBlY2UtOTUxMy00MjY1LTljZjItNjdhNGRhMzMwMjEzOg==

Therefore, the header Authorization will be:

Authorization: Basic OTQ5NjBlY2UtOTUxMy00MjY1LTljZjItNjdhNGRhMzMwMjEzOg==

Make sure the merchant put the string "Basic " before BASE 64 server_key.

URL Endpoint

URL endpoint is the URL address that will receive request charge transaction. You will need the URL endpoint from step-3 onwards. Veritrans provides separate URL endpoint for testing and production environments:

  • Testing / Development / Sandbox : https://api.sandbox.veritrans.co.id
  • Production : https://api.veritrans.co.id

During development phase, please use https://api.sandbox.veritrans.co.id. Once you are ready to “go live”, the merchant is required to change the URL endpoint to that for production.

HTTP Header

Given that the Payment API uses JSON to communicate with the merchant, there are several header that needs to be sent by the merchant for every transaction with the Payment API.

Below is the required header for every http request sent to the Payment API.

Header Value
Accept application/json
Content-Type application/json
Authorization Basic base64(server_key:)

Configuring Finish, Unfinish, and Error Redirect URL

After completing the payment process, customer will be redirected back to merchant’s website. There are 3 URL addresses that need to be configured through MAP under Settings » Configuration:

  • Finish Redirect URL

    URL address to where customer will be redirected when payment succeed.

  • Error Redirect URL

    URL address to where customer will be redirected when payment failed.

  • Unfinish Redirect URL

    URL address to where customer will be redirected when clicking “Back to Merchant Website” button (canceling payment process).

MAP page for configuring Finish, unfinish, and error redirect URLs.

Page after payment succeed where customer will be redirected according to Finish Redirect URL

When redirected to the merchant website, the redirect URL will take three parameters, order_id, status_code, and transaction_status. Status_code parameter is the status of the transaction shortly after the buyer make a payment on VT-web page. Details status_code can be viewed in the documentation section Payment API Status Code.
Example: http://sneeeekers.com/payment/finish?order_id=o1234&transaction_status=pending&status_code=201
