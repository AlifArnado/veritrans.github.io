VT-Web Introduction

VT-Web is an easy way to add payment feature for your online store. When using VT-Web, customer will be redirected from your online store to Veritrans payment page to complete payment. Veritrans payment page will help customer paying their order with many payment methods such as credit card, Mandiri Clickpay, and CIMB Clicks in a secure environment and at the same time will minimalized integration work with Veritrans.

Using VT-Web

To use VT-Web, merchants need to send the following request to the charge endpoint to conduct a transaction using VT-Web:

Description Proceed the transaction
Endpoint sandbox : https://api.sandbox.veritrans.co.id/v2/charge
  Production : https://api.veritrans.co.id/v2/charge
HTTP Method POST
HTTP Header Accept: application/json
  Content-Type: application/json
  Authentication: Basic base64(server_key:)

Example of Body Request (JSON) :

{
    "payment_type": "vtweb",
    "vtweb": {
        "enabled_payments": [
            "credit_card"
        ]
    },
    "transaction_details": {
        "order_id": "order45",
        "gross_amount": 200000
    },

When sending a request charge with payment_type: "vtweb", Veritrans will send back a response of JSON that consists of three parameters, which are: redirect_url, status_code dan status_message. merchant needs to direct customers to URL address that is included in parameter redirect_url, That specific URL address will then lead to Veritrans’ payment page of VT-Web.

When it is back on address of redirect_url, shoppers will see Veritrans payment page of VT-Web as below :

Redirect URL dan Notifikasi

Redirect URL

When shoppers are on Veritrans payment page of VT-Web, there will be three possibilities :

  1. Transaction is successful. After successfully paying, shoppers will be brought back to merchant site. Spesific URL address where shoppers are directed to after successful transaction is called Finish Redirect URL.

  2. Transaction is failed. If a transaction is failed (for example, credit card has reached its limit), shoppers will be brought back to merchant's site. Spesific URL address where shoppers are directed to after a failed transaction is called Error Redirect URL.

  3. Transaction is cancelled. If shoppers changes mind and cancel the transaction (by clicking on link "Back to Merchant Website"), shoppers will be brought back to merchant's site. Spesific URL address where shoppers are directed to after a cancelled transaction is called Unfinish Redirect URL.

Notification

After shoppers completed the payment process, either success or failure (for example, credit card has reached its limit), Veritrans will send 2 types of notification:

  1. Email notification. Email notification is sent to shopper as well as merchant.

  2. HTTP(S) POST notification. HTTP(S) POST notification is sent by Veritrans to merchant’s server. Merchant can use HTTP(S) POST notification to update payment status from a transaction/order in real time. This can be done with a notification from Veritrans right after shoppers complete the payment process.

All three URL addresses (Finish Redirect URL, Error Redirect URL, dan Unfinish Redirect URL), can be set by merchant in Merchant Administration Portal (MAP) in Settings - Configuration pada MAP menu.
