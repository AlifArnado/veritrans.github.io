Other Features

BIN Promo

Introduction

The BIN promo feature enables a merchant to offer promotions based on the customer's credit card BIN (Bank Identification Number). BIN is the first six digits in the credit card number that helps to identify which particular bank issues the card. This feature allows a merchant to cooperate with a particular bank for any marketing offers/programs.

How It Works

The best way to illustrate how this feature works is through a simple example. A merchant wishes to make a marketing agreement with BCA with the merchant giving discount to anyone using BCA Visa cards. When a customer wishes to pay with BCA Visa card, he will be redirected to VT-Web page with discounted total price. At this stage, only BCA Visa cards be the only cards that can finish the transaction and any other types of cards will be rejected for payment. This ensures that only the proper cardholders can get the benefit of the promotion.

Technical Implementation

BIN promo works by the whitelist method. When the merchant's server sends a customer's transaction with the discounted price to Veritrans, it also sends along a list of credit card BIN numbers that are allowed to make the transaction. Veritrans will check if the credit card number used by the customer is in the list of allowed BIN. If not, the customer will not be able to finish the transaction.

The promo_bins parameter can be set during the Get Token step of VT-Web. See Integrating VT-Web for further details.

PHP
<?php
//...
$veritrans->promo_bins = array("400000","411111");
//...
?>

3D Secure & Verified by Visa

3-D Secure is a specification developed by Visa to improve transaction performance online and to accelerate the growth of electronic commerce. Verified by Visa is an authentication program based on the 3-D Secure Specification.

3D Secure

The 3-D Secure protocol is a technical platform that includes technical specifications and requirements for issuers, acquirers, and merchants. In addition to utilizing the widely supported Internet technology Secure Sockets Layer (SSL) encryption to protect payment card information during transmission over the Internet, 3-D Secure uses cardholder authentication to verify the parties involved in the transaction.

Verified by Visa

Verified by Visa enables real-time cardholder authentication by participating card issuers. With authentication, an issuer ensures that the presenter of the card is authorized and entitled to use the card. Online purchases are authenticated when the cardholder correctly enters the information requested by the issuer via the issuer’s Access Control Server (ACS). Cardholders may activate their card a number of ways. If a U.S. issuer or cardholder has elected not to participate, Visa ensures that participating merchants receive “attempted authentication” responses.

Benefits

Verified by Visa has been designed to benefit all participants in an online payment transaction. A summary of the benefits is shown below.

Benefits for Cardholders:

  • Increased consumer confidence for purchases via the Internet.
  • No special application software is needed at the cardholder’s access device.
  • Ease of use.
  • Control over card use for online purchases.

Benefits for Merchants:

  • Increased consumer confidence in online purchasing, potentially leading to increased sales volume.
  • Reduced risk of fraudulent transactions.
  • Reduced operational expense due to fewer transaction disputes.

Benefits for Issuers:

  • Reduces fraudulent transactions.
  • Increased member online brand visibility because the issuer is involved in each transaction. This adds value and strengthens the issuer’s relationship with the cardholder.
  • Enables issuers to verify authentication data during Authorization Request processing.

Benefits for Acquirers:

  • Reduces fraudulent transactions.
  • Reduces operational expense by decreasing the volume of chargebacks.
  • Improved value to merchants through increased sales opportunities and decreased transaction disputes.

Technical Implementation

Before your shop can use 3D Secure, you have to check with our sales representative if your 3D Secure MID is ready. Once it's ready, you can use it by enabling the 3D Secure flag parameter. This 3D Secure flag parameter can be set during the Get Token step of VT-Web. See Integrating VT-Web for further details.

PHP
<?php
//...
$veritrans->enable_3d_secure = '1';
//...
?>

Installment Transaction

To accept installment transactions, merchants are required to sign a separate agreement with the acquiring bank. To conduct an installment transactions with VT-web, you need to add installment_banks parameter to choose which banks are allowed, and installment_terms parameter to specify monthly installment available.

PHP
<?php
//...
$veritrans->installment_banks 	= ["bni", "cimb"];

/* This will be the default parameters if you don't send any installment_terms parameter
$veritrans->installment_terms = array(
	"bni" => ["3", "6", "12"],
	"cimb"=> ["3", "6", "12"]
);
*/

$veritrans->installment_terms = array(
	"bni"    => ["4", "7", "11"],
	"cimb"   => ["6"]
);

//...
?>

Bank Points

To accept customer transactions using their credit card points, merchants are required to sign a separate agreement with the acquiring bank. To conduct this kind of transaction, you need to add point_banks parameter to choose which banks are allowed. The customer will choose how many points to use for the transaction in our VT-Web payment page.

PHP
<?php
//...
$veritrans->point_banks = ['bni', 'cimb'];
//...
?>

Acquiring Bank

If a merchant have multiple bank accounts registered to Veritrans, the merchant can choose which acquiring banks to use for any particular transaction. To choose which bank account to use in a transaction, you need to add bank parameter.

PHP
<?php
//...
$veritrans->bank = 'bni';
//...
?>
