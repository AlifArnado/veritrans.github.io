Other Features

The tutorial below will assist you in implementing other features with VT-Web. If further assistance is required, please contact us at support@veritrans.co.id or visit our support page.

Currently, VT-Web provides a few additional feature:

  1. Choose The Method of Payment
  2. Inactivate 3D Secure
  3. Installment
  4. BIN Promo

3D Secure, installment, and BIN Promo feature are exclusively available for the credit card payment method. The following tabs provide tutorials to integrate with these feature.

  • Choose The Method of Payment
  • Inactivate 3D Secure
  • Installment
  • Bin Promo

Currently, there are four methods of payment that can be utilised from VT-Web:

  1. Credit Card
  2. Mandiri ClickPay
  3. CIMB Clicks
  4. Virtual Account
  5. e-Pay BRI

On section VT-Web – Integration, we explained on how to automatically activate all available payments for the merchant. This can be implemented by not including the enabled_payments parameter. However, under a certain condition, you need to display only some of the payment methods available. If such circumstances occurr, please follow the steps below.

Credit Card

Following the test transaction on the integration section, the sample PHP code that we can make use of is as followed:

<?php
include_once('Veritrans.php');

...

// Fill transaction details
$transaction = array(
    "vtweb" => array (
          "enabled_payments" => array("credit_card"),
          "credit_card_3d_secure" => true,
        ),
    'transaction_details' => array(
        'order_id' => rand(),
        'gross_amount' => 10000
        )
    );

...

?>

Mandiri ClickPay

Mandiri ClickPay is one of the payment features from Mandiri Debit. This method allows customer to perform a transaction by inserting the 16 digits Mandiri Debit Card and PIN Token for transaction validation.

To activate Mandiri ClickPay method of payment (automatically active on Sandbox mode), merchant needs to:

  1. Sign on for Mandiri ClickPay method of payment by contacting Veritrans sales representative.
  2. Add the mandiri_clickpay parameter on variable enabled_payments.

Following the test transaction on integration, the sample PHP code that we can make use of is:

<?php
include_once('Veritrans.php');

...

// Fill transaction details
$transaction = array(
    "vtweb" => array (
          "enabled_payments" => array("mandiri_clickpay")
        ),
    'transaction_details' => array(
        'order_id' => rand(),
        'gross_amount' => 10000
        )
    );

...

?>


The Mandiri ClickPay tab will be shown at the VT-Web payment page. Information and procedure to perform Mandiri ClickPay transaction is displayed on the figure below.

Inactivate 3D Secure

It is highly advisable to activate 3D Secure feature on your credit card transaction. Implementing 3D Secure is a standard security practice for a credit card transaction. 3D Secure improves credit card transaction security by adding an authentication process between the customer and their issuing bank. Under certain cases where merchant unfortunately needs to deactivate the 3D Secure feature, you just have to add the parameter "credit_card_3d_secure" => false during redirect VT-Web URL request.

Following the test transaction on integration, the sample PHP code that we can make use of is:

<?php
include_once('Veritrans.php');

... 

$transaction = array(
    "vtweb" => array (
          "enabled_payments" => array("credit_card"),
          // use false to disable 3ds
          "credit_card_3d_secure" => false
        ),
    'transaction_details' => array(
        'order_id' => 7145,
        'gross_amount' => 10000
        )
    );

...

?>

Installment

To activate installment feature, merchant has to make an agreement with the acquiring bank. The merchant also needs to contact a Veritrans sales representative if the MID for instalment is ready. If MID is ready, merchant simply has to add the parameter installment during redirect VT-Web URL request.

Following the test transaction on integration, the sample PHP code that we can make use of is:

<?php
include_once('Veritrans.php');

...

$transaction = array(
    "vtweb" => array (
          "enabled_payments" => array("credit_card"),
          "credit_card_3d_secure" => true,
          // Add installment payment option
          "payment_options" => array(
              "installment" => array(
                  "required" => true,
                  "installment_terms" => array(
                      "bni" => array(6,12),
                      "mandiri" => array(6)
                    )
                )
            )
        ),
    'transaction_details' => array(
        'order_id' => rand(),
        'gross_amount' => 10000
        )
    );

...

?>


The instalment feature on VT-Web consists of a few parameters. The description of the parameters that could be used are:

Attribute Data Type Required Description
required Boolean Yes Status if transaction is required to utilise instalment feature
installment_terms Array Object Yes List of bank and the term of instalment


Description of every object under installment_terms:

Attribute Data Type Required Description
bank String(50) Yes Bank that provides instalment. Currently, the valid value is bni or mandiri
terms Array Integer Yes The term of instalment. Currently, the valid value is 6 atau 12

Note: You need at least one bank and one installment term.

BIN Promo

BIN (Bank Identification Number) is the first 1-8 digit of a credit card, which identifies the bank that issues the card. A bank generally has more than one BIN. BIN promo is a feature that allows merchant to advertise bank a promotion according to the credit card issued by the bank.

To use this feature, merchant needs to accumulate the list of BIN that currently receives promotion. This list of BIN will then become a transaction parameter credit_card_bins. This transaction can only be performed exclusively by the credit card that is included in the BIN list.

Following the test transaction on integration, the sample PHP code that we can make use of is:

<?php
include_once('Veritrans.php');

...

$transaction = array(
    "vtweb" => array (
          "enabled_payments" => array("credit_card"),
          "credit_card_3d_secure" => true,
          //Register BIN list
          "credit_card_bins" => array("48111111", "3111", "5")
        ),
    'transaction_details' => array(
        'order_id' => 8033,
        'gross_amount' => 10000
        )
    );

$vtweb_url = Veritrans_Vtweb::getRedirectionUrl($transaction);

...

?>
