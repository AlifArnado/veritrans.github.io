Integrating VT-Web

Three Quick Steps

  1. Get Tokens
  2. Create Redirection Form
  3. Handling Response
  • Step 1Get Tokens
  • Step 2Create Redirection Form
  • Step 3Handling Response

Get Tokens

This is the first step of the payment process using VT-Web. In this step, you will get two tokens from VT-Web:

  • token_merchant: Used to verify payment notification.
  • token_browser: Used for redirection parameter.

To get the tokens, there are several mandatory parameters that you would need to send. See API Parameters Page for the required parameters and further details.

Note: If your system/database already contains your customers' information (phone, email, address, etc), you can pass those parameters as well so customers don't need to refill those details at the Veritrans payment page again.

  • PHP
  • Ruby
<?php
$veritrans = new Veritrans();

$veritrans->merchant_id         = 'T100000000000001000001';
$veritrans->merchant_hash_key   = '305e0328a366cbce8e17a385435bb7eb3f0cbcfbfc0f1c3ef56b658';

//TODO: Change with your actual order_id.
$veritrans->order_id            = 'your_unique_order_id';

// Set commodity items
$items = array(
            array(
                "item_id" => 'itemsatu',
                "price" => 250000,

                "item_id" => 'itemdua',

$veritrans->items = $items;

$veritrans->billing_address_different_with_shipping_address = 1; // Set 0 if shipping address = billing address

$veritrans->first_name  = "Andri";

    'bni' => [3,12],

Creating the Form for Redirection

The next step in the process is to redirect the customer from the checkout page to VT-Web's payment page. This action will require you to create a small form with Javascript auto submit. Please see the sample code below to get a clearer picture of the implementation.

Handling Response

Once the payment process is complete, VT-Web will send email notifications to both the merchant admin and the customer. VT-Web will also send a HTTP(S) POST request as notification to the merchant server. Please set the target URL for the HTTP(S) POST request notification and the email notification at VT-Web Configuration Page.

SECURITY TIP: The merchant needs to verify every notification request. The parameters that will be verified are orderId and token_merchant. Please make sure that both parameters match with the ones inside your merchant database.

Assume payment_notification is the action to handle the HTTP(S) POST request notification from Veritrans server.

def payment_notification
	#change with your actual merchant_id and merchant_hash_key
	merchant_id       = 'T100000000000001001862'
	merchant_hash_key = '21565170a5a85543b98b0e34071656c104f31266ff436be07edb2d5a33983f97'

	param = {:orderId => params[:orderId], :mStatus => params[:mStatus], :TOKEN_MERCHANT => params[:TOKEN_MERCHANT]}
	@p = param.to_json
	@notification = ActiveMerchant::Billing::Integrations::Veritrans::Notification.new @p, merchant_id: merchant_id, merchant_hash_key: merchant_hash_key

	if @notification.acknowledge
		@order = Order.find(params[:orderId])
		if @order
		  @order.paid!
		else
		  return false
		end
	else
		return false
	end

end
