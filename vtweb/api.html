API Parameters

Veritrans VT-Web API currently consists of request tokens and redirection to payment page. Please see the detail constraints of each API parameter and examples of success or error response.

Request Token API

URL Endpoint

Resource URL:

https://vtweb.veritrans.co.id/v1/tokens

Type:

JSON

HTTP Method:

POST

Request Token Parameters

Required Field

Required Field Type Length Format Description
version String 1 1 to be able to use params in this documentation, you have to set '1' as the value.
merchant_id String 22 A-Z0-9 Merchant identifier
merchanthash String 128 SHA-512 hex Generated SHA-512 hash of merchant_id, merchant_hash_key, and order_id. You can see a sample code to generate the merchanthash here.
order_id String 30 a-zA-Z0-9 Order Identifier
billing_different_with_shipping String 1 1 or 0 1 = different with shipping; 0 = same with shipping
required_shipping_address String 1 0 and 1 1 = required; 0 = not required
repeat_line String 12 0-9 Total item in a cart
item_id Array 12 a-zA-Z0-9 Item Identifier
price Array 9 > 0 and < 100000000 IDR. Price of a commodity.
quantity Array 4 > 0 and < 1000 Quantity of a commodity
item_name1 Array 20 a-zA-Z0-9 -_',.@&+/ Name of commodity (English Name)
item_name2 Array 20 a-zA-Z0-9 -_',.@&+/ Name of commodity (Indonesian Name)

Required Field if required_shipping_address = 1

if required_shipping_address = 1 Type Length Format Description
shipping_first_name String 20 a-z A-Z Shipping first name
shipping_last_name String 20 a-z A-Z Shipping last name
shipping_address1 String 100 a-zA-Z0-9-_',.@()/ \ Shipping address line 1
shipping_address2 String 100 a-zA-Z0-9-_',.@()/ \ Shipping address line 2
shipping_city String 20 a-zA-Z-_', .@ Shipping city
shipping_country_code String 3 A-Z ISO 3166-1 alpha-3 – Upper Case; example "IDN" for Indonesia
shipping_postal_code String 9 0-9 Shipping postal code
shipping_phone String 19 +0-9 - Requires more than 9 digits can be started with '+'
email String 30 email Email address

Optional (Not Required) Field

Optional Field Input Type Length Format Description
payment_methods Array - a-z A-Z Payment method showed, e.g. ['credit_card', 'mandiri_clickpay']
finish_payment_return_url String 256 URL URL redirection after finishing payment.
unfinish_payment_return_url String 256 URL URL redirection for transaction that never completed
error_payment_return_url String 256 URL URL redirection if there is an error ocurred after payment
first_name String 20 a-z A-Z Billing first name
last_name String 20 a-z A-Z Billing last name
address1 String 100 a-zA-Z0-9 -_',.@()/\ Billing address line 1
address2 String 100 a-zA-Z0-9 -_',.@()/\ Billing address line 2
city String 20 a-zA-Z-_', .@ Billing city
country_code String 3 A-Z ISO 3166-1 alpha-3 – Upper Case; example "IDN" for Indonesia.
postal_code String 9 0-9 Billing postal code
phone String 19 +0-9 - Billing phone number, requires more than 9 digits
promo_bins Array 16 0-9 Promo BIN code, 1 to 16 digits; e.g. ['411111', '5'] more..
enable_3d_secure String 1 1 or 0 Enable 3D Secure, 1 = enabled; 0 = not enabled. more..
point_banks Array - 'bni', 'cimb' Point Banks, e.g. ['bni', 'cimb'] more..
installment_banks Array - 'bni', 'cimb', 'mandiri' Installment Banks, e.g. ['bni', 'cimb', 'mandiri'] more..
installment_terms JSON Array - bank as key and terms as value Installment terms, e.g. {"bni":["3", "6", "12"], "cimb":["3", "6", "12"]} more..
bank String - 'bni', 'cimb' or 'mandiri' Acquiring Bank more..

Response

Type:

JSON

Example of successful response:

{
    "token_merchant": "3a258a55-6729-4eda-aeaa-c19e5a9af022",
    "token_browser": "320b9231-a93b-4fd0-803f-54e4f093937d"
}

Example of error response:

{
    "errors": {
        "shipping_address1": "should be alphanumeric and specified symbol",
        "shipping_country_code": [
            "should be in ISO 3166-1 alpha-3",
            "is the wrong length (should be 3 characters)"
        ],
        "installment_banks": "must be an Array"
    }
}

Redirection API

URL Endpoint

Resource URL:

https://vtweb.veritrans.co.id/v1/payments

Type:

JSON

HTTP Method:

POST

Redirection Parameters

Field Name Description
merchant_id Your Merchant ID.
order_id Unique and generated by Merchant. Order ID needs to be the same with the previous Order ID used.
token_browser Use the value of token_browser you got from the request key process.

Response

None – Customer will see the Veritrans payment page directly.

Payment Notification

After successful payment process, Veritrans will send the notification email to Merchant and Customer. Veritrans will send the HTTP POST request notification to Merchant Server as well. The target URL can be configured as the Merchant desires.

To set the target URL to where message notification will be sent to, you can contact us for the further support.

Sample of Payment notification parameters

Parameters Sample value Description
orderId QBOHHJ  
mStatus success Status from transaction : success, failure, challenge, or fatal.
TOKEN_MERCHANT af27bd7b-c4c1-45bd-b730-a46de616dbe5  

mStatus parameter is related to the Veritrans Fraud Detection System. mStatus represents the fraud detection system's response.

mStatus Fraud Detection System's response
success ACCEPT
failure DENY
challenge CHALLENGE
fatal There is an internal system error. Please contact Veritrans Support when this occurs.
