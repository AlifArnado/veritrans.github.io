Obtaining Redirect URL

In this phase, merchant will receive URL to VT-Web payment page. To obtain the URL, merchant needs to send request to Veritrans with the following specifications:

URL endpoint /v2/charge
Header Content-Type: application/json
Accept: application/json
HTTP Method POST
Authorization Basic (server_key:)

Credit card transaction using VT-WEB

Messages that will be send during request contain transaction details in JSON format. Example:

{
    "payment_type": "vtweb",
    "vtweb": {
        "enabled_payments": ["credit_card"],
        "credit_card_3d_secure" : "true"
    },
    "transaction_details": {
        "order_id": "order45",
        "gross_amount": 200000
    },
    "item_details": [{
        "id": "item1",
        "price": 100000,
        "quantity": 1,
        "name": "Adidas f50"
    }, {
        "id": "item2",
        "price": 50000,
        "quantity": 2,
        "name": "Nike N90"
    }],
    "customer_details": {
        "first_name": "Andri",
        "last_name": "Setiawan",
        "email": "andri.setiawan@veritrans.co.id",
        "phone": "08195010132",
        "billing_address": {
            "first_name": "Andri",
            "last_name": "Setiawan",
            "address": "Karet",
            "city": "Jakarta",
            "postal_code": "523112",
            "phone": "08195010132",
            "country_code": "IDN"
        },
        "shipping_address": {
            "first_name": "John",
            "last_name": "Watson",
            "address": "Bakerstreet 221B",
            "city": "London",
            "postal_code": "12354",
            "phone": "081323123123",
            "country_code": "IDN"
        }
    }
}

Code above is the default JSON format to make transactions on the Payment API.

Object Description JSON Request:

JSON Attribute Data Type Required Description
payment_type String Yes * VT-Direct: Fill “CREDIT_CARD” for payment with VT-Direct.
* VT-Web: Fill “VTWEB” for payment with VT-Web
transaction_details JSON Object Yes Transaction data detail that will be paid by customer
item_details JSON Array Yes Shopping item detail that will be paid by customer
customer_details JSON Object Yes Customer data detail that make payment

Object Description transaction_details:

JSON Attribute Data Type Required Description
order_id String(50) Yes Customer Payment ID
gross_amount Long Yes Total of transaction price that will be paid by customer

Object Description item_details:

JSON Attribute Data Type Required Description
id String(50) Yes Item ID
price Long Yes Item price
quantity Integer Yes Total Item
name String(50) Yes Item name

Object Description customer_details:

JSON Attribute Data Type Required Description
first_name String(20) Yes Customer first name
last_name String(20) Recommended Customer last name
email String(45) Yes Customer email.
Note: there is a validation to check valid email
phone String(19) Yes Customer phone number.
Note: Minimum length 5, can only accept numeric and symbols +, -, (space), and (). For the + symbol itself can only be placed in the front.
billing_address JSON Object Recommended Billing address customer
shipping_address JSON Object Recommended Shipping address customer

Object Description billing_address:

JSON Attribute Data Type Required Description
first_name String(20) Recommended First name billing address.
Note: No need to be filled if same as the customer details - first_name
last_name String(20) Recommended Last name of billing address
address String(200) Recommended Address of billing address
city String(20) Recommended Billing address city
postal_code String(10) Recommended Billing address postal code
Note: format alpha-numeric, can accept “-” and space
phone String(19) Recommended Billing address phone number.
Note : No need to be filled if same as the customer details - phone. Minimum length 5, can only accept numeric and symbols +, -, (space), and (). For the + symbol itself can only be placed in the front.
country_code String(10) Recommended Country ID(example: IDN)

Object Description shipping_address:

JSON Attribute Data Type Required Description
first_name String(20) Recommended First name shipping address
last_name String(20) Recommended Last name shipping address
address String(200) Recommended Address of shipping address
city String(20) Recommended Shipping address city
postal_code String(10) Optional, except for payment type using credit card, this attribute is mandatory Shipping address postal code
Note: format alpha-numeric, can accept “-” and space
phone String(19) Recommended Billing address phone number.
Note: No need to be filled if same as the customer details - phone. Minimum length 5, can only accept numeric and symbols +, -, (space), and (). For the + symbol itself can only be placed in the front.
country_code String(10) Recommended Country ID (contoh: IDN)

Below is a PHP example to obtain URL redirect to VT-Web payment page:

<?php
// TODO: change with your actual server_key that can be found on Merchant Administration Portal (MAP)
$server_key = "1a794e3f-f023-4cda-92b7-dcb7a22a46ff";

// TODO : change to production URL for your production Environment
// sandbox/development/testing environment:
$endpoint = "https://api.sandbox.veritrans.co.id/v2/charge";
// production environment:
//$endpoint = "https://api.sandbox.veritrans.co.id/v2/charge";


$transaction_details = array(
    'order_id'      => uniqid(),
    'gross_amount'  => 200000
);

// Populate items
$items = [
    array(
        'id'        => 'item1',
        'price'     => 100000,
        'quantity'  => 1,
        'name'      => 'Adidas f50'
    ),
    array(
        'id'        => 'item2',
        'price'     => 50000,
        'quantity'  => 2,
        'name'      => 'Nike N90'
    )
];

// Populate customer's billing address
$billing_address = array(
    'first_name'    => "Andri",
    'last_name'     => "Setiawan",
    'address'       => "Karet Belakang 15A, Setiabudi.",
    'city'          => "Jakarta",
    'postal_code'   => "51161",
    'phone'         => "081322311801",
    'country_code'  => 'IDN'
    );

// Populate customer's shipping address
$shipping_address = array(
    'first_name'    => "John",
    'last_name'     => "Watson",
    'address'       => "Bakerstreet 221B.",
    'city'          => "Jakarta",
    'postal_code'   => "51162",
    'phone'         => "081322311801",
    'country_code'  => 'IDN'
    );

// Populate customer's Info
$customer_details = array(
    'first_name'    => "Andri",
    'last_name'     => "Setiawan",
    'email'         => "andrisetiawan@me.com",
    'phone'         => "081322311801",
    'billing_address'  => $billing_address,
    'shipping_address' => $shipping_address
    );

// Data will send to request redirect_url.
// Uncomment 'credit_card_3d_secure' => true if you want using 3D Secure process.

$transaction_data = array(
    'payment_type'          => 'vtweb', 
    'vtweb'             => array(
        'enabled_payments'  => ['credit_card']
        //‘credit_card_3d_secure' => true,
    ),
    'transaction_details'   => $transaction_details,
    'item_details'          => $items,
    'customer_details'      => $customer_details
);

$json_transaction_data = json_encode($transaction_data);

// Mengirimkan request dengan menggunakan CURL
// HTTP METHOD : POST
// Header:
//  Content-Type : application/json
//  Accept: application/json
//  Basic Auth using server_key
$request = curl_init($endpoint);
curl_setopt($request, CURLOPT_CUSTOMREQUEST, "POST");
curl_setopt($request, CURLOPT_POSTFIELDS, $json_transaction_data);
curl_setopt($request, CURLOPT_RETURNTRANSFER, true);
curl_setopt($request, CURLOPT_SSL_VERIFYPEER, false);
$auth = sprintf('Authorization: Basic %s', base64_encode($server_key.':'));
curl_setopt($request, CURLOPT_HTTPHEADER, array(
    'Content-Type: application/json',
    'Accept: application/json',
    $auth 
    )
);

// Excute request and parse the response
$response = json_decode(curl_exec($request));

// Check Response
if($response->status_code == "201")
{
    //success
    //redirect to vtweb payment page
    header("Location: ".$response->redirect_url);
}
else
{
    //error
    echo "Terjadi kesalahan pada data transaksi yang dikirim.<br />";
    echo "Status message: [".$response->status_code."] ".$response->status_message;

    echo "<h3>Response:</h3>";
    var_dump($response);
}

?>

Success Response

If request succeed, the following response will be returned in JSON format: 

{ 
    "redirect_url": "https://vtweb.sandbox.veritrans.co.id/vtweb/ce72694f-3284-408d-9198-bc0ac269df2d", 
    "status_code": "201",
    "status_message": "OK, success do VTWeb transaction, please go to redirect_url" 
}

Error Response

If there is error during the request, for example using the same order_id again for another request, the following response will be returned: 

{
    "status_code": "406", 
    "status_message": "Merchant charges the same order id more than once or transaction has been paid" 
}

Please refer to Reference-Status Code section for complete description of the status codes.
