Integrasi VT-Direct (CIMB Clicks)

Setiap metode pembayaran memiliki tahapan integrasi yang berbeda. Dokumentasi pada halaman ini bertujuan untuk membantu anda melakukan integrasi VT-Direct dengan metode pembayaran CIMB Clicks. Apabila butuh bantuan, silahkan hubungi kami di support@veritrans.co.id atau kunjungi halaman support kami.

CIMB Clicks adalah salah satu metode pembayaran internet banking yang disediakan oleh Bank CIMB Niaga. Jika metode ini digunakan, pelanggan akan diarahkan ke halaman pembayaran CIMB Clicks untuk melakukan transaksi. Info lebih lengkap dapat diperoleh di website CIMB Niaga.

Untuk mengaktifkan metode pembayaran CIMB Clicks (otomatis aktif pada mode Sandbox), anda perlu menghubungi sales representative Veritrans.

Tahapan Integrasi VT-Direct CIMB Clicks

  1. Membuat Form
  2. Charge Transaksi
  3. Redirect
  4. Status Transaksi

Berbeda dengan metode pembayaran lain (kartu kredit dan Mandiri Clickpay), proses utama pembayaran CIMB Clicks tidak dilakukan di halaman anda. Pelanggan akan di redirect ke halaman CIMB Clicks untuk menyelesaikan transaksi.

Membuat Form

Form pembayaran CIMB Niaga akan kita buat di file form_vtdirect_cimb.php. Karena sebagian besar proses transaksi akan dilakukan di halaman CIMB Clicks, form yang akan kita buat hanya berisi petunjuk pembayaran dan tombol bayar menggunakan CIMB Clicks. Ketika pelanggan anda menekan tombol ini, proses charge transaksi dilakukan pada tahap selanjutnya.

Berikut adalah contoh form pada file form_vtdirect_cimb.php:

<html>
  <head>
    <title>Contoh Form VT-Direct CIMB Clicks</title>
  </head>
<body>
    <h1>Transaksi CIMB Clicks</h1>
    <ol>
      <li>Pastikan anda memiliki User ID CIMB Clicks dan mPIN </li>
      <li>Saldo akan didebet secara otomatis sesuai jumlah pembayaran anda</li>
      <li>Nilai minimum transaksi yang diperbolehkan adalah Rp. 10.000,00</li>
      <li>Anda memiliki waktu 1 jam untuk menyelesaikan transaksi ini sebelum terjadi pembatalan secara otomatis</li>
      <li>Pastikan tidak ada pop-up blocker pada browser anda</li>
    </ol>
    <form action="checkout_cimb.php" method="POST" id="payment-form">
      <button id="submit-button" type="submit">Pay</button>
    </form>
  </body>

Charge Transaksi

Untuk melakukan charge transaction menggunakan CIMB Clicks, beberapa langkah harus dilakukan sebagai berikut:

  • payment_type harus di-set menjadi cimb_clicks.

  • JSON object baru harus ditambahkan, dengan nama cimb_clicks,dan mempunyai nama/value pasangan sebagai berikut:

    • description → keterangan transaksi

Tahap ini membahas cara charge transaksi CIMB Clicks di file checkout_cimb.php. Untuk melakukan charge transaksi, kita dapat menggunakan library Veritrans-PHP. Berikut adalah contoh implementasi charge transaksi pada file checkout_cimb.php:

<?php
include_once('Veritrans.php');

// TODO: change with your actual server_key that can be found on Merchant Administration Portal (MAP)
Veritrans_Config::$serverKey = "< Your Server Key >";
Veritrans_Config::$isProduction = false;

$transaction_details = array(
  'order_id'    => rand(),
  'gross_amount'  => 20000,
);

$customer_details = array(
    'first_name'    => "Andri",
    'last_name'     => "Litani",
    'email'         => "andri@litani.com",
    'phone'         => "081122334455"
    );

// Data yang akan dikirim untuk request charge transaction dengan credit card.
$transaction_data = array(
  'payment_type' => 'cimb_clicks',
  'cimb_clicks' => array(
      'description' => "Contoh Deskripsi",
    ),
  'transaction_details' => $transaction_details,
  'customer_details' => $customer_details
  );

$result = Veritrans_VtDirect::charge($transaction_data);
?>

Pada contoh di atas, kita menggunakan data CIMB Clicks, transaksi, dan pelanggan sebagai parameter charge transaksi. Anda sebenarnya dapat menambahkan data lain saat charge transaksi, seperti: alamat pelanggan, alamat pengiriman, dan informasi barang. Untuk menambahkan data ini, anda dapat melihat Referensi API - Charge Transaksi.

Redirect

Server Veritrans akan mengembalikan respon untuk setiap charge transaksi dalam bentuk JSON. Library Veritrans-PHP akan melakukan konversi JSON menjadi object PHP secara otomatis. Contoh respon JSON yang dikembalikan oleh veritrans adalah sebagai berikut:

$result = array(
  "status_code" => "201",
  "status_message" => "Success, CIMB Clicks transaction is successful",
  "transaction_id" => "ff05337c-6c94-4f70-8e81-35acd89b688e",
  "order_id" => "201404141421",
  "payment_type" => "cimb_clicks",
  "transaction_time" => "2014-04-14 16:03:51",
  "transaction_status" => "pending",
  "redirect_url" => "https://api.sandbox.veritrans.co.id/cimb-clicks/request?id=ff05337c-6c94-4f70-8e81-35acd89b688e",
  "gross_amount" => "20000"
);

Dua informasi penting dalam respon JSON adalah status_code dan redirect_url. Jika nilai status_code adalah 201, maka proses charge transaksi berhasil disimpan pada server Veritrans dengan status pending. Anda selanjutnya harus me-redirect pelanggan anda ke redirect_url untuk menyelesaikan transaksi CIMB Clicks. Untuk melakukan hal tersebut, anda dapat menambahkan kode PHP berikut pada bagian akhir file checkout_cimb.php:

<?php
include_once('Veritrans.php');

...

$result = Veritrans_VtDirect::charge($transaction_data);

if($result->status_code == "201")
{
  //success
  echo "Redirect CIMB Clicks. <br />";
  var_dump($result);
  header('Location: ' . $result->redirect_url);
} else {
  //error
  echo "Terjadi kesalahan pada data transaksi yang dikirim.<br />";
  echo "<h3>Result:</h3>";
  var_dump($result);
}
?>

Informasi terkait nilai transaksi, order ID, dan keterangan transaksi akan ditampilkan pada halaman redirect URL. Pada environment sandbox, halaman pembayaran CIMB Clicks bersifat dummy. Pelanggan anda cukup menekan tombol bayar untuk menyelesaikan proses transaksi.

Ketika transaksi selesai, pelanggan anda dapat kembali ke website anda. Untuk melakukan hal ini, anda perlu mengisi konfigurasi callback URL di MAP (Merchant Administration Portal), silahkan melihat menu langkah awal untuk informasi lebih lengkap. Kesalahan dalam Charge Transaksi

Pada beberapa kasus, charge transaksi bisa gagal. Hal ini menyebabkan anda tidak mendapatkan redirect url ke halaman CIMB Clicks. Veritrans akan mengembalikan pesan error ketika terjadi kegagalan, penyebab gagalnya charge transaksi dapat dilihat dari status code yang ada di dalam pesan error. Status code yang umumnya muncul adalah:

Kode Error Keterangan
400 Validation error Format atau konten data transaksi tidak valid. Contoh: Salah tipe variabel, metode pembayaran tidak valid.
401 Access Denied Client atau server key tidak valid.
402 Access Denied Merchant tidak memiliki izin menggunakan metode pembayaran tertentu.
406 Duplicate Order ID Order ID telah digunakan dan dibayar sebelumnya.
410 Account Inactive Akun merchant tidak aktif dalam waktu lama

Daftar status code lengkap dapat dilihat di Referensi API - Status Code. Jika mengalami kesulitan, silahkan hubungi kami.

Jika charge transaksi berhasil, anda dapat mengetahui status transaksi melalui beberapa cara. Cara untuk melihat status transaksi CIMB Clicks akan dibahas pada bagian selanjutnya.

Status Transaksi

Pada saat charge transaksi berhasil, status transaksi tersebut masih pending. Pelanggan anda harus menyelesaikan pembayaran di ATM terdekat atau melalui internet banking. Begitu proses ini selesai, status transaksi akan berubah menjadi success. Anda dapat mengetahui perubahan status transaksi melalui MAP.
