Langkah Awal

Tujuan: Merchant mengetahui apa yang perlu dipersiapkan sebelum melakukan integrasi ke VT-Direct

Ada beberapa hal yang harus dipersiapkan sebelum kita menggunakan VT-Direct:

  1. Membuat akun Sandbox Veritrans.

    Digunakan untuk development. Data dan transaksi yang digunakan pada akun ini bersifat dummy. Pada tahap pengembangan, diharuskan untuk menggunakan akun sandbox. Akun sandbox dapat dibuat di sini.

  2. Mengisi informasi dasar tentang anda di MAP. Dapat ditemukan di menu Settings - General Settings.



  3. Mengunduh library Veritrans-PHP [ Github | Zip ]

  4. Mencatat server key dan client key akun anda. Kedua key tersebut dapat diperoleh dengan

    • Login Sandbox MAP (Merchant Administration Portal).

    • Pilih menu Settings dan Access keys

    • Perhatian!!! Jaga baik-baik kerahasiaan server key anda.

  5. Menangani callback URL

    Tahap terakhir adalah menentukan bagaimana pelanggan anda kembali ke website menggunakan konfigurasi Finish URL, Unfinish URL, dan Error URL. Pada VT-Direct, field terpenting adalah finish URL. Pelanggan anda akan diredirect ke finish URL ketika transaksi berhasil dilakukan. Hal ini terjadi pada transaksi menggunakan metode pembayaran CIMB Clicks.

    Unfinish URL dan error URL adalah field yang digunakan pada produk VT-Web dan tidak relevan untuk VT-Direct. Namun karena semua field harus diisi, gunakan alamat website anda sebagai unfinsih dan error URL. Anda dapat mengatur konfigurasi URL di MAP.



    Keterangan :

    • Callback URL dimasukan di field Finish URL
    • Isi field Unfinish URL dan Error URL dengan URL website anda.

Jika semua poin di atas selesai dilakukan, integrasi VT-Direct dapat dimulai.
