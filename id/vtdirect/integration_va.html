Integrasi VT-Direct (Virtual Account)

Setiap metode pembayaran memiliki tahapan integrasi yang berbeda. Dokumentasi pada halaman ini bertujuan untuk membantu anda melakukan integrasi VT-Direct dengan metode pembayaran Virtual Account. Apabila butuh bantuan, silahkan hubungi kami di support@veritrans.co.id atau kunjungi halaman support kami.

Salah satu metode pembayaran yang di tawarkan oleh Veritrans adalah virtual account, saat ini Veritrans sudah terintegrasi dengan Bank Permata untuk proses virtual account. Dengan menggunakan layanan ini, pelanggan anda akan memiliki pilihan untuk melakukan pembayaran melalui transfer bank ke rekening virtual account Bank Permata.

Untuk mengaktifkan metode pembayaran Virtual Account (otomatis aktif pada mode Sandbox), anda perlu menghubungi sales representative Veritrans.

Tahapan Integrasi VT-Direct Virtual Account

  1. Membuat Form
  2. Charge Transaksi

Berbeda dengan metode pembayaran lain (kartu kredit, Mandiri Clickpay dan CIMB Clicks), proses utama pembayaran Virtual Account tidak dilakukan di halaman anda. Pelanggan melakukan pembayaran melalui transfer di ATM terdekat, Internet Banking dll.

Membuat Form

Form pembayaran Virtual Account akan kita buat di file form_vtdirect_va.html. Karena sebagian besar proses pembayaran tidak dilakukan di halaman merchant, form yang akan kita buat hanya berisi petunjuk pembayaran dan tombol bayar menggunakan Virtual Account. Ketika pelanggan anda menekan tombol ini, proses charge transaksi dilakukan pada tahap selanjutnya.

Berikut adalah contoh form pada file form_vtdirect_va.html:

<head>
	<title>Bank Transfer</title>
</head>
<body>
	<h1>Bank Transfer</h1>
	<b>Saya akan melakukan pembayaran dengan metode bank transfer</b>
	<form action="checkout_va.php" method="POST" id="payment-form">
      <button id="submit-button" type="submit">Pay</button>
    </form>
</body>

Charge Transaksi

Untuk melakukan charge transaction menggunakan Virtual Account, beberapa langkah harus dilakukan sebagai berikut:

  • payment_type harus di-set menjadi bank_transfer.

  • JSON object baru harus ditambahkan, dengan nama bank_transfer,dan mempunyai nama/value pasangan sebagai berikut:

    • bank → keterangan bank

Tahap ini membahas cara charge transaksi Virtual Account di file checkout_va.php. Untuk melakukan charge transaksi, kita dapat menggunakan library Veritrans-PHP. Berikut adalah contoh implementasi charge transaksi pada file checkout_va.php:

<?php
include_once('Veritrans.php');

// TODO: change with your actual server_key that can be found on Merchant Administration Portal (MAP)
Veritrans_Config::$serverKey = "< Your Server Key >";
Veritrans_Config::$isProduction = false;

$transaction_details = array(
  'order_id'    => rand(),
  'gross_amount'  => 145000,
);

$customer_details = array(
    'first_name'    => "Andri",
    'last_name'     => "Litani",
    'email'         => "andri@litani.com",
    'phone'         => "081122334455"
    );

// Data yang akan dikirim untuk request charge transaction dengan credit card.
$transaction_data = array(
  'payment_type' => 'bank_transfer',
  'bank_transfer' => array(
      'bank' => "permata",
    ),
  'transaction_details' => $transaction_details,
  'customer_details' => $customer_details
  );

$result = Veritrans_VtDirect::charge($transaction_data);
?>

Pada contoh di atas, kita menggunakan data Virtual Account, transaksi, dan pelanggan sebagai parameter charge transaksi. Anda sebenarnya dapat menambahkan data lain saat charge transaksi, seperti: alamat pelanggan, alamat pengiriman, dan informasi barang. Untuk menambahkan data ini, anda dapat melihat Referensi API - Charge Transaksi.

Menampilkan Virtual Account

Server Veritrans akan mengembalikan respon untuk setiap charge transaksi dalam bentuk JSON. Library Veritrans-PHP akan melakukan konversi JSON menjadi object PHP secara otomatis. Contoh respon JSON yang dikembalikan oleh veritrans adalah sebagai berikut:

$result = array(
  "status_code" => "201",
  "status_message" => "OK, PERMATA transaction is successful",
  "transaction_id" => "ff05337c-6c94-4f70-8e81-35acd89b688e",
  "order_id" => "201404141421",
  "payment_type" => "bank_transfer",
  "transaction_time" => "2014-04-14 16:03:51",
  "transaction_status" => "pending",
  "permata_va_number => "8778999933221111",
  "gross_amount" => "145000"
);

Dua informasi penting dalam respon JSON diatas adalah status_code dan permata_va_number. Jika nilai status_code adalah 201, maka proses charge transaksi berhasil disimpan pada server Veritrans dengan status pending. Anda selanjutnya harus menampilkan permata_va_number ke pelanggan anda, agar pelanggan anda dapat melakukan pembayaran. Untuk melakukan hal tersebut, anda dapat menambahkan kode PHP berikut pada bagian akhir file checkout_va.php:

<?php
include_once('Veritrans.php');

...

$result = Veritrans_VtDirect::charge($transaction_data);

if($result->status_code == "201")
{
  //success
  echo "<b>Bank Transfer</b> <br /><br />";
  echo "<b>Silahkan transfer ke Bank Permata dengan No rekening:</b> ".$result->permata_va_number;
  echo "<br /><br />";
  echo "1. Anda memiliki waktu 2 jam untuk melakukan pembayaran, sebelum transaksi tersebut dibatalkan <br />";
  echo "2. Jika anda menemui kendala, silahkan hubungi kami di 021-22334456";

} else {
  //error
  echo "Terjadi kesalahan pada data transaksi yang dikirim.<br />";
  echo "<h3>Result:</h3>";
  var_dump($result);
}
?>

Sampai tahap ini, transaksi sudah tersimpan di MAP dengan status pending. Status transaksi akan berubah menjadi sukses saat pelanggan anda selesai melakukan pembayaran.

Untuk memudahkan proses pembayaran, kami menyediakan simulator pembayaran untuk mode sandbox. Masukan nomor account yang didapat sebelumnya untuk cek apakah ada transaksi dengan nomor virtual account tersebut.

Apabila transaksi dengan nomor virtual account tersebut ada, anda dapat melakukan simulasi untuk melakukan pembayaran.

Sukses melakukan pembayaran.

Kesalahan dalam Charge Transaksi

Pada beberapa kasus, charge transaksi bisa gagal. Hal ini menyebabkan anda tidak mendapatkan nomor virtual account. Veritrans akan mengembalikan pesan error ketika terjadi kegagalan, penyebab gagalnya charge transaksi dapat dilihat dari status code yang ada di dalam pesan error. Status code yang umumnya muncul adalah:

Kode Error Keterangan
400 Validation error Format atau konten data transaksi tidak valid. Contoh: Salah tipe variabel, metode pembayaran tidak valid.
401 Access Denied Client atau server key tidak valid.
402 Access Denied Merchant tidak memiliki izin menggunakan metode pembayaran tertentu.
406 Duplicate Order ID Order ID telah digunakan dan dibayar sebelumnya.
410 Account Inactive Akun merchant tidak aktif dalam waktu lama

Daftar status code lengkap dapat dilihat di Referensi API - Status Code. Jika mengalami kesulitan, silahkan hubungi kami.

Jika charge transaksi berhasil, anda dapat mengetahui status transaksi melalui beberapa cara. Cara untuk melihat status transaksi Virtual Account akan dibahas pada bagian selanjutnya.

Status Transaksi

Pada saat charge transaksi berhasil, status transaksi tersebut masih pending. Pelanggan anda harus menyelesaikan transaksi dalam waktu maksimal 2 jam. Begitu proses ini selesai, status transaksi akan berubah menjadi success. Anda dapat mengetahui perubahan status transaksi melalui MAP.
