Fitur-Fitur Lain

BIN Promo

Fitur BIN Promo memungkinkan merchant untuk menawarkan promosi berdasarkan BIN (Bank Identification Number) dari kartu kredit yang dimiliki pelanggan. BIN adalah 6 digit pertama dari nomor kartu kredit untuk mengidentifikasi bank mana yg menerbitkan kartu kredit tersebut. Fitur ini juga memungkinkan merchant untuk bekerja sama dengan bank tertentu untuk suatu program pemasaran.

Contoh sederhana berikut ini bisa menggambarkan bagaimana cara kerja BIN Promo. Merchant bermaksud untuk mengadakan program kerjasama dengan Bank A dengan memberikan diskon pada pelanggan yang menggunakan kartu Bank A. Ketika pelanggan melakukan transaksi pembayaran di halaman VT-Web Veritrans, pelanggan dengan kartu Bank A akan membayar dengan harga diskon. VT-Direct dapat memfilter bahwa hanya pelanggan dengan kartu bernomor BIN bank A saja yang dapat melakukan pembayaran.

Untuk mengimplementasikan BIN Promo, Anda hanya perlu untuk menambahkan parameter bins yang merepresentasikan nomor BIN kartu kredit yang diperbolehkan ke data JSON (Body Content) yang dikirim saat melakukan Charge Request. Lihat langkah ke-3 (Melakukan Charge Transaksi) pada halaman Integrasi untuk melihat informasi detail tentang Charge Request.

Parameter Tambahan:
Parameter Deskripsi Contoh
bins Integer array with numbers of 1-6 digits "4","4321","456789"

3D Secure Transaction

Sebelum Anda dapat mengaktifkan fitur 3D Secure, Anda terlebih dahulu perlu untuk menanyakan sales representative Veritrans untuk memastikan apakah MID 3D Secure Anda sudah siap. Jika sudah siap, Anda dapat mengaktifkan fitur 3D Secure dengan cara menambahkan parameter 3dsecure, 3dsecure_callback_url, dan 3dsecure_notification_url ke data JSON (Body Content) yang dikirim saat melakukan Charge Request. Lihat langkah ke-3 (Melakukan Charge Transaksi) pada halaman Integrasi untuk melihat informasi detail tentang Charge Request.

Parameter Tambahan:
Parameter Deskripsi Contoh
3dsecure 3D Secure Flag "true", or "false"
3dsecure_callback_url Merchant callback url "http://sample.co.id/callback"
3dsecure_notification_url Merchant notification url "http://sample.co.id/notification"
Contoh JSON (Body Content)

3D Secure Callback Response

Setelah pelanggan menyelesaikan transaksi, VT-Direct akan mengirimkan sebuah HTTP(S) POST message ke merchant (ke alamat yang telah dicantumkan di parameter 3dsecure_callback_url dengan POST parameter vtdirect. Merchant dapat menangkap POST parameter seperti biasa, misalnya jika menggunakan PHP: 

$data = $_POST[‘vtdirect’];

maka $data akan berisi JSON String sebagai berikut:

{
  "data": {
    "order_id": "768798",
    "transaction_status": "authorize" 
    },
  "code": "VD00",
  "status": "success",
  "message": "Success get status transaction"
}
3D Secure Notification Response

Selain Callback Response, VT-Direct juga mengirim notification ke alamat 3dsecure_notification_url yang telah ditentukan merchant sebelumnya. Untuk alasan keamanan, sebaiknya merchant menggunakan Notification Response ini untuk mengupdate status transaksi pelanggan, bukan pada Callback Response. Pastikan bahwa server_key yang didapat dari notification sama dengan server_key yang ada di server merchant.

Berikut ini adalah contoh data yang dikirim melalui 3dsecure_notification_url:

{
  "data": {
    "order_id": "77897982",
    "transaction_status": "authorize",
    "server_key": "..."
    },
  "code": "VD00",
  "status": "success",
  "message": "Success get status transaction"
}

Point Transaction

Anda juga dapat menerima pembayaran melalui point kartu kredit yang telah dikumpulkan oleh pemegang kartu kredit. Point dapat digunakan untuk membayar sebagian atau seluruh jumlah yang harus dibayar dalam suatu transaksi.

Mengecek jumlah point (point balance)

Sebelum pelanggan dapat melakukan transaksi dengan menggunakan point, Anda perlu untuk menunjukkan ke pelanggan berapa jumlah point yang dimilikinya. Berikut ini adalah API untuk mendapatkan jumlah point dalam kartu kredit pelanggan.

Endpoint:
Endpoint : https://payments.veritrans.co.id/vtdirect/v1/balance
HTTP Method : POST
Header:
Accept : application/json
Content-Type : application/json
Authorization : Basic Base64 (server-key:) (Please look at the section on Charge transaction)
Body Content:
{
  "token_id" : "e0fcc426-4a7c-4902-a332-3a3e563ec86e-410505-1244", 
  "bank" : "bni"
}
Parameter Description Example
Token_id String Token e0fcc426-4a7c-4902-a332-3a3e563ec86e-410505-1244
Bank Name of bank BNI, CIMB
Contoh Response:
{
  "data": {
    "card_number": "4105XXXXXXXXXX44",
    "balance": 4907950
  },
  "code": "VD00",
  "status": "success",
  "message": "Success get balance inquiry"
}

Point Transaction

Setelah memastikan bahwa pemegang kartu kredit memiliki point, merchant kemudian meminta pemegang kartu kredit untuk menentukan berapa banyak point yang akan digunakan untuk transaksi. Pastikan bahwa point yang akan digunakan tidak melebihi point yang dimiliki. Berikut ini adalah contoh pemanggilan Request Charge dengan menggunakan point:

Endpoint:
Endpoint : https://payments.veritrans.co.id/vtdirect/v1/charges
HTTP Method : POST
Header:
Accept : application/json
Content-Type : application/json
Authorization : Basic Base64 (server-key:) (Please look at the section on Charge transaction)

Catatan: Endpoint, HTTP Method, dan Header sama persis dengan yang digunakan saat charge transaction normal.

bank Nama bank bni or cimb

Installment Transaction

Untuk mengaktifkan fitur Installment Transaction (cicilan bulanan), merchant terlebih dahulu harus sudah memiliki kesepakatan dengan acquiring bank.

Endpoint:
Endpoint : https://payments.veritrans.co.id/vtdirect/v1/charges
HTTP Method : POST
Header:
Accept : application/json
Content-Type : application/json
Authorization : Basic Base64 (server-key:) (Please look at the section on Charge transaction)

Catatan: Endpoint, HTTP Method, dan Header sama persis dengan yang digunakan saat charge transaction normal.

Parameter tambahan:

Agar suatu transaksi memiliki pilihan pembayaran secara cicilan, Anda perlu menambahkan parameter type dan installment. Kedua parameter tersebut ditambahkan pada tahap Charge Transaction.

Parameter Deskripsi Contoh
type Type of transaction installment
installment Details of installment plan JSON object. See details below.
Parameter Deskripsi Contoh
bank Name of bank bni atau cimb
term Integer. Jumlah bulan installment. 3, 6, 12, ...
type Kode installment yang Anda terima dari bank 1
Contoh JSON (Body Content)

Contoh JSON (Body Content) dengan tambahan parameter type dan installment saat mengirim Charge Request:
