Perkenalan VT-Direct

VT-Direct adalah RESTful API yang memungkinkan merchant untuk menggunakan sistem pembayaran Veritrans untuk memproses pembayaran di website merchant. VT-Direct membolehkan pelanggan untuk menyelesaikan proses pembayaran di website merchant tanpa perlu dialihkan (di-redirect) ke halaman Veritrans, tetapi dengan tetap meminimalisasi resiko keamanan yang muncul saat memproses pembayaran menggunakan kartu kredit. Dengan menggunakan script JavaScript berukuran kecil yang dipasang di website merchant, VT-Direct mengirim data kartu kredit secara langsung ke server Veritrans tanpa terlebih dahulu disimpan di server merchant.

Alur kerja VT-Direct

Ada tiga pihak yang terkait dalam proses pembayaran melalui VT-Direct: pelanggan, merchant dan Veritrans. Silakan lihat kembali alur kerja ini saat mengintegrasikan sistem Anda dengan Veritrans VT-Direct.

  1. Click Pay

    Pelanggan memasukkan informasi kartu kreditnya di website merchant, kemudian mengeklik tombol "Pay (bayar)" untuk menyelesaikan pembayaran.

  2. Send Card Data

    Ketika pelanggan mengeklik tombol "Pay (Bayar)", Veritrans.js, sebuah script JavaScript yang di-load dari server veritrans akan menggabungkan data sensitif(nomor kartu kredit, CVV, dan expired date) dengan public-key kemudian mengirimkannya ke server Veritrans untuk me-request sebuah onetime-token (Token).

  3. Get Token

    Veritrans akan mengembalikan sebuah Token ke browser pelanggan. Token hanya akan valid untuk jangka waktu 75 detik, dan tidak dapat digunakan lagi setelahnya.

  4. Send Token

    Token kemudian dikirimkan ke server merchant. Perlu diperhatikan bahwa merchant hanya mendapatkan Token yang merepresentasikan informasil kartu kredit, dan tidak secara langsung menyimpan ataupun memproses informasi sensitif kartu kredit.

  5. Request Charge

    Merchant mengirim request 'charge' ke Veritrans yang berisi parameter Token dan detail lain yang dibutuhkan Veritrans untuk memproses transaksi (Misalnya: detail produk dan alamat pengiriman).

  6. Transaction Response Handling

    Veritrans mengembalikan respon ke server merchant hasil dari proses transaksi ke server merchant(Contoh: successful atau failure).
