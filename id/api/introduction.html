Pengenalan Veritrans API

Tutorial berikut akan membantu anda memahami apa itu Veritrans API beserta dasar-dasar implementasinya. Apabila butuh bantuan, silahkan hubungi kami di support@veritrans.co.id atau kunjungi halaman support kami.

Selamat datang di Veritrans API!

Veritrans API adalah sebuah RESTful Web Service yang dikembangkan oleh Veritrans sebagai jembatan komunikasi antara Anda dengan layanan kami.
Veritrans API menggunakan konsep One Message, dimana semua channel pembayaran menggunakan format pesan yang sama.

Info: Note:
Veritrans memiliki beragam plugins dan libraries yang tersedia untuk mempercepat proses integrasi.



HTTP(S) Request

Veritrans API dapat diakses melalui HTTP(S) Request ke endpoint Veritrans Base URL.
HTTP(S) Header harus digunakan untuk memungkinan otentikasi yang tepat.

API Base URL

Development Environment https://api.sandbox.veritrans.co.id/v2
Production Environment https://api.veritrans.co.id/v2


API End Point

Endpoint HTTP Method Definisi
BASE_URL/token POST Tokenisasi informasi Kartu Kredit sebelum charge transaksi
BASE_URL/charge POST Melakukan transaksi dengan metode pembayaran dan fitur yang tersedia
BASE_URL/capture POST Charge transaksi Pre-Authorized untuk pembayaran Kartu Kredit
BASE_URL/order_id/approve POST Approve transaksi untuk order_id dengan status challenge dari Sistem Deteksi Fraud Veritrans
BASE_URL/order_id/cancel POST Cancel transaksi untuk order_id tertentu. Pembatalan hanya bisa dilakukan sebelum proses settlement
BASE_URL/order_id/status GET Mendapatkan informasi status dari order_id tertentu
BASE_URL/order_id/expire POST Merubah status transaksi menjadi expire dari order_id dengan status pending


Caution: NOTE:
Data yang dikirimkan ke Development Environment bersifat pencobaan.
Servis Veritrans dapat sepenuhnya digunakan di Production Environment.


HTTP(S) Header

Header Value Definisi
Content-Type application/json Field Content-Type menunjukkan bahwa data tipe JSON diperbolehkan untuk dikirim ke penerima.
Accept application/json Field Accept menunjukkan bahwa data tipe JSON diperbolehkan sebagai respon.
Authorization Basic base64(‘server key’ , ‘:’) Field Authorization dapat ditemukan melalui Veritrans Merchant Admin Portal.

Content-Type dan Accept Header

Konten request dan response Veritrans API akan dalam format JSON.

  • Untuk menerima parameter input dan output JSON, Anda perlu menambahkan HTTP(S) header sebagai berikut:

Content-type: application/json
Accept: application/json

Authorization Header

Authorization Header membutuhkan Veritrans Server Key yang mengikuti konvensi HTTP(S) BASIC AUTH

  • Untuk Development environment, dapatkan Server Key di Sandbox MAP Settings - Access Keys.
  • Untuk Production environment, dapatkan Server Key di Settings - Access Keys.


Karena tidak ada Password untuk Server Key, Anda tidak perlu mengisi kolom Password.

`Username:Password`

Apabila Server Key adalah seperti berikut:

`94960ece-9513-4265-9cf2-67a4da330213`

Maka Username BASIC AUTH dan format passwordnya adalah:

`94960ece-9513-4265-9cf2-67a4da330213:`

(Note: Pastikan ada karakter “:” setelah Server Key!)

Encode nilai tersebut ke dalam format BASE 64, maka hasilnya akan seperti berikut:

`OTQ5NjBlY2UtOTUxMy00MjY1LTljZjItNjdhNGRhMzMwMjEzOg==`
  • Untuk mendapatkan authorization menggunakan Veritrans API, Anda perlu menambahkan header HTTP(S) header berikut:

Authorization: Basic OTQ5NjBlY2UtOTUxMy00MjY1LTljZjItNjdhNGRhMzMwMjEzOg==

Caution: NOTE:
Authorization Header ini hanya contoh. 
Silahkan gunakan Sandbox/Production Server Key Anda!
