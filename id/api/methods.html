Veritrans API Methods

Endpoint Definisi
GET
https://api.veritrans.co.id/v2/token		 Tokenisasi informasi Kartu Kredit sebelum charge transaksi
POST
https://api.veritrans.co.id/v2/charge		 Melakukan transaksi dengan metode pembayaran dan fitur yang tersedia
POST
https://api.veritrans.co.id/v2/capture		 Charge transaksi Pre-Authorized untuk pembayaran Kartu Kredit
POST
https://api.veritrans.co.id/v2/query/approve		 Approve transaksi untuk order_id atau transaction_id dengan status challenge dari Sistem Deteksi Fraud Veritrans
POST
https://api.veritrans.co.id/v2/query/cancel		 Cancel transaksi untuk order_id atau transaction_id tertentu. Pembatalan hanya bisa dilakukan sebelum proses settlement
GET
https://api.veritrans.co.id/v2/query/status		 Mendapatkan informasi status dari order_id atau transaction_id tertentu
POST
https://api.veritrans.co.id/v2/query/expire		 Expire transaksi untuk order_id atau transaction_id tertentu. Expire hanya bisa dilakukan untuk transaksi dengan status pending

Info: Note: Pada sandbox gunakan https://api.sandbox.veritrans.co.id



Get Token

Token ID adalah nilai unik yang berhubungan dengan informasi kartu kredit pelanggan selama transaksi.

Metode Get Token mengirimkan info kartu kredit via Veritrans.min.js ke server Veritrans dan mengembalikan Token ID melalui server Veritrans untuk Charge Transaction.

Info: Note: Metode dan fitur Get Token hanya bisa digunakan untuk transaksi kartu kredit


Veritrans Javascript Library

Library Javascript Veritrans memungkinkan informasi kartu kredit dikirim secara aman ke server Veritrans.
Untuk menggunakan library tersebut, tambahkan kode berikut didalam tag <head> di halaman pembayaran.

<script src="https://api.veritrans.co.id/v2/assets/js/veritrans.min.js"></script>

function callback(response) {
    if (response.redirect_url) {
      // If 3Dsecure transaction. Open 3Dsecure dialog
      console.log('Open Dialog 3Dsecure');
      openDialog(response.redirect_url);

    } else if (response.status_code == '200') {
      // success 3d secure or success normal
      //close 3d secure dialog if any
      closeDialog();

      // store token data in input #token_id
      $("#token-id").val(response.token_id);

    } else {
      // failed request token
      //close 3d secure dialog if any
      closeDialog();
      $('#submit-button').removeAttr('disabled');
      // Show status message.
      $('#message').text(response.status_message);
      console.log(JSON.stringify(response));
    }
  }

Menangani Notifikasi

Untuk meningkatkan keamanan notifikasi yang diterima, ada beberapa cara yang bisa dilakukan oleh anda untuk memastikan notifikasi yang diterima adalah benar berasal dari Veritrans.


1. Menanyakan Status transaksi

Melakukan proses get status sesaat setelah mendapatkan http notifikasi dari Veritrans dan mencocokan hasilnya.


2. Signature Key

Mencocokan nilai signature key yang dikirim oleh notifikasi dengan nilai signature key yang dihasilkan oleh merchant. Tujuan utama signature key untuk mem-validasi apakah notifikasi yang diterima berasal dari Veritrans atau bukan. Merchant dapat mengabaikan notifikasi jika hasil signature key tidak sama. Berikut formula dari signature key yang digunakan oleh Veritrans.

SHA512(order_id+status_code+gross_amount+serverkey)


Berikut adalah contoh kode untuk menghasilkan signature key.

<?php
$orderId = "1111";
$statusCode = "200";
$grossAmount = "100000.00";
$serverKey = "askvnoibnosifnboseofinbofinfgbiufglnbfg";
$input = $orderId.$statusCode.$grossAmount.$serverKey;
$signature = openssl_digest($input, 'sha512');
echo "INPUT: " , $input."<br/>";
echo "SIGNATURE: " , $signature;
?>
