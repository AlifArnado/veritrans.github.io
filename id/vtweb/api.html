Parameter API

API VT-Web Veritrans terdiri dari API untuk request token dan redirection ke halaman pembayaran. Silakan lihat secara rinci batasan-batasan untuk setiap parameter API dan contoh-contoh dari response yang sukses atau gagal.

Request Token API

URL Endpoint

Resource URL:

https://vtweb.veritrans.co.id/v1/tokens.json

Type:

JSON

HTTP Method:

POST

HTTP Header:

Content-Type: application/json
Accept: application/json

Request Token Parameters

Parameter-parameter yang wajib diisi

Nama Tipe data Panjang Format Deskripsi
version Integer 1 1 Agar dapat menggunakan parameter pada dokumentasi ini, isi version dengan nilai 1.
merchant_id String 22 A-Z0-9 Identifier Merchant..
merchanthash String 128 SHA-512 hex Nilai sha512 hash yang dihasilkan dari kombinasi merchant_id, merchant_hash_key, dan order_id. Klik di sini untuk melihat contoh kode untuk men-generate merchanthash: PHP, Ruby.
order_id String 30 a-zA-Z0-9 Identifier Order.
billing_different_with_shipping Integer 1 1 or 0 1 = jika informasi billing berbeda dengan informasi shipping; 0 = jika informasi billing sama dengan informasi shipping.
required_shipping_address Integer 1 0 and 1 1 = jika informasi shipping dibutuhkan; 0 = jika informasi shipping tidak dibutuhkan.
repeat_line Integer 12 0-9 Total banyaknya jenis barang yg ada di keranjang belanja.
item_id Array 12 a-zA-Z0-9 Identifier barang.
price Integer 9 > 0 and < 100000000 Harga (satu jenis) barang dalam rupiah.
quantity Integer 4 > 0 and < 1000 Banyaknya (satu jenis) barang.
item_name1 Array 20 a-zA-Z0-9 -_',.@&+/ Nama barang (Misalkan nama dalam bahasa Inggris.).
item_name2 Array 20 a-zA-Z0-9 -_',.@&+/ Nama barang (Misalkan nama dalam bahasa Indonesia. Boleh sama dengan item_name1).

Parameter-parameter yang wajib diisi jika required_shipping_address = 1

Nama Tipe data Panjang Format Deskripsi
shipping_first_name String 20 a-z A-Z Nama depan untuk pengiriman.
shipping_last_name String 20 a-z A-Z Nama belakang untuk pengiriman.
shipping_address1 String 100 a-zA-Z0-9-_',.@()/ \ Alamat pengiriman (baris pertama).
shipping_address2 String 100 a-zA-Z0-9-_',.@()/ \ Alamat pengiriman (baris kedua).
shipping_city String 20 a-zA-Z-_', .@ Kota alamat pengiriman.
shipping_country_code String 3 A-Z Negara alamat pengiriman. ISO 3166-1 alpha-3 – Upper Case; Contoh "IDN" untuk Indonesia.
shipping_postal_code String 9 0-9 Kodepos alamat pengiriman.
shipping_phone String 19 +0-9 - Nomor telepon alamat pengiriman. Lebih dari 9 digit dan boleh diawali dengan tanda '+'.
email String 30 email Alamat email.

Parameter-parameter yang tidak wajib diisi (opsional).

Nama Tipe data Panjang Format Deskripsi
payment_methods Array - a-z A-Z Metode pembayaran yang ditampilkan. Misalnya: ['credit_card', 'mandiri_clickpay'].
finish_payment_return_url String 256 URL Url untuk kembali ke website merchant setelah pelanggan menyelesaikan transaksi.
unfinish_payment_return_url String 256 URL Url untuk kembali ke website merchant saat proses transaksi tidak selesai.
error_payment_return_url String 256 URL Url untuk kembali ke website merchant jika terjadi error pada saat transaksi dilakukan.
first_name String 20 a-z A-Z Nama depan untuk informasi billing.
last_name String 20 a-z A-Z Nama belakang untuk informasi billing.
address1 String 100 a-zA-Z0-9 -_',.@()/\ Alamat untuk informasi billing (baris pertama).
address2 String 100 a-zA-Z0-9 -_',.@()/\ Alamat untuk informasi billing (baris kedua).
city String 20 a-zA-Z-_', .@ Kota untuk alamat billing.
country_code String 3 A-Z Negara alamat billing. ISO 3166-1 alpha-3 – Upper Case; Contoh "IDN" untuk Indonesia.
postal_code String 9 0-9 Kodepos alamat billing.
phone String 19 +0-9 - Nomor telepon untuk billing. Lebih dari 9 digit, boleh diawal dengan tanda '+'.
promo_bins Array 16 0-9 Kode promo BIN, 1 sampai 16 digit; Contoh: ['411111', '5']. Info lebih lanjut..
enable_3d_secure Integer 1 1 or 0 Mengaktifkan fitur 3D Secure, 1 = aktif; 0 = tidak aktif. Info lebih lanjut..
point_banks Array - 'bni', 'cimb' Pengaturan bank yang aktif untuk fitur Point Banks. Contoh: ['bni', 'cimb']. Info lebih lanjut..
installment_banks Array - 'bni', 'cimb', 'mandiri' Pengaturan bank yang aktif untuk fitur Installment Banks. Contoh: ['bni', 'cimb', 'mandiri']. Info lebih lanjut..
installment_terms JSON Array - Bank sebagai key, dan terms sebagai value. Pengaturan installment terms. Contoh: {"bni":[3, 6, 12], "cimb":[3, 6, 12]}. Info lebih lanjut..
bank String - 'bni', 'cimb' or 'mandiri' Pengaturan untuk acquiring bank yang dipilih. Contoh: 'bni'. Info lebih lanjut..

Response

Type:

JSON

Contoh response jika sukses:

{
    "token_merchant": "3a258a55-6729-4eda-aeaa-c19e5a9af022",
    "token_browser": "320b9231-a93b-4fd0-803f-54e4f093937d"
}

Contoh response jika terjadi kesalahan:

{
    "errors": {
        "shipping_address1": "should be alphanumeric and specified symbol",
        "shipping_country_code": [
            "should be in ISO 3166-1 alpha-3",
            "is the wrong length (should be 3 characters)"
        ],
        "installment_banks": "must be an Array"
    }
}

Redirection API

URL Endpoint

Resource URL:

https://vtweb.veritrans.co.id/v1/payments.json

Type:

JSON

HTTP Method:

POST

HTTP Header:

Content-Type: application/json
Accept: application/json

Redirection Parameters

Nama parameter Deskripsi
merchant_id Merchant ID
order_id Order ID digenerate oleh Merchant. Order ID yang sama yang digunakan pada saat request token.
token_browser token_browser didapatkan dari proses request token.

Response

Tidak ada. Pelanggan akan diarahkan ke halaman pembayaran VT-Web di website Veritrans.

Payment Notification

Setelah proses pembayaran berhasil. Veritrans akan mengirimkan notifikasi berupa email yang ditujukan kepada Merchant dan Customer. Selain itu, Veritrans juga akan mengirimkan notifikasi berbentuk HTTP(S) POST request ke server Merchant. Adapun target URL yang akan dituju Veritrans dapat dikonfigurasikan sesuai dengan keinginan Merchant di MAP.

Contoh HTTP(S) POST parameter payment notification yang dikirim ke server merchant.

Parameter Contoh Deskripsi
orderId QBOHHJ  
mStatus success Status transaksi pembayaran: success, failure, challenge, or fatal.
TOKEN_MERCHANT af27bd7b-c4c1-45bd-b730-a46de616dbe5  

Parameter mStatus terkat dengan Veritrans Fraud Detection System. mStatus merepresentasikan respon dari fraud detection system.

mStatus Respon dari Fraud Detection System
success ACCEPT
failure DENY
challenge CHALLENGE
fatal Terjadi kesalahan pada sistem Veritrans. Silakan hubungi Veritrans Support jika hal ini terjadi.
