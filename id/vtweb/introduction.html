Perkenalan VT-Web

VT-Web merupakan sebuah cara mudah untuk menambahkan fitur pembayaran pada toko online Anda. Dengan menggunakan VT-Web, pembeli akan dialihkan dari toko online Anda ke halaman pembayaran Veritrans untuk melakukan pembayaran. Halaman pembayaran Veritrans membantu pembeli untuk melakukan pembayaran dengan menggunakan berbagai macam metode pembayaran seperti kartu kredit, Mandiri Clickpay, dan CIMB Clicks di dalam lingkungan yang aman, dan pada waktu yang bersamaan juga meminimalisir pekerjaan dari tim teknis Anda untuk melakukan integrasi dengan sistem Veritrans.

Bagaimana melakukan transaksi dengan VT-Web

Untuk melakukan transaksi dengan VT-Web, merchant perlu mengirim request ke endpoint charge dengan spesifikasi sebagai berikut :

Deskripsi Melakukan suatu transaksi
Endpoint sandbox : https://api.sandbox.veritrans.co.id/v2/charge
  Production : https://api.veritrans.co.id/v2/charge
HTTP Method POST
HTTP Header Accept: application/json
  Content-Type: application/json
  Authentication: Basic base64(server_key:)

Contoh Body Request (JSON) :

{
    "payment_type": "vtweb",
    "vtweb": {
        "enabled_payments": [
            "credit_card"
        ]
    },
    "transaction_details": {
        "order_id": "order45",
        "gross_amount": 200000
    },

Saat mengirim request charge dengan payment_type: "vtweb", Veritrans akan mengembalikan response berupa json yang berisi tiga parameter, yaitu : redirect_url, status_code dan status_message. Merchant perlu mengarahkan customer ke alamat url yang terdapat di parameter redirect_url, alamat tersebut akan mengarah ke halaman pembayaran VT-Web Veritrans.

Setelah diarahkan ke alamat redirect_url, customer akan melihat halaman pembayaran VT-Web seperti berikut :

Redirect URL dan Notifikasi

Redirect URL

Saat berada di halaman pembayaran Veritrans VT-Web, ada tiga kemungkinan yang akan terjadi :

  1. Customer sukses melakukan pembayaran. Setelah sukses melakukan pembayaran, customer akan di alihkan kembali ke halaman website merchant. Alamat spesifik kemana customer akan dialihkan setelah sukses melakukan pembayaran di sebut Finish Redirect URL.

  2. Customer gagal melakukan pembayaran. Jika customer gagal melakukan pembayaran, customer akan dialihkan kembali ke website merchant. Alamat spesifik kemana customer akan dialihkan setelah gagal melakukan pembayaran disebut Error Redirect URL.

  3. Customer batal melakukan pembayaran. Jika customer batal melakukan pembayaran (menekan tombol "back to merchant website"), customer akan dialihkan kembali ke website merchant. Alamat spesifik kemana customer akan dialihkan saat membatalkan pembayaran pembayaran disebut Unfinish Redirect URL.

Notification

Setelah customer menyelesaikan proses pembayaran, apapun hasil proses pembayaran tersebut baik berhasil ataupun gagal, Veritrans akan mengirimkan dua jenis notifikasi.

  1. Notifikasi berupa email. Notifikasi ini dikirim ke alamat email customer yang melakukan pembayaran serta ke alamat email merchant, merchant dapat melakukan setting pada menu Setttings - email notification pada MAP.

  2. Notifikasi berupa HTTP(S) POST. Notifikasi berupa HTTP(S) POST dikirim oleh Veritrans ke server merchant. Merchant dapat memanfaatkan HTTP(S) POST notification ini untuk meng-update status pembayaran dari suatu transaksi secara real time.

Ketiga alamat redirect URL diatas (Finish, Unfinish, Error Redirect URL) serta alamat URL untuk payment notification dapat merchant tentukan melalui menu Settings - Configuration pada MAP
