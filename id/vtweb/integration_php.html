Mendapatkan Redirect URL

Pada tahap ini, kita akan mencoba memperoleh halaman pembayaran VT-Web untuk suatu pemesanan. Setidaknya ada 3 informasi yang dibutuhkan untuk mendapatkan halaman pembayaran VT-Web, yaitu:

  1. server key
  2. ID transaksi
  3. Nominal transaksi

Pada contoh berikut, toko anda menerima sebuah pemesanan dengan ID 7009 senilai Rp. 10.000,00. Contoh kode PHP yang dapat digunakan untuk memperoleh dan mengakses halaman pembayaran VT-Web adalah:

<?php 
include_once('Veritrans.php');

// Use sandbox account
Veritrans_Config::$isProduction = false;

// Set our server key
Veritrans_Config::$serverKey = '<your server key>';

// Fill transaction data
$transaction = array(
    "vtweb" => array (
          "credit_card_3d_secure" => true,
        ),
    'transaction_details' => array(
        'order_id' => rand(),
        'gross_amount' => 10000,
        )
    );

$vtweb_url = Veritrans_Vtweb::getRedirectionUrl($transaction);

// Redirect 
header('Location: ' . $vtweb_url);
?>

Contoh kode di atas memanfaatkan library Veritrans-PHP yang telah diunduh pada bagian Langkah awal. File Veritrans.php merupakan bagian dari library Veritrans-PHP. File ini diasumsikan berada pada direktori yang sama dengan kode sumber.

Pertama-tama, tentukan jenis akun yang akan kita gunakan: akun produksi atau sandbox. Pada contoh, akun yang digunakan adalah sandbox. Oleh karena itu, nilai Veritrans_Config::$isProduction diubah menjadi false.

Ketika halaman VT-Web ditampilkan, semua metode pembayaran yang anda miliki akan ditampilkan dalam bentuk tab. Hal ini merupakan default jika anda tidak memasukkan parameter enabled_payments pada saat melakukan charge transaksi. Jika anda bermaksud menentukan sendiri metode pembayaran yang ditampilkan pada halaman VT-Web, anda dapat mengikuti petunjuk VT-Web - Fitur Tambahan.

URL Halaman Pembayaran VT-Web dapat diperoleh dengan memanggil fungsi Veritrans_Vtweb::getRedirectionUrl dimana data transaksi adalah argumennya. Jika proses ini berhasil, pembeli akan di-redirect menuju URL tersebut.

Menambahkan Informasi Transaksi (Opsional)

Merchant dapat menambahkan berbagai data tambahan ke dalam variabel transaksi $transaction, seperti data pembeli, data pesanan barang, dan metode pembayaran. Data tambahan ini akan mempengaruhi halaman pembayaran VT-Web dan log transaksi pada MAP (Merchant Administration Portal).

Sebagian besar field pada form halaman VT-Web akan otomatis terisi dan Informasi terkait daftar barang serta lokasi pengiriman akan ditampilkan. Keterangan dan struktur lengkap data transaksi dalam bentuk JSON dapat dilihat di sini.

Contoh kode PHP yang mengikutsertakan semua data transaksi adalah sebagai berikut:

<?php
include_once('Veritrans.php');

// Set our server key
Veritrans_Config::$serverKey = '<your server key>';

// Use sandbox account
Veritrans_Config::$isProduction = false;

// Required
$transaction_details = array(
  'order_id' => rand(),
  'gross_amount' => 145000,
  );

// Opsional
$item1_details = array(
    'id' => 'a1',
    'price' => 50000,
    'quantity' => 2,
    'name' => "Apel"
    );

// Opsional
$item2_details = array(
    'id' => 'a2',
    'price' => 45000,
    'quantity' => 1,
    'name' => "Jeruk"
    );

// Opsional
$items_details = array ($item1_details, $item2_details);

// Opsional
$billing_address = array(
    'first_name'    => "Andri",
    'last_name'     => "Litani",
    'address'       => "Mangga 20",
    'city'          => "Jakarta",
    'postal_code'   => "16602",
    'phone'         => "081122334455",
    'country_code'  => 'IDN'
    );

// Opsional
$shipping_address = array(
    'first_name'    => "Obet",
    'last_name'     => "Supriadi",
    'address'       => "Manggis 90",
    'city'          => "Jakarta",
    'postal_code'   => "16601",
    'phone'         => "08113366345",
    'country_code'  => 'IDN'
    );

// Opsional
$customer_details = array(
    'first_name'    => "Andri",
    'last_name'     => "Litani",
    'email'         => "andri@litani.com",
    'phone'         => "081122334455",
    'billing_address'  => $billing_address,
    'shipping_address' => $shipping_address
    );

// Fill transaction details
$transaction = array(
    'payment_type' => 'vtweb',
    'vtweb' => array(
        'credit_card_3d_secure' => true,
        ),
    'transaction_details' => $transaction_details,
    'customer_details' => $customer_details,
    'item_details' => $items_details,
    );

$vtweb_url = Veritrans_Vtweb::getRedirectionUrl($transaction);

// Go to VT-Web page
header('Location: ' . $vtweb_url);
?>

Berikut adalah contoh tampilan halaman pembayaran VT-Web hasil kode di atas. Perlu diperhatikan bahwa field pada form tagihan sudah otomatis terisi. Selain itu, informasi terkait daftar barang dan alamat pengiriman juga ditampilkan pada halaman VT-Web.

Error saat Request URL VT-Web

Pada beberapa kondisi, request URL pembayaran VT-Web bisa gagal. Penyebab gagalnya request URL VT-Web dapat dilihat dari status code. Status code yang umumnya muncul adalah:

Kode Error Keterangan
400 Validation error Format atau konten data transaksi tidak valid. Contoh: Salah tipe variabel, metode pembayaran tidak valid.
401 Access Denied Client atau server key tidak valid.
402 Access Denied Merchant tidak memiliki izin menggunakan metode pembayaran tertentu.
406 Duplicate Order ID Order ID telah digunakan dan dibayar sebelumnya.
410 Account Inactive Akun merchant tidak aktif dalam waktu lama

Daftar status code lengkap dapat dilihat di Referensi - Status Code. Jika mengalami kesulitan, silahkan hubungi kami.

Tes Transaksi

Anda sangat disarankan untuk melakukan tes transaksi menggunakan VT-Web. Manfaat utama melakukan tes transaksi adalah:

Mengetahui Status Transaksi

Pada tahap sebelumnya, pelanggan dapat mengetahui status transaksi segera setelah proses pembayaran selesai dilakukan. Namun, bagaimana merchant mengetahui hal tersebut?

Veritrans menyediakan dua cara agar merchant dapat mengetahui status transaksi.

  1. MAP (Merchant Administration Portal)
  2. Notifikasi

Status transaksi melalui MAP

Pertama-tama, merchant harus melakukan login MAP Sandbox terlebih dahulu. Merchant dapat melihat daftar transaksi beserta ringkasan status transaksi dengan memilih menu Transaksi pada MAP.

Merchant dapat melihat rincian status transaksi dengan memilih Order ID salah satu item transaksi.

Melihat status transaksi melalui MAP cukup mudah untuk dilakukan. Namun, merchant harus mengecek secara manual setiap saat. Cara ini sulit untuk diintegrasikan dengan sistem informasi merchant. Untuk mengatasi permasalahan tersebut, mekanisme notifikasi status transaksi dapat digunakan. Petunjuk dan keterangan menggunakan notifikasi status transaksi akan dijelaskan pada tahap selanjutnya.

Menangani Notifikasi

Tahap ini akan memberikan penjelasan kepada anda, bahwa Veritrans akan mengirimkan notifikasi saat transaksi dilakukan. Setelah pelanggan anda menyelesaikan proses pembayaran, apapun hasil proses pembayaran tersebut, baik berhasil maupun gagal (misalnya karena kartu kredit telah melampaui limit), Veritrans akan mengirim dua jenis notifikasi.

1. Notifikasi berupa email

Notifikasi berupa email di kirim ke alamat email pelanggan anda yang melakukan pembayaran, serta ke alamat email anda sebagai penerima pembayaran. Anda dapat melakukan setting seputar email notifikasi pada menu Settings - email notification di dalam MAP.

Note: Untuk multiple emails, mohon gunakan tanda koma sebagai separator. Contoh : Andri@myemail.com, litani@myemail.com

2. Notifikasi berupa HTTP(S) POST

Notifikasi berupa HTTP(S) POST dikirim oleh Veritrans ke server anda. Anda dapat memanfaatkan HTTP(S) POST notification ini untuk mengupdate status pembayaran dari suatu transaksi (order) secara real time. Hal ini bisa dilakukan karena Veritrans mengirim notifikasi ini seketika setelah pelanggan menyelesaikan proses pembayaran.

Untuk memanfaat fasilitas ini, anda harus menyediakan endpoint untuk menerima notifikasi. Endpoint notifikasi bisa di-set melalui MAP, di halaman Settings - Configuration pada MAP.

Note: Pastikan alamat notifikasi HTTP POST yang anda masukan tidak berupa alamat localhost.

Berikut adalah contoh kode untuk menerima HTTP (POST) dan JSON object menggunakan Library Veritrans-PHP. File ini adalah file PHP yang akan dieksekusi ketika URL endpoint notifikasi (http://sneekers.com/payment/notification) diakses.

<?php

require_once('Veritrans.php');

Veritrans_Config::$isProduction = false;

$notif = new Veritrans_Notification();

$transaction = $notif->transaction_status;
$type = $notif->payment_type;
$order_id = $notif->order_id;
$fraud = $notif->fraud_status;

if ($transaction == 'capture') {
  // For credit card transaction, we need to check whether transaction is challenge by FDS or not
  if ($type == 'credit_card'){
    if($fraud == 'challenge'){
    // TODO set payment status in merchant's database to 'Challenge by FDS'
    // TODO merchant should decide whether this transaction is authorized or not in MAP

      echo "Transaction order_id: " . $order_id ." is challenged by FDS";
    } else {
    // TODO set payment status in merchant's database to 'Success'

      echo "Transaction order_id: " . $order_id ." successfully captured using " . $type;
    }
  } else if ($transaction == 'settlement'){
  // TODO set payment status in merchant's database to 'Settlement'

    echo "Transaction order_id: " . $order_id ." successfully transfered using " . $type;
  } else if($transaction == 'pending'){
  // TODO set payment status in merchant's database to 'Pending'

    echo "Waiting customer to finish transaction order_id: " . $order_id . " using " . $type;
  } else if ($transaction == 'deny') {
  // TODO set payment status in merchant's database to 'Denied'

    echo "Payment using " . $type . " for transaction order_id: " . $order_id . " is denied.";
  }
}
?>
