Fitur-Fitur Lain

BIN Promo

Pengenalan

Fitur BIN Promo memungkinkan merchant untuk menawarkan promosi berdasarkan BIN (Bank Identification Number) dari kartu kredit yang dimiliki pelanggan. BIN adalah 6 digit pertama dari nomor kartu kredit untuk mengidentifikasi bank mana yg menerbitkan kartu kredit tersebut. Fitur ini juga memungkinkan merchant untuk bekerja sama dengan bank tertentu untuk suatu program pemasaran.

Cara Kerja

Contoh sederhana berikut ini bisa menggambarkan bagaimana cara kerja BIN Promo. Merchant bermaksud untuk mengadakan program kerjasama dengan Bank A dengan memberikan diskon pada pelanggan yang menggunakan kartu Bank A. Ketika pelanggan melakukan transaksi pembayaran di halaman VT-Web Veritrans, pelanggan dengan kartu Bank A akan membayar dengan harga diskon. VT-Web dapat memfilter bahwa hanya pelanggan dengan kartu bernomor BIN bank A saja yang dapat melakukan pembayaran.

Implementasi Teknis

Ketika server merchant mengirim informasi transaksi (pada langkah Get Token) dengan harga yang telah didiskon ke VT-Web Veritrans, merchant juga mengirim list BIN nomor kartu kredit yang diperbolehkan untuk melakukan transaksi pembayaran di parameter promo_bins. Veritrans akan memeriksa apakah nomor kartu kredit yang digunakan oleh pelanggan termasuk dalam BIN yang telah ditentukan merchant. Jika nomor kartu kredit tidak termasuk dalam BIN, maka pelanggan tidak dapat melanjutkan pembayaran.

Parameter promo_bins ditambahkan pada tahap Get Token. Lihat halaman Integrasi VT-Web untuk mendapatkan informasi lebih detail tentang tahap Get Token.

PHP
<?php
//...
$veritrans->promo_bins = array("400000","411111");
//...
?>

3D Secure & Verified by Visa

Sebelum Anda dapat mengaktifkan fitur 3D Secure, Anda terlebih dahulu perlu untuk menanyakan sales representative Veritrans untuk memastikan apakah MID 3D Secure Anda sudah siap. Jika sudah siap, Anda dapat mengaktifkan fitur 3D Secure dengan cara mengirimkan parameter enable_3d_secure pada saat tahap Get Token.

PHP
<?php
//...
$veritrans->enable_3d_secure = 1;
//...
?>

Pembayaran Cicilan (Installment Transaction)

Untuk mengaktifkan fitur Installment Transaction (cicilan bulanan), merchant terlebih dahulu harus sudah memiliki kesepakatan dengan acquiring bank. Agar suatu transaksi memiliki pilihan pembayaran secara cicilan, Anda perlu menambahkan parameter installment_banks untuk menentukan bank mana saja yang diperbolehkan, dan parameter installment_terms untuk menentukan pilihan cicilan bulanan yang tersedia. Kedua parameter tersebut ditambahkan pada tahap Get Token.

PHP
<?php
//...
$veritrans->installment_banks 	= ["bni", "cimb"];

/* This will be the default parameters if you don't send any installment_terms parameter
$veritrans->installment_terms = array(
	"bni" => ["3", "6", "12"],
	"cimb"=> ["3", "6", "12"]
);
*/

$veritrans->installment_terms = array(
	"bni"    => [4, 6, 12],
	"cimb"   => [6]
);

//...
?>

Bank Points

Agar dapat menerima transaksi menggunakan point dari kartu kredit, merchant terlebih dahulu harus sudah memiliki kesepakatan dengan acquiring bank. Agar suatu transaksi memiliki pilihan pembayaran menggunakan point, Anda perlu menambahkan parameter point_banks untuk menentukan bank mana saja yang menerima pembayaran menggunakan point. Parameter tersebut ditambahkan pada tahap Get Token. Pada saat melakukan pembayaran di halaman pembayaran VT-Web, pelanggan akan menentukan berapa banyak point yang akan digunakan untuk membayar.

PHP
<?php
//...
$veritrans->point_banks = ['bni', 'cimb'];
//...
?>

Acquiring Bank

Jika merchant memiliki lebih dari satu akun bank yang terdaftar di Veritrans, merchant dapat memilih acquiring banks mana yang akan digunakan untuk suatu transaksi. Untuk memilih akun bank mana yang akan digunakan dalam suatu transaksi, Anda perlu untuk menambahkan parameter bank pada saat tahap Get Token.

PHP
<?php
//...
$veritrans->bank = 'bni';
//...
?>
