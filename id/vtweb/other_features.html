VT-Web: Fitur Tambahan

Tutorial berikut akan membantu anda meng-implementasikan fitur tambahan di VT-Web. Apabila butuh bantuan, silahkan hubungi kami di support@veritrans.co.id atau kunjungi halaman support kami.

Saat ini VT-Web memiliki fitur tambahan:

  1. Memilih Metode Pembayaran
  2. Set Pengalihan URL
  3. Set Custom Fields

  1. Menon-aktifkan 3D Secure

Memilih Metode Pembayaran

Saat ini, Veritrans memiliki beberapa metode pembayaran yang dapat digunakan melalui VT-Web:

Card Payment Direct Debit e-Wallet Bank Transfer Convenience Store
Kartu Kredit Mandiri Clickpay Telkomsel Cash Virtual Account Indomaret
  CIMB Clicks XL Tunai Mandiri Bill Payment  
  e-Pay BRI BBM Money    
    Indosat Dompetku    



Pada bagian VT-Web – Integrasi, kami telah menjelaskan bagaimana mengaktifkan semua metode pembayaran yang anda miliki secara otomatis. Hal ini dapat diimplementasikan dengan tidak menyertakan parameter enabled_payments. Namun, pada kondisi tertentu anda mungkin perlu menampilkan beberapa metode pembayaran saja pada VT-Web. Jika anda menghadapi kondisi ini, silahkan ikuti petunjuk berikut.

Info: Info:
Silahkan lihat API Documentation - Fitur VT-Web untuk informasi lebih mengenai atribut JSON.

Mandiri Clickpay

Mandiri Clickpay merupakan salah satu fitur pembayaran dari mandiri debit. Metode ini memungkinkan pelanggan melakukan transaksi pembayaran dengan memasukkan 16 digit nomor kartu mandiri debit dan Token PIN Mandiri sebagai validasi transaksi.

Untuk mengaktifkan metode pembayaran Mandiri Clickpay (otomatis aktif pada mode Sandbox), merchant perlu:

  1. Daftarkan metode pembayaran Mandiri Clickpay dengan menghubungi sales representative Veritrans.
  2. Menambahkan parameter mandiri_clickpay pada variabel enabled_payments.

Menggunakan contoh transaksi pada bagian Integrasi, contoh kode PHP yang dapat digunakan adalah:

<?php
include_once('Veritrans.php');

...

// Fill transaction details
$transaction = array(
    "vtweb" => array (
          "enabled_payments" => array("mandiri_clickpay")
        ),
    'transaction_details' => array(
        'order_id' => rand(),
        'gross_amount' => 10000
        )
    );

...

?>


Tab Mandiri Clickpay dapat digunakan oleh pelanggan untuk melakukan transaksi. Form beserta petunjuk pembayaran menggunakan Mandiri Clickpay akan ditampilkan pada halaman ini.

Menon-aktifkan 3D Secure

Kami sangat menyarankan agar fitur 3D secure digunakan pada transaksi kartu kredit anda. Penggunaan 3D Secure merupakan standar keamanan transaksi kartu kredit. 3D Secure mampu menambah lapisan keamanan transaksi kartu kredit melalui tambahan proses autentikasi pihak bank dan pelanggan anda. Jika pada kasus tertentu anda terpaksa untuk menon-aktifkan fitur 3D Secure, anda cukup menambahkan parameter "credit_card_3d_secure" => false saat request URL VT-Web.

Menggunakan transaksi yang sama dengan bagian Integrasi, contoh kode PHP yang dapat digunakan adalah sebagai berikut:

<?php
include_once('Veritrans.php');

... 

$transaction = array(
    "vtweb" => array (
          "enabled_payments" => array("credit_card"),
          // use false to disable 3ds
          "credit_card_3d_secure" => false
        ),
    'transaction_details' => array(
        'order_id' => 7145,
        'gross_amount' => 10000
        )
    );

...

?>

Fitur Cicilan

Untuk mengaktifkan fitur cicilan, merchant harus memiliki kesepakatan dengan acquiring bank terlebih dahulu. Acquiring bank akan menerbitkan MID Cicilan yang memungkinkan transaksi diubah menjadi cicilan secara otomatis. Saat ini bank yang sudah support adalah BNI dan Mandiri. Merchant juga perlu menanyakan kepada sales representative Veritrans apakah MID untuk cicilan sudah siap. Jika sudah siap, merchant cukup menambahkan parameter transaksi installment saat request URL VT-Web.

Menggunakan transaksi yang sama dengan bagian Integrasi, contoh kode PHP yang dapat digunakan adalah sebagai berikut:

<?php
include_once('Veritrans.php');

...

$transaction = array(
    "vtweb" => array (
          "enabled_payments" => array("credit_card"),
          "credit_card_3d_secure" => true,
          // Add installment payment option
          "payment_options" => array(
              "installment" => array(
                  "required" => true,
                  "installment_terms" => array(
                      "bni" => array(6,12),
                      "mandiri" => array(6)
                    )
                )
            )
        ),
    'transaction_details' => array(
        'order_id' => rand(),
        'gross_amount' => 10000
        )
    );

...

?>

BIN Promo

BIN (Bank Identification Number) promo adalah fitur yang memungkinkan anda untuk menawarkan promosi berdasarkan bank penerbit kartu kredit tertentu yang dimiliki pelanggan anda. Bank penerbit kartu kredit dapat ditentukan dengan menggunakan BIN atau nama bank penerbit kartu. BIN merupakan 6 digit pertama dari kartu kredit. Sebagai alternatif, anda juga boleh menggunakan 1-8 digit pertama kartu kredit atau nama penerbit kartu yang akan digunakan pada fitur BIN Promo.

Untuk menggunakan fitur ini, anda terlebih dahulu mengumpulkan daftar BIN yang menerima promo atau bisa menggunakan nama bank penerbit kartu. Daftar BIN atau nama bank tersebut menjadi parameter transaksi credit_card_bins, transaksi hanya dapat dilakukan oleh kartu kredit yang termasuk dalam daftar BIN atau kartu yang diterbitkan oleh bank tersebut. Gunakan daftar BIN yang anda miliki sebagai nilai dari atribut credit_card_bins saat charge transaksi.


Menggunakan transaksi yang sama dengan bagian Integrasi, contoh kode PHP yang dapat digunakan adalah sebagai berikut:

<?php
include_once('Veritrans.php');

...

$transaction = array(
    "vtweb" => array (
          "enabled_payments" => array("credit_card"),
          "credit_card_3d_secure" => true,
          //Register BIN list
          "credit_card_bins" => array("48111111", "3111", "5", "bni", "mandiri")
        ),
    'transaction_details' => array(
        'order_id' => 8033,
        'gross_amount' => 10000
        )
    );

$vtweb_url = Veritrans_Vtweb::getRedirectionUrl($transaction);

...

?>

Set Redirect URL

Pelanggan akan dialihkan ke website anda setelah proses pembayaran diselesaikan.
Secara default, redirection URL dapat dikonfigurasi melalui MAP Veritrans melalui Persiapan Awal.

Dalam beberapa situasi, pelanggan dapat mengesampingkan konfigurasi redirection URL VT-Web untuk setiap transaksi dengan menambahkan parameter finish_redirect_url, unfinish_redirect_url, dan error_redirect_url.

Menggunakan transaksi yang sama dengan bagian Integrasi, contoh kode PHP yang dapat digunakan adalah sebagai berikut:

<?php
include_once('Veritrans.php');

...

$transaction = array(
    "vtweb" => array (
          "enabled_payments" => array("credit_card"),
          "credit_card_3d_secure" => true,
          //Set Redirection URL Manually
          "finish_redirect_url" => "http://www.sneekers.co.id/payment/finish", 
          "unfinish_redirect_url" => "http://www.sneekers.co.id/payment/unfinish",
          "error_redirect_url" => "http://www.sneekers.co.id/payment/error"
  }

        ),
    'transaction_details' => array(
        'order_id' => 8033,
        'gross_amount' => 10000
        )
    );

$vtweb_url = Veritrans_Vtweb::getRedirectionUrl($transaction);

..

?>

Silahkan lihat API Documentation - Set Redirect URL untuk informasi lebih mengenai atribut JSON.

Set Custom Expiry

Fitur Pengaturan Custom Expiry memungkinkan merchant untuk set masa waktu berlaku pembayaran dengan status pending di setiap transaksi.

Berikut merupakan daftar pembayaran dengan status Pending:

  • Bank Transfer: Permata Virtual Account, Mandiri Bill Payment
  • Direct Debit: CIMB Clicks, e-pay BRI, BCA KlikPay
  • e-Wallet: XL Tunai, BBM Money

Menggunakan transaksi yang sama dengan bagian Integrasi, contoh kode PHP yang dapat digunakan adalah sebagai berikut:

<?php
include_once('Veritrans.php');

...

$transaction = array(
    "vtweb" => array (
          "enabled_payments" => array("vtweb"),
          "credit_card_3d_secure" => true,
  }

        ),
    'transaction_details' => array(
        'order_id' => 8033,
        'gross_amount' => 10000
        ),

    //Set Custom Expiry
    'custom_expiry' => array ( 
      "expiry_duration" => 30,
      "unit" => "minute"
  )
    );

$vtweb_url = Veritrans_Vtweb::getRedirectionUrl($transaction);

...

?>
