Charge Transaksi

Charge transaksi memungkinkan anda menerima pembayaran secara online. Pada bagian ini, fokus pembahasan adalah spesifikasi dan contoh charge transaksi dasar pada produk VT-Web dan VT-Direct. Pembahasan untuk kedua produk dipisah karena karena tujuan charge transaksi yang berbeda. Pada VT-Web, fungsi charge transaksi digunakan untuk memperoleh URL pembayaran, sedangkan pada VT-Direct fungsi ini digunakan untuk melakukan pembayaran.

Penjelasan mengenai charge transaksi akan dibagi menjadi 5 bagian:

  1. Spesifikasi HTTP(S) POST
  2. Contoh VT-Web (Kartu Kredit, Mandiri Clickpay, CIMB Clicks)
  3. Contoh VT-Direct Kartu Kredit
  4. Contoh VT-Direct Mandiri Clickpay
  5. Contoh VT-Direct CIMB Clicks

Sebenarnya, parameter charge transaksi cukup bervariasi. Banyak parameter yang dapat dikonfigurasi untuk mengaktifkan fitur cicilan, BIN Promo, one click, dan two clicks. Agar konten materi tetap sederhana dan sistematis, fitur charge transaksi tersebut akan dibahas pada bab yang terpisah.

  • Spesifikasi
  • VT-Web
  • Kartu Kredit
  • Mandiri Clickpay
  • CIMB Clicks

Spesifikasi HTTP(S) POST

Deskripsi Melakukan charge transaksi
Endpoint BASE_URL/charge
HTTP Method POST
HTTP Header Accept: application/json
Content-Type: application/json
Authentication: Basic Base64(server_key:)

Kesalahan dalam Charge Transaksi

Pada beberapa kasus, charge transaksi VT-Direct bisa gagal. Veritrans akan mengembalikan pesan error ketika terjadi kegagalan. Penyebab gagalnya charge transaksi VT-Direct dapat dilihat dari status code yang ada di dalam pesan error, status code yang umumnya muncul adalah:

Kode Error Keterangan
400 Validation error Format atau konten data transaksi tidak valid. Contoh: Salah tipe variabel, metode pembayaran tidak valid.
401 Access Denied Client atau server key tidak valid.
402 Access Denied Merchant tidak memiliki izin menggunakan metode pembayaran tertentu.
406 Duplicate Order ID Order ID telah digunakan dan dibayar sebelumnya.
410 Account Inactive Akun merchant tidak aktif dalam waktu lama
411 Token error Token hilang, tidak valid, atau kadaluarsa

Daftar status code lengkap dapat dilihat di Referensi API - Status Code. Jika mengalami kesulitan, silahkan hubungi kami.

Struktur dan Contoh JSON Response VT-Direct Kartu Kredit

Jika charge transaksi tidak mengalami error, server Veritrans akan mengembalikan JSON response. Struktur JSON response VT-Direct kartu kredit adalah sebagai berikut:

Atribut JSON Tipe Data Keterangan
transaction_id String ID transaksi yang diberikan oleh Veritrans
order_id String ID pemesanan yang diberikan oleh pihak merchant
gross_amount String Nilai transaksi
payment_type String Metode pembayaran transaksi
transaction_time String Timestamp transaksi
transaction_status String Status transaksi. Setelah charge transaksi kartu kredit, nilai yang mungkin adalah
capture : transaksi diterima oleh bank. Jika nilai fraud_status adalah challenge, merchant harus accept transaksi secara manual.
deny: transaksi ditolak oleh bank atau FDS.
authorize: kartu kredit valid untuk transaksi (pre-authorization)
fraud_status String Hasil deteksi Fraud Detection System (FDS). NIlai yang mungkin:
accept : lolos FDS.
deny: tidak lolos FDS. Transaksi otomatis gagal.
challenge: Dipertanyakan oleh FDS. Jika transaction_status adalah capture, merchant harus accept transaksi secara manual
masked_card String 6 digit depan dan 4 digit belakang nomor kartu kredit pelanggan
status_code String Status code hasil charge transaksi. Daftar status code lengkap dapat dilihat di sini
status_message String Penjelasan hasil charge transaksi
approval_code String Kode approval. Dapat digunakan untuk melakukan refund. Tidak ada pada denied transaction

Kesalahan dalam Charge Transaksi

Pada beberapa kasus, charge transaksi VT-Direct bisa gagal. Veritrans akan mengembalikan pesan error ketika terjadi kegagalan. Penyebab gagalnya charge transaksi VT-Direct dapat dilihat dari status code yang ada di dalam pesan error, status code yang umumnya muncul adalah:

Kode Error Keterangan
400 Validation error Format atau konten data transaksi tidak valid. Contoh: Salah tipe variabel, metode pembayaran tidak valid.
401 Access Denied Client atau server key tidak valid.
402 Access Denied Merchant tidak memiliki izin menggunakan metode pembayaran tertentu.
406 Duplicate Order ID Order ID telah digunakan dan dibayar sebelumnya.
410 Account Inactive Akun merchant tidak aktif dalam waktu lama

Daftar status code lengkap dapat dilihat di Referensi API - Status Code. Jika mengalami kesulitan, silahkan hubungi kami.

Struktur dan Contoh JSON Response VT-Direct CIMB Clicks

Jika charge transaksi tidak mengalami error, server Veritrans akan mengembalikan JSON response. Struktur JSON response VT-Direct CIMB Clicks adalah sebagai berikut:

Atribut JSON Tipe Data Keterangan
status_code String Status code hasil charge transaksi. Daftar status code lengkap dapat dilihat di sini
status_message String Penjelasan hasil charge transaksi
transaction_id String ID transaksi yang diberikan oleh Veritrans
order_id String ID pemesanan yang diberikan oleh pihak merchant
payment_type String Metode pembayaran transaksi
transaction_time String Timestamp transaksi
transaction_status String Status transaksi. Setelah charge transaksi CIMB Clicks, nilai yang mungkin adalah
pending : menunggu hasil transaksi di halaman redirect_url
redirect_url String URL halaman pembayaran CIMB Clicks. Pelanggan harus diredirect ke halaman ini untuk menyelesaikan pembayaran
gross_amount String Nilai transaksi

Dua informasi penting dalam respon JSON adalah transaction_status dan redirect_url. Jika nilai transaction_status adalah pending, maka transaksi berhasil dicatat pada server Veritrans. Namun, transaksi masih belum selesai dan masih menunggu proses pembayaran pelanggan di halaman redirect_url.

Contoh JSON response saat charge transaksi berhasil: 

{
	"status_code": "201",

	"order_id": "H17550",
	"payment_type": "cimb_clicks",
	"transaction_time": "2014-08-24 15:58:44",

}
