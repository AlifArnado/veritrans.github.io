Pengenalan Veritrans API

Tutorial berikut akan membantu anda memahami apa itu Veritrans API beserta dasar-dasar implementasinya. Apabila butuh bantuan, silahkan hubungi kami di support@veritrans.co.id atau kunjungi halaman support kami.

Veritrans API adalah sebuah RESTful Web Service yang dikembangkan oleh Veritrans Indonesia sebagai jembatan komunikasi antara anda dengan payment provider. Anda dapat menggunakan Veritrans API untuk mengakses berbagai service dan produk yang Veritrans sediakan untuk melakukan pembayaran online.

Veritrans API mengusung konsep One Message, dimana format pesan yang digunakan untuk transaksi pembayaran apapun menggunakan format yang sama. Veritrans API menggunakan format JSON (JavaScript Object Notation) sebagai format komunikasi antara merchant dan Veritrans. JSON dikirim menggunakan HTTP(S) POST.

Kami telah menyediakan beberapa plugin eCommerce dan library PHP. Pastikan anda mengecek pilihan plugin dan library yang tersedia sebelum menggunakan API ini untuk mempercepat proses integrasi.

Jenis pembayaran

Berikut adalah beberapa jenis pembayaran yang tersedia di Veritrans API :

  • Credit Card
    • Normal
    • 3D Secure
  • Internet Banking
    • Mandiri ClickPay
    • CIMB Clicks
  • Mobile Banking (TO COME)
    • T-Cash
    • XL-Tunai
  • Virtual Accounts (Bank Transfer) (TO COME)
    • Permata Virtual Account

Request dan Response Veritrans API

Seperti yang telah disampaikan sebelumnya, komunikasi dengan Veritrans API dikirimkan menggunakan HTTP(S). Konten request dan response Veritrans API memiliki format JSON. Bagian ini menjelaskan secara rinci format HTTP(S) request dan response yang digunakan dalam Veritrans API.

Request HTTP(S) ke Veritrans API umumnya memiliki 3 komponen utama:

  1. URL Endpoint
  2. Header Authorization
  3. Header Accept dan Content-Type

API URL Endpoint

Setiap fungsi Veritrans API memiliki URL endpoint yang berbeda. Meskipun URL endpoint berbeda, semua fungsi Veritrans API memiliki BASE_URL yang sama. Berikut adalah BASE_URL untuk environment sandbox dan produksi.

  1. Environment Sandbox:
    https://api.sandbox.veritrans.co.id/v2
  2. Environment Produksi:
    https://api.veritrans.co.id/v2

Environment sandbox digunakan ketika anda sedang dalam proses integrasi dan testing. Semua aktivitas dan transaksi yang dilakukan dalam environment sandbox merupakan data dummy. Jika anda sudah siap untuk melakukan transaksi sebenarnya, gunakan URL endpoint environment produksi.

Pada penjelasan spesifikasi charge transaksi menggunakan Veritrans API, misalnya, anda akan menjumpai penjelasan URL endpoint sebagai berikut:

URL endpoint : BASE_URL/charge

Hal ini berarti URL endpoint charge transaksi adalah

https://api.sandbox.veritrans.co.id/v2/charge

jika anda menggunakan environment sandbox dan 

https://api.veritrans.co.id/v2/charge

jika anda menggunakan environment produksi.

Header Authorization

Untuk melewati proses autentikasi, anda perlu mengirim server_key di header Authorization sebagai spesifikasi standard BASIC AUTH. Server key environment sandbox dapat anda peroleh di Settings - Access Keys MAP.

BASIC AUTH harus mengikuti format sebagai berikut :

Username:Password

Karena tidak ada pasword untuk server key, password dapat dibiarkan kosong. Jika anda memiliki server key seperti berikut :

94960ece-9513-4265-9cf2-67a4da330213

Pasangan username dan password BASIC AUTH akan menjadi:

94960ece-9513-4265-9cf2-67a4da330213:

(Note: pastikan ada karakter “:” setelah server key)

Selanjutnya anda perlu mengubah ke format BASE 64, sehingga hasilnya seperti berikut

OTQ5NjBlY2UtOTUxMy00MjY1LTljZjItNjdhNGRhMzMwMjEzOg==

Berikut adalah contoh header untuk basic auth:

Authorization: Basic OTQ5NjBlY2UtOTUxMy00MjY1LTljZjItNjdhNGRhMzMwMjEzOg==

Pastikan merchant menaruh string “Basic ” sebelum BASE 64 server key.

Header Accept dan Content-Type

Pada Veritrans API, anda akan menjumpai fungsi dengan parameter input dan output berupa JSON. Jika fungsi tersebut memiliki input parameter JSON, tambahkan header HTTP(S):

Content-Type: application/json

Jika fungsi RESTful API memiliki output parameter JSON, tambahkan header HTTP(S):

Accept: application/json

Berikut adalah rangkuman header HTTP(S) request Veritrans API:

Header Value
Accept application/json
Content-Type application/json
Authorization Basic base64(server_key:)
