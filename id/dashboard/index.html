Penggunaan Dashboard

Tutorial di bawah ini akan membantu anda mengoperasikan Veritrans Merchant Admin Portal (MAP). Apabila anda memerlukan bantuan lebih lanjut, silahkan hubungi kami di support@veritrans.co.id atau kunjungi Veritrans support page.

Merchant Admin Portal (MAP) adalah Veritrans Dashboard bagi merchants untuk mengkonfigurasi pengaturan pembayaran dan mengatur transaksi dari pelanggan.

Veritrans Dashboard memiliki berbagai kegunaan yang dapat dimanfaatkan untuk memenuhi kebutuhan merchant.

Sandbox vs. Production

Veritrans Dashboard tersedia untuk Development Environment dan Production Environment:

Sandbox Environment

Diawal mula, merchant akan diminta untuk mendaftar di Development Environment (Sandbox).

URL: https://my.sandbox.veritrans.co.id.

Anda dapat mencoba berbagai metode pembayaran yang tersedia tanpa melakukan transaksi sungguhan di Environment ini. Kredensial untuk mencoba dapat dilihat sini.

Production Environment

Production Environment kami tersedia apabila merchant sudah menyelesaikan integrasi teknis di Development Environment serta melengkapi dokumen persyaratan legal.

URL: https://my.veritrans.co.id.

Merchant dapat menggunakan jasa Veritrans sepenuhnya dan mulai menerima pembayaran dari pelanggan di Production Environment.

Info: Note:
Silahkan hubungi tim support kami di support@veritrans.co.id untuk bantuan migrasi dari Sandbox ke Production Environment.


Status Transaksi

Transaksi dari setiap metode pembayaran akan menghasilkan Status Transaksi. Tabel di bawah ini menjelaskan Status transaksi yang ada di Veritrans

Status Transaksi Definisi
Authorize Transaksi diperbolehkan untuk pengambilan dana tertentu untuk pemilik kartu kredit.
Success Transaksi diterima dan akan diproses untuk Settlement.
Challenge Transaksi kartu kredit dianggap sebagai transaksi yang mencurigakan oleh Veritrans Fraud Detection System. Merchant harus menerima atau menolak transaksi melalui Veritrans Dashboard.
Lihat di bawah ini untuk mengetahui lebih lanjut tentang Fraud Management.
Failure Gagalnya transaksi dapat disebabkan oleh beberapa alasan:
1. Transaksi ditolak oleh bank.
2. Transaksi kartu kredit ditolak oleh Fraud Detection System Veritrans.
3. Pending Transaction belum dibayar atau sudah tidak berlaku.
Pending Transaksi yang asynchronous — Pembayaran harus diselesaikan di luar domain Veritrans — akan menghasilkan Pending status.
Apabila pelanggan sudah menyelesaikan transaksi, statusnya akan berubah menjadi 'success'.
Cancel Transaksi yang dibatalkan (Cancel) disebabkan oleh beberapa alasan:
1. Success Transaction dibatalkan sebelum Settlement.
2. Challenge Transaction ditolak oleh merchant.
Settlement Perpindahan dana dari akun pelanggan ke akun merchant.
Lihat di bawah ini untuk mengetahui lebih lanjut tentang Settlement.


Info: Note:
Merchant dapat memeriksa rincian Transaction Status dengan mengklik Order ID tertentu.


Settlement

Settlement adalah saat dimana dana berpindah tangan dari akun pelanggan ke akun merchant. 
Setiap metode pembayaran memiliki waktu Settlement yang berbeda:

  • Transaksi Non-Credit Card:
    Success Transaction akan langsung diproses untuk Settlement.
  • Transaksi Credit Card:
    Success Transactions dari pukul 00:00 - 23:59 WIB akan diproses untuk Settlement keesokan harinya dalam satu batch pada pukul 16:00 WIB.


Pembatalan Transaksi

Merchant dapat membatalkan transaksi (Cancel) sebelum pembayaran diproses untuk Settlement karena dana belum berpindah tangan ke akun merchant.

Berikut adalah metode pembayaran yang dapat dibatalkan sebelum proses Settlement:

Platform Metode Pembayaran
Card Payment Credit Card
Bank Transfer Permata Virtual Account, Mandiri Bill Payment
Direct Debit CIMB Clicks, e-pay BRI, BCA KlikPay
e-Wallet XL Tunai, BBM Money


Untuk membatalkan transaksi yang belum diproses untuk Settlement:

  1. Masuk ke tab Transaction.
  2. Klik 'Order ID' transaksi.
  3. Di pojok kanan atas, klik 'Action: Cancel'.

Info: Note:
Transaksi yang sudah diproses untuk Settlement tidak dapat dibatalkan. Tapi Anda bisa request refund dengan menghubungi support@veritrans.co.id.


Pengertian Fraud Management

Pembayaran melalui Credit Card di Veritrans akan di filter oleh Aegis — Fraud Detection System Veritrans. 
Aegis akan memberikan beberapa rekomendasi:

  1. Accept - Pembayaran berhasil melewati peraturan deteksi kecurangan dari Aegis.
  2. Challenge - Pembayaran dianggap mencurigakan oleh Aegis.
  3. Deny - Pembayaran gagal melewati peraturan deteksi kecurangan dari Aegis.

Challenge transaction dapat diatur secara manual oleh merchant dengan:

  1. Masuk ke tab Transaction.
  2. Klik status pembayaran 'Challenge'.
  3. Merchant dapat memilih 'Accept' untuk menerima transaksi atau 'Deny' untuk menolak transaksi.


Mendapatkan Laporan Transaksi

Merchant bisa mendapatkan laporan transaksi dalam bentuk Excel atau CSV.

Ikuti langkah berikut ini untuk mendapatkan laporan transaksi Anda:

  1. Masuk ke tab Transactions.
  2. Di pojok kanan atas, Anda akan menemukan link untuk 'Download as Excel or CSV'.
  3. Pilih data yang Anda ingin masukkan ke laporan Anda.
  4. Klik 'Download CSV' atau 'Download Excel' untuk mengunduh laporan transaksi.

Merchant juga bisa mendapatkan laporan transaksi harian via email dengan:

  1. Masuk ke tab Settings.
  2. Pada 'daily report', atur flag dan masukkan email anda.

Merchant juga dapat menyaring transaksi dengan mengklik 'Search' di pojok kiri atas pada tab Transaction.


Pengambilan Dana Di Veritrans

Merchant Veritrans bisa menarik dana 7 hari setelah transaksi dilakukan. 
Merchant dapat memilih waktu yang diinginkan untuk mentransfer dana.

Ikuti langkah berikut untuk mentransfer dana Anda:

  1. Log in ke MAP Production.
  2. Masuk ke tab Settings. 
    Pada 'billings', masukkan akun bank anda untuk mentransfer dana.
  3. Masuk ke tab Billings. 
    Merchant akan mendapatkan informasi mengenai dana yang dapat diambil.
  4. Klik 'Transfer'.
  5. Merchant akan mendapatkan uangnya di hari yang sama jika request dilakukan sebelum jam 11 siang. 
    Merchant akan mendapatkan dana pada keesokan harinya bila request dilakukan pada/setelah pukul 11 siang. 
    Delay mungkin terjadi pada hari libur. Silahkan hubungi kami jika uang Anda belum tiba dalam 3 hari.

Caution: Caution!
Akun Bank anda harus sesuai dengan Akun Bank yang didaftarkan di Perjanjian Kerja Sama.


Pengaturan Pengguna Dashboard

Ikuti langkah berikut untuk menambahkan akun Veritrans Dashboard untuk anggota tim Anda:

  1. Log in ke Veritrans Dashboard Anda.
  2. Masuk ke tab Account pada User Management.

Merchant dapat menambahkan user dan menentukan peranan, contoh:

  • Administrator (akses ke seluruh menu MAP menu, biasanya ditempati oleh pemilik bisnis/CEO/GM).
  • Operations (akses ke informasi transaksi dan status. Bisa membatalkan (void), menerima, dan menolak transaksi).
  • Finance (akses ke informasi yang berkaitan dengan tagihan. Dapat menarik uang dan mengedit info akun Bank).
  • Developer/ IT Staff (akses ke setting dan integrasi VT-Web/VT-Direct).
  • Customer Service (akses ke informasi transaksi dan status).

Demi keamanan, kami sarankan Anda menentukan pengguna dengan hati-hati, terutama di Production Environment.

Anda juga dapat memeriksa aktivitas user di Dashboard pada 'Activity Log' atau memasang IP address khusus untuk login ke Veritrans Dashboard pada 'IP Whitelist'.


Pengaturan Nama Custom Fields

Veritrans menyediakan merchant 3 custom field untuk kegunaan yang beragam.

Ikuti langkah-langkah di bawah ini untuk mengatur Custom Field Anda:

  1. Masuk ke tab Configuration pada General Settings, atur label Custom Field.
  2. Request Charge Transaction dengan value Custom Fields.
  3. Label dan value Custom Fields tersebut dapat dicek di Order ID transaksi.
  4. Veritrans akan memasukkan Custom Fields tersebut pada saat pengiriman HTTP POST Notification.
